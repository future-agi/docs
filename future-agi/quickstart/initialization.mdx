# Initializing the Future AGI Client

To begin using the Future AGI Client, you'll need to set up your environment and initialize the client with your API credentials. Follow this step-by-step guide to get started quickly and securely.

## 1. Obtain Your API Keys

When you create an account with Future AGI, you'll receive a set of API keys. These keys are crucial for authenticating your requests and connecting your data to the right spaces.

![API Keys Interface](/images/product-guides/quickstart/keys_api.png)

## 2. Set Up Your Environment

```python
export FI_API_KEY=your_api_key
export FI_SECRET_KEY=your_secret_key
export FI_API_URL="https://api.futureagi.com"
```

Then, reload your shell configuration:

```python
source ~/.bashrc
```

## 3. Instrument Your Code

If you are using the Future AGI Python client, you can initialize the client in your code as follows:

```python
import os
from fi.client import Client

api_key = os.environ["FI_API_KEY"]
secret_key = os.environ["FI_SECRET_KEY"]
base_url = os.environ["FI_API_URL"]

logger = Client(
    api_key=api_key,
    secret_key=secret_key,
    uri=base_url,
    max_workers=8,
    max_queue_bound=5000,
    timeout=200,
    additional_headers=None,
)
```

Here’s a breakdown of the parameters used to initialize the Fi Client:

| Key Name            | Descriptions                                                                                                 |
| ------------------- | ------------------------------------------------------------------------------------------------------------ |
| `api_key`           | The provided API key associated with your account.                                                           |
| `secret_key`         | The secret key used to connect records to your spaces.                                                       |
| `uri`                | The base URL to send your records to Fi AI.                                                                  |
| `max_workers`        | The maximum number of concurrent requests to Fi AI. The default is 8.                                        |
| `max_queue_bound`   | The maximum number of concurrent future objects generated for publishing to Fi AI. The default is 5000.      |
| `timeout`             | The duration (in seconds) to wait for the server to send data before timing out. The default is 200 seconds. |
| `additional_headers` | A dictionary of additional headers to append to the request.                                                 |

**Note**: It is strongly suggested to store the API key and other sensitive information as secrets or environment variables to maintain security and prevent accidental exposure of your credentials.
