This guide walks you through the complete process of setting up and running an experiment in Future AGI platform. You will learn how to execute experiments, analyse results, compare different configurations, and ultimately select the best-performing setup based on structured evaluation.

---

## **1. Building a Dataset**

Before running any experiments, we need to make sure we have a well-structured dataset. This dataset provides the necessary input information, allowing the model to generate responses that can later be evaluated.

After the dataset is available, it is essential to verify that the structure is correct. You can do this by inspecting the table in the dashboard, ensuring that all fields are appropriately populated. Once verified, the dataset is now available for experiments.

## **2. Creating an Experiment**

To create a new experiment:

1. Navigate to the **Experiments** tab within the dataset view.
2. Click **"Create Experiment"** to initiate the setup.
3. Assign a **name** to the experiment for easy identification.
4. Select the **dataset** that will serve as input for testing.

[screenshot]

---

## 3. Configuring Experiment

[screenshot]

### **a. Input Source**

- Select the **column** in the dataset that contains the input text for the model.
- This column provides the context for the experiment and determines how the model will generate responses.

### **b. Model Selection and Configuration**

- Choose the **LLM model** that will process the input.
- Adjust key parameters to control how the model generates responses:
    - **Temperature**: Controls randomness; lower values produce more deterministic outputs.
    - **Top P**: Regulates sampling diversity by restricting token probability mass.
    - **Max Tokens**: Defines the maximum response length.
    - **Presence & Frequency Penalty**: Adjusts token repetition behavior for more diverse outputs.
    - **Response Format**: Specifies the expected structure of the model's output.

### **c. Prompt Template**

- Define the **prompt template** that will be used during inference.
- Use placeholders **{{variable}}** to dynamically inject dataset column values into the prompt.
- Ensure that the prompt aligns with the goal of the experiment.

### **d. Setting Up Evaluation Metrics**

- Evaluation metrics help quantify the experiment's success.
- You can either:
    - **Create new evaluation metrics** tailored to the experiment.
    - **Use saved evaluations** from previous experiments to maintain consistency.

## **4. Running the Experiment**

Once the experiment is fully configured:

1. Review all settings to ensure alignment with experiment objectives.
2. Click **"Save and Run"** to begin the experiment.
3. Monitor execution progress and results in the **Summary** tab.

[screenshot]

At this stage, the system processes the dataset through the configured model, applying the defined prompt structure and evaluation criteria. Once completed, the experiment results can be analysed for further refinement or comparison against other configurations.

---

## 5. **Choosing the Best Prompt**

After an experiment is complete, the results must be analysed to determine which configuration and prompt performed best.

### **Accessing Experiment Results**

- Navigate to the **Experiments** tab and select the completed experiment.
- Go to the **Summary** tab to view detailed performance metrics.
- Compare key parameters such as response time, token usage, accuracy, and quality scores.

[screenshot]

### **Selecting the Winner**

1. Click **"Choose Winner"** in the summary view.
2. Adjust the weight of different metrics (e.g., response quality vs. response time) to prioritise factors most relevant to your use case.
3. Once satisfied, confirm your selection.

[Screenshot]

The winning configuration will be identified as the best-performing setup, serving as the optimal choice for deployment and future iterations in production.
