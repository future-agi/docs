## Add attributes to a span

Attributes are key/value pairs that provide more information about the operation being traced. They help paint a complete picture of what's happening in your application.

To avoid naming conflicts with [semantic conventions](/future-agi/products/observability/tracing-manual/semantic-conventions), it's recommended to prefix your custom attributes with your company name (e.g., mycompany.).

<Tabs>
  <Tab title="Python" language="python">
```python
from opentelemetry import trace

current_span = trace.get_current_span()

current_span.set_attribute("operation.value", 1)
current_span.set_attribute("operation.name", "Saying hello!")
current_span.set_attribute("operation.other-stuff", [1, 2])
```
  </Tab>
  <Tab title="Typescript" language="typescript">
```typescript
import { trace, context } from "@opentelemetry/api";

const currentSpan = trace.getSpan(context.active());

if (currentSpan) {
    currentSpan.setAttribute("mycompany.operation.value", 1);
    currentSpan.setAttribute("mycompany.operation.name", "Saying hello!");
    currentSpan.setAttribute("mycompany.operation.other-stuff", [1, 2]);
}
```
  </Tab>
</Tabs>

## Leveraging Semantic Convention Attributes

Semantic Conventions provides a structured schema to represent common LLM application attributes. These are well known names for items like messages, prompt templates, metadata, and more. We've built a set of semantic conventions as part of the traceAI package. 

Defining attributes is vital for comprehending the data and message flow within your LLM application and helps in debugging and analysis. By defining attributes like `OUTPUT_VALUE` and `OUTPUT_MESSAGES`, you can capture essential output information and interaction messages within a span's context. This enables you to log the response and systematically categorize and store messages exchanged by components.

To use traceAI Semantic Attributes, ensure you have the appropriate FI Instrumentation Package installed:

<Tabs>
  <Tab title="Python" language="python">
```bash
pip install fi-instrumentation-otel
```

Then run the following to set semantic attributes:

```python
from opentelemetry import trace # Assuming span is current_span or obtained otherwise
from fi_instrumentation.fi_types import SpanAttributes, MessageAttributes # Assuming these constants and 'response' are defined

span = trace.get_current_span() # Example: get current span

if span.is_recording(): # Check if span is recording before setting attributes
    span.set_attribute(SpanAttributes.OUTPUT_VALUE, response)

    # This shows up under `output_messages` tab on the span page
    span.set_attribute(
        f"{SpanAttributes.LLM_OUTPUT_MESSAGES}.0.{MessageAttributes.MESSAGE_ROLE}",
        "user",
    )
    span.set_attribute(
        f"{SpanAttributes.LLM_OUTPUT_MESSAGES}.0.{MessageAttributes.MESSAGE_CONTENT}",
        response,
    )
```

## Adding attributes to multiple spans at once

Our tracing system allows you to set attributes at the OpenTelemetry Context level, which automatically propagates to child spans within a parent trace. In OpenTelemetry, this is often achieved using Baggage. Attributes set in Baggage can be picked up by instrumentation (like traceAI's auto-instrumentation) and added to spans.

Key Context Attributes include:

- **Metadata**: Metadata associated with a span.
- **Tags**: List of tags to give the span a category.
- **Session ID**: Unique identifier for a session.
- **User ID**: Unique identifier for a user.
- **Prompt Template**:
  - Template: Used to generate prompts as Python f-strings.
  - Version: The version of the prompt template.
  - Variables: key-value pairs applied to the prompt template.

Below are examples showing how to manage these attributes. The Python examples use helpers from `fi_instrumentation`. The Typescript examples use standard OpenTelemetry JS API (`context` and `propagation` for Baggage).

## `using_metadata`

This context manager enriches the current OpenTelemetry Context with metadata. Our auto-instrumentators will apply this metadata as span attributes following traceAI [semantic conventions](/future-agi/products/observability/tracing-manual/semantic-conventions). The metadata must be provided as a string-keyed dictionary, which will be JSON-serialized in the context.

```python
    "key-1": value_1,
    "key-2": value_2,
}
with using_metadata(metadata):
    # Calls within this block will generate spans with the attributes:
    # "metadata" = "{"key-1": value_1, "key-2": value_2, ... }" # JSON serialized
    pass # Your code here
```
  </Tab>

## `using_prompt_template`

Context manager to add a prompt template (including its version and variables) to the current OpenTelemetry Context. traceAI auto instrumentators will read this Context and pass the prompt template fields as span attributes, following the traceAI [semantic conventions](/future-agi/products/observability/tracing-manual/semantic-conventions). Its inputs must be of the following type:

- Template: non-empty string.
- Version: non-empty string.
- Variables: a dictionary with string keys. This dictionary will be serialized to JSON when saved to the OTEL Context and remain a JSON string when sent as a span attribute.

## `get_attributes_from_context`

For convenience, our core instrumentation package provides `get_attributes_from_context` to retrieve context attributes. This is particularly useful for manual instrumentation scenarios. In OpenTelemetry JS, you can achieve this by reading from Baggage in the active context.

In the following example, we assume the following are set in the OTEL context (e.g., via Baggage):
```Python
# Python conceptual setup for context
# session_id = "my-session-id"
# user_id = "my-user-id"
# metadata = { "key-1": 1, "key-2": "2" }
# tags = ["tag_1", "tag_2"]
# prompt_template = "Please describe the weather forecast for {city} on {date}"
# # Based on the Python helper, the version string and variables dict are set to result in these attributes:
# # llm.prompt_template.version = "v1.0"
# # llm.prompt_template.variables = {"city": "Johannesburg", "date":"July 11"}
```

We then use a function to extract them from the OTEL context. You can use it in your manual instrumentation to attach these attributes to your spans.

<Tabs>
  <Tab title="Python" language="python">
```python
from fi_instrumentation import get_attributes_from_context
from opentelemetry import trace # Assuming span is current_span or obtained otherwise

span = trace.get_current_span() # Example: get current span
if span.is_recording():
    span.set_attributes(dict(get_attributes_from_context()))
# The span will then have the following attributes attached (example values):
# {
#    'session.id': 'my-session-id',
#    'user.id': 'my-user-id',
#    'metadata': '{"key-1": 1, "key-2": "2"}',
#    'tag.tags': ['tag_1', 'tag_2'],
#    'llm.prompt_template.template': 'Please describe the weather forecast for {city} on {date}',
#    'llm.prompt_template.version': 'v1.0',
#    'llm.prompt_template.variables': '{"city": "Johannesburg", "date": "July 11"}'
# }
```
  </Tab>
  <Tab title="Typescript" language="typescript">
```typescript
import { trace, context, propagation, Attributes } from "@opentelemetry/api";

// This function mimics the Python `get_attributes_from_context`
// by reading known items from OpenTelemetry Baggage.
function getAttributesFromActiveBaggage(): Attributes {
    const currentBaggage = propagation.getBaggage(context.active());
    const attributes: Attributes = {};

    if (currentBaggage) {
        const sessionId = currentBaggage.getEntry("session.id")?.value;
        if (sessionId !== undefined) attributes["session.id"] = sessionId;

        const userId = currentBaggage.getEntry("user.id")?.value;
        if (userId !== undefined) attributes["user.id"] = userId;

        const metadataStr = currentBaggage.getEntry("metadata")?.value;
        if (metadataStr !== undefined) attributes["metadata"] = metadataStr; // Kept as JSON string

        const tagsStr = currentBaggage.getEntry("tag.tags")?.value;
        if (tagsStr !== undefined) {
            try {
                const parsedTags = JSON.parse(tagsStr);
                if (Array.isArray(parsedTags) && parsedTags.every(item => typeof item === 'string')) {
                    attributes["tag.tags"] = parsedTags as string[];
                } else {
                    attributes["tag.tags"] = tagsStr;
                }
            } catch (e) {
                attributes["tag.tags"] = tagsStr;
            }
        }

        const template = currentBaggage.getEntry("llm.prompt_template.template")?.value;
        if (template !== undefined) attributes["llm.prompt_template.template"] = template;

        const version = currentBaggage.getEntry("llm.prompt_template.version")?.value;
        if (version !== undefined) attributes["llm.prompt_template.version"] = version;

        const variablesStr = currentBaggage.getEntry("llm.prompt_template.variables")?.value;
        if (variablesStr !== undefined) attributes["llm.prompt_template.variables"] = variablesStr; // Kept as JSON string
    }
    return attributes;
}
```
