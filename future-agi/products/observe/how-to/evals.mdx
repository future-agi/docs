## Evals Reference
Here is a list of all the evals and config for all the evals we support for observation spans.

### Conversation Coherence
Ensures logical and consistent conversation flow. [Learn more →](/future-agi/products/evaluation/eval-definition/conversation-coherence)

**Config:**
- **model**: `string` (default: `"gpt-4o-mini"`)

**Mapping:**
- **messages**

### Conversation Resolution
Checks if the conversation arrives at a clear resolution. [Learn more →](/future-agi/products/evaluation/eval-definition/conversation-resolution)

**Config:**
- **model**: `string` (default: `"gpt-4o-mini"`)

**Mapping:**
- **messages**

### Deterministic Evals
Uses deterministic methods for multi-choice evaluation. [Learn more →](/future-agi/products/evaluation/eval-definition/deterministic-eval)

**Config:**
- **multi_choice**: `bool` (default: `False`)
- **choices**: `list`
- **rule_prompt**: `string`

### Content Moderation
Checks output for disallowed or unsafe content. [Learn more →](/future-agi/products/evaluation/eval-definition/content-moderation)

**Mapping:**
- **text**

## Usage Example

```python
from typing import Dict, Any
from enum import Enum

# Create an eval tag
eval_tag = EvalTag(
    type=EvalTagType.OBSERVATION_SPAN,
    value=EvalSpanKind.LLM,
    eval_name=EvalName.CONTEXT_ADHERENCE,
    config={
        "criteria": "Check if response stays within provided context"
    },
    custom_eval_name="context_check"
