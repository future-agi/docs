This guide provides a detailed walkthrough of setting up and using Observe in your LangChain projects.

---

### **1. Install FutureAGI**
Install the FutureAGI package, which includes the Observe framework. 
```bash
pip install futureagi
```
This package provides the essential tools for tracing, monitoring, and evaluating LangChain applications.

### 2. Configure Environment Variables
Set up your environment with the necessary API keys for OpenAI and FutureAGI. These credentials are crucial for:        
- Accessing OpenAI's language models.
- Authenticating with FutureAGI's observability services.
- Securely transmitting telemetry data.

```python
import os
os.environ["OPENAI_API_KEY"] = "your-openai-api-key"
os.environ["FI_API_KEY"] = "your-futureagi-api-key"
os.environ["FI_SECRET_KEY"] = "your-futureagi-secret-key"
```

### 3. Initialize Trace Provider
The trace provider is the core component that enables observability.
- Creates a new project in the Observe platform
- Establishes a connection for sending telemetry data
- Sets up versioning for tracking changes over time
- Initializes OpenTelemetry integration for distributed tracing

```python
from fi_instrumentation import register
from fi_instrumentation.types import ProjectType

trace_provider = register(
    project_type=ProjectType.OBSERVE,
    project_name="langchain_demo",
    project_version_name="v1"
)
```

### 4. Setup LangChain Instrumentation
Configure automatic instrumentation for your LangChain components. The instrumentation:
- Automatically captures all LangChain operations
- Tracks prompt templates and their parameters
- Monitors model calls and responses
- Records timing information for each step
- Enables detailed tracing of the chain execution

```python
from fi_instrumentation import LangChainInstrumentor
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

LangChainInstrumentor().instrument(tracer_provider=trace_provider)

prompt = ChatPromptTemplate.from_template("{x} {y} {z}?").partial(x="why is", z="blue")
chain = prompt | ChatOpenAI(model_name="gpt-3.5-turbo")
```

### 5. Execute Chain with Observability
Run your chain with full observability enabled:

```python
def run_chain():
    try:
        result = chain.invoke({"y": "sky"})
        print(f"Response: {result}")
    except Exception as e:
        print(f"Error executing chain: {e}")

if __name__ == "__main__":
    run_chain()
```
During execution:
- Each step of the chain is automatically traced
- Prompts and completions are logged
- Errors are captured and recorded
- Performance metrics are collected
- All data is sent to your Observe dashboard

---
With the instrumentation setup from SDK, all telemetry data flows into Observe's UI, where you can interact with real-time dashboards, apply filters, and analyze performance.


### 6. LLM Tracing
LLM Tracing is pivotal for obtaining granular insights into your application's interactions with language models. After adding a project to the observability platform, users can access a detailed view of trace information, which includes:

- **Node Type**: Specifies the type of operation (e.g., input handling, model inference).
- **Trace ID**: A unique identifier that allows you to track individual interactions.
- **Input/Output**: Displays the input provided to the model along with its output.

You can also view the evaluation metrics for your traces here. You can setup the evals from the [Evals & Tasks](#8-evals-and-tasks) section. These metrics quantify response quality, allowing you to apply evaluations that compute key statistics, such as count, average, and standard deviation—and visualize insights through interactive graphs.

---

### 7. Sessions
Sessions enable to group and manage related tracing activities, providing comprehensive insights into the interaction history between users and AI. This includes:
- **Timeframe Filtering**: You can filter sessions by specific time intervals, making trend analysis straightforward.
- **Detailed Interaction Histories**: Access a chronological log of interactions within a session, crucial for understanding conversation flows in applications like chatbots.
- **Contextual Analysis**: View complete session data to analyze user behavior and model response patterns effectively.

---

### 8. Evals & Tasks
This section automates model evaluation and monitoring, streamlining performance analysis by enabling you to define sampling rates, apply filters, set evaluation criteria, and manage tasks. By specifying the percentage of data to evaluate, you can optimize resource efficiency while ensuring meaningful insights. 

[Click here to learn about all the evals provided by FutureAGI.]( /future-agi/products/evaluation/eval-definition/overview)

Filters allow you to narrow evaluations to specific data subsets based on custom criteria, providing a targeted approach to analysis. Evaluation criteria can be established to assess model performance comprehensively, incorporating key metrics that reflect accuracy, consistency, and relevance.

With features like detailed LLM Tracing, session-centric data analysis, and automated evaluation tasks, Observe empowers teams to maintain high-quality, reliable AI systems.
