# Adding Prompt Templates to Spans

By instrumenting the prompt template, users can take full advantage of the Arize prompt playground. You don't need to deploy a new template version to see if prompt text or prompt variables changes have the intended effect. Instead, you can experiment with these changes directly in the playground UI.

## Implementation

We provide a `using_prompt_template` context manager to add a prompt template to the current OpenTelemetry Context. OpenInference auto-instrumentors will read this Context and pass the prompt template fields as span attributes, following the OpenInference semantic conventions.

### Required Parameters

| Parameter  | Type        | Description | Example |
|------------|-------------|-------------|---------|
| template   | str         | The prompt template string | "Please describe the weather forecast for {city} on {date}" |
| version    | str         | Version identifier for the template | "v1.0" |
| variables  | Dict[str]   | Dictionary of variables to fill the template | `{"city": "Johannesburg", "date": "July 11"}` |

### Example Implementation

First, install the required dependencies:

```bash
pip install fi-instrumentation-otel traceai_openai openai
```


Here's a complete example showing how to implement prompt template tracing:

```python
import os
import openai
import opentelemetry
from fi_instrumentation import register, using_prompt_template
from openai import OpenAI
from traceai_openai import OpenAIInstrumentor

# Set up Environment Variables
os.environ["OPENAI_API_KEY"] = "your-openai-api-key"
os.environ["FI_API_KEY"] = "your-futureagi-api-key"
os.environ["FI_SECRET_KEY"] = "your-futureagi-secret-key"

my_first_model = "my first model"

# Setup OTel via our register function
trace_provider = register(
    project_type=ProjectType.EXPERIMENT,
    project_name="Project_name",
    project_version_name="project_version_name",
)
OpenAIInstrumentor().instrument(tracer_provider=trace_provider)

# Setup OpenAI
client = OpenAI()

# Define prompt template and variables
prompt_template = "Please describe the weather forecast for {city} on {date}"
prompt_template_variables = {"city": "Johannesburg", "date":"July 11"}

# Use the context manager to add template information
with using_prompt_template(
    template=prompt_template,
    variables=prompt_template_variables,
    version="v1.0",
):
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {
                "role": "user",
                "content": prompt_template.format(**prompt_template_variables)
            },
        ]
    )
