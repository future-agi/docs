## 2. Manual Instrumentation

In some cases, you might want full control over what is being traced in your application. Future AGI supports OpenTelemetry (OTEL) which means that you can create and customize your spans using the OpenTelemetry Trace API.

## Example code

### Python

### Step 1: Install Prerequisites

Requirements:
- Python >= 3.9, < 3.13
- [Future AGI Instrumentation](https://pypi.org/project/fi-instrumentation-otel/)

```bash
pip install fi-instrumentation-otel
pip install traceAI-openai
```

### Step 2: Set Environment Variables

```python
import os
os.environ["FI_API_KEY"] = "your-futureagi-api-key"
os.environ["FI_SECRET_KEY"] = "your-futureagi-secret-key"
```

### Step 3: Configuring a Tracer

Manually configuring an OTEL tracer involves some boilerplate code that Future AGI takes care of for you. We recommend using the register helper function below to configure a tracer.

```python
from traceai_openai import OpenAIInstrumentor
from fi_instrumentation import register

# Setup OTel via our register function
trace_provider = register(
    project_type=ProjectType.EXPERIMENT,
    project_name="FUTURE_AGI",
    project_version_name="openai-exp",
)
```

### Step 4: Creating Spans

If you're using our Auto-Instrumenters we will take care of automatically creating the spans for your application. You can then further customize those spans.

```python
OpenAIInstrumentor().instrument(tracer_provider=trace_provider)
```

For fully customized spans, you can use manual instrumentation:

```python
from opentelemetry import trace

trace.set_tracer_provider(trace_provider)
tracer = trace.get_tracer(__name__)
```

Next we create spans by starting spans and defining our name and other attributes:

```python
def do_work():
    with tracer.start_as_current_span("span-name") as span:
        # do some work that 'span' will track
        print("doing some work...")
        # When the 'with' block goes out of scope, 'span' is closed for you
```

You can also use start_span to create a span without making it the current span. This is usually done to track concurrent or asynchronous operations.

#### Creating Nested Spans

If you have a distinct sub-operation you'd like to track as a part of another one, you can create span to represent the relationship:

```python
def do_work():
    with tracer.start_as_current_span("parent") as parent:
        # do some work that 'parent' tracks
        print("doing some work...")
        # Create a nested span to track nested work
        with tracer.start_as_current_span("child") as child:
            # do some work that 'child' tracks
            print("doing some nested work...")
            # the nested span is closed when it's out of scope
```

When you view spans in our platform, `child` will be tracked as a nested span under `parent`.

#### Using Span Decorators

```python
@tracer.start_as_current_span("do_work")
def do_work():
    print("doing some work...")
```

Use of the decorator is equivalent to creating the span inside `do_work()` and ending it when `do_work()` is finished.

To use the decorator, you must have a tracer instance in scope for your function declaration.
