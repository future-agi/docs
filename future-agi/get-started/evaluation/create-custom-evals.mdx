## What it is

**Custom evals** let you define your own assessment criteria, rules, and output format instead of using only built-in templates. You write a **rule prompt** (with variables you map to dataset columns or inputs), choose **output type** (pass/fail, score, or choices), pick a **model**, and optionally add tags, description, and **Check Internet** for up-to-date validation. Once created, the custom eval appears in your evaluation list and can be used on datasets, in simulations, and via the SDK like any other template. Use custom evals when you need domain-specific validation, business-rule compliance, or scoring logic that built-in evals don’t cover.

---

## Use cases

- **Domain-specific validation** — Assess content against industry or regulatory standards that aren’t in the default templates.
- **Business rule compliance** — Enforce your organization’s guidelines (tone, format, disclosures) in a repeatable eval.
- **Complex or weighted scoring** — Implement multi-criteria or custom scoring logic via your rule prompt.
- **Custom output formats** — Validate specific response structures or formats (e.g. JSON shape, required fields) with a tailored eval.

---

## How to

You can create custom evals from the **UI** or via the **SDK** (by calling the REST API from your code). After the template is saved, run it from the UI or from the evaluation SDK using the template name.

<Tabs sync={false}>
  <Tab title="Using UI" icon="display">
    <Steps>
      <Step title="Open evaluation creation">
        Open your **dataset**, click **Evaluate** in the top-right, then **Add Evaluation**. Select **Create your own eval** to start the custom-eval flow.
        ![Open evaluation creation](/screenshot/product/evaluation/create-custom-evals/1.png)
      </Step>
      <Step title="Configure basic settings">
        **Name** — Unique name for the eval (lowercase letters, numbers, hyphens, and underscores only; cannot start or end with `-` or `_`). Used when you add the eval to a dataset or call it from the SDK.

        **Model** — **Use Future AGI Models** (e.g. turing_large, turing_flash, turing_small, protect, protect_flash) or **Use other LLMs** (your own or external providers). For model details, see [Future AGI models](/future-agi/get-started/evaluation/future-agi-models) and [Use custom models](/future-agi/get-started/evaluation/use-custom-models).
        ![Configure basic settings](/screenshot/product/evaluation/create-custom-evals/2.png)
      </Step>
      <Step title="Define evaluation rules">
        In **Rule prompt** (criteria), write the instructions the model will follow to evaluate each row. Use **`{{variable_name}}`** for placeholders; you'll map these to dataset columns (or SDK input keys) when you add or run the eval. Be specific about what counts as pass/fail or how to score.
        ![Define evaluation rules](/screenshot/product/evaluation/create-custom-evals/3.png)
      </Step>
      <Step title="Configure output type">
        **Pass/Fail** — Binary result (e.g. 1.0 pass, 0.0 fail). **Percentage (score)** — Numeric score between 0 and 100. **Deterministic choices** — Categorical result; provide a dict of allowed choices.
        ![Configure output type](/screenshot/product/evaluation/create-custom-evals/4.png)
      </Step>
      <Step title="Add optional settings and save">
        **Tags** — For filtering and organization. **Description** — Shown in the evaluation list. **Check Internet** — Allow the eval to use up-to-date information when needed. **Required keys** — List the input variable names the eval expects (e.g. `input`, `output`, `user_query`, `chatbot_response`). Click **Create Evaluation** to save; the new template appears in your list and can be added to datasets or called via the SDK.
        ![Add optional settings and save](/screenshot/product/evaluation/create-custom-evals/5.png)
      </Step>
      <Step title="Run the evaluation">
        In your dataset, click **Evaluate** → **Add Evaluation**, select the custom eval you created, map the **columns** to the rule-prompt variables, then click **Add & Run**. See [Running your first eval](/product/evaluation/how-to/running-your-first-eval) for the full UI flow.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Using SDK" icon="code">
    Custom eval templates are created by calling the Future AGI REST API from your code (e.g. with Python `requests` or curl). The **ai-evaluation** SDK is for *running* evals; it does not include a create-template method, so you use the API directly to create, then run with the SDK.

    <Steps>
      <Step title="Authenticate">
        Use your Future AGI API credentials (e.g. `fi_api_key` and `fi_secret_key`, or your project's auth header). See [Accessing API keys](/admin-settings#accessing-api-keys) for how to obtain them.
      </Step>
      <Step title="POST to create custom eval">
        Send a **POST** request to your API base URL path **`/model-hub/create_custom_evals/`** with a JSON body. Required: **name** (lowercase, numbers, hyphens, underscores only; no leading/trailing `-` or `_`), **criteria** (rule prompt with `{{variable_name}}` placeholders), **output_type** (`"Pass/Fail"`, `"score"`, or `"choices"`), **required_keys** (list of input variable names). Optional: **description**, **tags** (list), **config** (e.g. `{"model": "turing_small"}`), **check_internet** (boolean), **choices** (dict; required when output_type is `"choices"`), **multi_choice** (boolean).
      </Step>
      <Step title="Use the new template">
        The response returns **eval_template_id**. Use this template in the UI (add to dataset and run) or call it from the **SDK** with the template **name** as `eval_templates` and the same keys as **inputs**. See [Running your first eval](/product/evaluation/how-to/running-your-first-eval).
      </Step>
    </Steps>

    Example request:

    ```python
    import requests

    response = requests.post(
        "https://<your-api-host>/model-hub/create_custom_evals/",
        headers={"Authorization": "Bearer <your_token>"},  # or use your project's auth
        json={
            "name": "chatbot_politeness_and_relevance",
            "description": "Evaluates if the response is polite and relevant.",
            "criteria": "Evaluate: 1) Politeness. 2) Relevance to: {{user_query}}. Response: {{chatbot_response}}. Pass only if both.",
            "output_type": "Pass/Fail",
            "required_keys": ["user_query", "chatbot_response"],
            "config": {"model": "turing_small"},
            "check_internet": False,
            "tags": ["customer-service"],
        },
    )
    result = response.json()
    eval_template_id = result.get("eval_template_id")  # use in UI or SDK by name
    ```

</Tabs>
