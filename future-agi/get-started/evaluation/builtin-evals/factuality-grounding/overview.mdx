## Factuality & grounding

Evals in this category measure **factuality and grounding**: whether output is factually correct, consistent with context, free of hallucination, and grounded in the provided source.

| Eval | Description | Inputs | Output | Use case |
| --- | --- | --- | --- | --- |
| Factual Accuracy | Verifies that output is factually correct given the provided information. | `input`, `output`, optional `context` | Score | Fact-checking model answers. |
| Is Factually Consistent | Checks that output is factually consistent with input or context. | `input`, `output`, optional `context` | Pass/Fail | RAG or context-based QA. |
| Detect Hallucination | Identifies when the model fabricated facts or added information not in input or context. | `input`, `output`, `context` | Pass/Fail | Reducing hallucinations. |
| Groundedness | Assesses whether the response is firmly based on the provided context. | `output`, `context` | Score | Ensuring answers stay grounded. |
| Context Adherence | Measures if output contains information not present in the given context. | `output`, `context` | Score | Preventing context drift. |
| Context Relevance | Evaluates whether retrieved context is sufficient and relevant to answer the query. | `context`, `input` | Score | RAG retrieval quality. |
| Context Similarity | Evaluates how closely the provided context matches the expected context. | `context`, reference/expected | Score | Retrieval or context alignment. |
