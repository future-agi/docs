## Similarity & string metrics

Evals in this category provide **similarity and string metrics**: lexical overlap (BLEU, ROUGE), semantic similarity (embeddings), fuzzy match, Levenshtein, numeric similarity, and semantic list or answer comparison.

| Eval | Description | Inputs | Output | Use case |
| --- | --- | --- | --- | --- |
| BLEU Score | Measures lexical overlap between generated and reference text. | `reference`, `hypothesis` | Score | Translation, generation quality. |
| ROUGE Score | Recall-oriented measurement of lexical overlap between hypothesis and reference. | `reference`, `hypothesis` | Score | Summarization, extraction. |
| Embedding Similarity | Measures semantic similarity between generated and reference content. | `output`, `expected` | Score | Meaning-level comparison. |
| Fuzzy Match | Compares two texts for approximate similarity (wording, spelling, formatting). | `output`, `expected` (or `text`, `expected_text`) | Pass/Fail or score | Relaxed string matching. |
| Levenshtein Similarity | Measures similarity by minimum single-character edits between two texts. | `output`, `expected` | Score | Edit distance, string similarity. |
| Numeric Similarity | Extracts numeric values and computes absolute or normalised difference vs reference. | `output`, `expected` | Score | Numeric or metric comparison. |
| Semantic List Contains | Checks if a response semantically contains reference phrases or keywords. | `output`, `expected` | Score | Required content or keywords. |
| Answer Similarity | Assesses similarity between expected and actual response using comparison methods. | (expected vs actual) | Score | Expected-vs-actual answer comparison. |
