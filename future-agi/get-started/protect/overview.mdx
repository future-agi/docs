## What is Protect?

Protect is Future AGI's real-time safety layer for AI applications. It sits between your AI model and your users, screening every input and output for harmful content before anything reaches the end user.

Unlike traditional safety checks that run offline or in batches, Protect works **live** — in the same moment your model processes a request. Potential violations are caught instantly, not discovered after the fact.
<Info>
Think of Protect as a smart gatekeeper: every message in, every response out passes through a safety check. If something fails, it's blocked or flagged automatically — with a clear explanation of why.
</Info>

Built on Google's **Gemma 3n** architecture with specialized fine-tuned adapters, Protect natively handles **text, images, and audio** without requiring separate preprocessing pipelines or any configuration changes per modality.

## The Four Safety Dimensions

Protect evaluates content across four critical areas, each powered by a specialized detection model.

<CardGroup cols={2}>
  <Card title="Content Moderation" icon="shield-halved">
    Detects harmful or offensive language including hate speech, threats, harassment, and toxic content.

    Goes beyond keyword matching — understands context to minimize false positives while catching genuine violations.
  </Card>
  <Card title="Bias Detection" icon="scale-balanced">
    Identifies gender-based discrimination, stereotyping, and sexist language in AI outputs.

    Recognizes subtle forms of bias beyond surface-level patterns, promoting fairness and inclusivity.
  </Card>
  <Card title="Security" icon="lock">
    Identifies adversarial attempts to manipulate your AI through prompt injection attacks.

    Detects instruction overrides, unauthorized role assumption, and safety guideline bypass attempts.
  </Card>
  <Card title="Data Privacy Compliance" icon="user-shield">
    Evaluates content for PII including names, emails, phone numbers, financial data, and health records.

    Ensures compliance with GDPR, HIPAA, and other data privacy standards.
  </Card>
</CardGroup>

## Use Cases

Protect supports a wide range of real-world applications:

- **Content moderation** — Flag harmful or inappropriate content across text, images, and video; detect hate speech, misinformation, and abusive language on social and UGC platforms.
- **Chatbots, voice & customer-facing AI** — Block spam, phishing, and prompt injection; screen chat and voice messages for policy violations in support and voice assistants.
- **Compliance & regulated domains** — Enforce safety in healthcare (e.g. HIPAA), education (e.g. COPPA), and other sensitive contexts; protect PII and filter unverified or harmful advice.
- **Bias, fairness & security** — Detect bias in critical AI outputs (e.g. hiring, lending); defend against adversarial manipulation and misuse.

## QuickStart

Use the Protect module from the FutureAGI SDK to evaluate and filter AI-generated content based on safety metrics like toxicity.

### Step 1 : Install the SDK

```python
pip install ai-evaluation
```

### Step 2 : Set Your API Keys

Make sure to set your API keys as environment variables:

```bash
export FI_API_KEY=xxxx123xxxx
export FI_SECRET_KEY=xxxx12341xxxx
```

### Step 3 : Use Protect

```python
from fi.evals import Protect

# Initialize (reads FI_API_KEY / FI_SECRET_KEY from env if not passed)
protector = Protect()

rules = [{'metric': 'content_moderation'}]

protected_response = protector.protect(
    "AI Generated Message",
    protect_rules=rules,
    action="I'm sorry, I can't help you with that.",
    reason=True,        # include reasons list
    timeout=25000       # milliseconds (25s)
)

print(protected_response)
```

## The Fail-Fast Principle

Protect stops evaluating as soon as one rule fails. This keeps latency low and gives you an immediate signal. If you have four rules and the first one fails, the other three are skipped and listed as `uncompleted_rules`.

<Tip>
Order your rules from most critical to least critical. If security is your top concern, put it first — it will always be checked before the others.
</Tip>

## Multi-Modal Support

Protect works identically across text, images, and audio. You pass your input as a string — Protect detects the format automatically.

| Input Type | What to Pass | Supported Formats |
|---|---|---|
| Text | Plain text string | Any text |
| Image (URL) | `https://...` link | JPG, PNG, WebP, GIF, BMP, TIFF, SVG |
| Image (local file) | `/path/to/image.png` | Same as above (max 20MB) |
| Image (data URI) | `data:image/png;base64,...` | Same as above |
| Audio (URL) | `https://...` link | MP3, WAV |
| Audio (local file) | `/path/to/audio.wav` | Same as above |

<Check>
All four safety dimensions work across all three modalities without any extra setup or separate preprocessing.
</Check>

## Protect vs Protect Flash

Two variants are available — they work identically, the difference is speed and cost.

| | Protect | Protect Flash |
|---|---|---|
| **Best for** | Accuracy-critical, low-volume flows | High-volume production, real-time checks |
| **Latency** | Standard | Lower |
| **Billing (dataset)** | Dataset Protect | Dataset Protect Flash |
| **Billing (standalone)** | Protect Evaluator | Protect Flash Evaluator |
| **call_type value** | `protect` | `protect_flash` |
| **SDK flag** | Default | `protect_flash=True` |

<Warning>
When in doubt, use **Protect Flash** for any production API or large dataset. Use full **Protect** when you need maximum accuracy and can tolerate slightly higher latency.
</Warning>
