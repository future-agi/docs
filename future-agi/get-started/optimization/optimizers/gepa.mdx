GEPA (Genetic Evolutionary Prompt Adaptation) is a powerful evolutionary algorithm that evolves a population of prompts over generations. It uses a teacher model for reflective feedback on failures, guiding the mutation process toward better performance.

## **How It Works**

GEPA uses an evolutionary loop to refine prompts:

1. **Initialization**: Starts with an initial prompt or population of prompts.
2. **Reflection & Evolution**: A reflection LLM analyzes batches of results, identifies failures, and creates a "reflective dataset" that guides the evolution process.
3. **Mutation**: Prompts are mutated based on the reflective feedback to create a new generation of candidates.
4. **Selection**: The best-performing prompts are selected to continue to the next generation.

<Steps>
  <Step title="Seed Population">
    Start with an initial prompt.
  </Step>
  
  <Step title="Evaluate and Reflect">
    Run the prompt population, analyze failures with a reflection model.
  </Step>
  
  <Step title="Evolve Prompts">
    Mutate prompts based on reflection to create a new generation.
  </Step>
  
  <Step title="Select Best">
    The process repeats until a budget (e.g., max metric calls) is met.
  </Step>
</Steps>

## **Underlying Research**

GEPA is based on recent advancements in evolutionary algorithms for prompt engineering, showing significant gains over traditional methods.

- **Core Paper**: The method is detailed in "[GEPA: Reflective Prompt Evolution Can Outperform Reinforcement ...](https://arxiv.org/abs/2507.19457)", which demonstrates that it can outperform RL-based methods with far fewer evaluations.
- **Efficiency**: As highlighted by the Databricks Blog, GEPA can lead to massive cost reductions for agent optimization. It is integrated into leading optimization frameworks like Opik and SuperOptiX.
