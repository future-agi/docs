## What it is

Simulate is a testing and evaluation layer for AI agents and prompts. You define **agent definitions** (voice or text) and **scenarios** (graph-based flows, scripts, or dataset-backed cases). You then create **run tests** that combine an agent (or prompt template) with selected scenarios and eval configs. When you execute a run test, the system runs **test executions**: it simulates conversations (voice calls or chat), stores **call executions** and transcripts, runs **evals** on the results, and surfaces **KPIs**, **eval summaries**, and **session comparison** so you can see how well the agent performed and where it failed or diverged. **Personas** and **simulator agents** help shape how the “customer” side behaves in scenarios. You can also run **prompt simulations** (e.g. Prompt Workbench) and use **agent prompt optimiser** to tune prompts based on these runs.

## Purpose

**Simulate** exists to let you run your AI agents and prompt templates through controlled, repeatable tests and evaluate how they perform. The main goals are:

- **Test agents before or after changes** – Run the same scenarios against different agent versions or prompt configs so you can compare behaviour and catch regressions.
- **Validate behaviour in realistic flows** – Use scenarios (conversation graphs, scripts, or dataset-driven flows) to simulate user intents and paths instead of one-off prompts.
- **Evaluate quality at scale** – Execute many “calls” or chat sessions (voice or text), collect transcripts and metrics, then run evals and view KPIs, summaries, and session comparisons.
- **Support both voice and text** – Work with voice agents (e.g. via providers like Vapi) and text/chat simulations so you can test phone-style and chat-style experiences in one place.
- **Improve agents over time** – Use eval results, session comparison, and (where available) agent prompt optimiser runs to refine prompts and agent behaviour.

In short: Simulate is for **testing, validating, and improving** voice and text AI agents (and related prompts) using scenarios and structured evaluation.
## What you can do

- Create and manage **scenarios** (graph, script, or dataset type) and link them to agent definitions or prompt templates.
- Define **agent definitions** (voice/text), configure provider and behaviour, and manage **agent versions** for A/B or before/after comparison.
- Create **run tests** that tie an agent (or prompt) to scenarios and eval configs, then **execute** to produce test executions and call executions.
- Run **voice calls** (e.g. via Vapi) or **chat simulations** against your agent and review **transcripts**, **call logs**, and **session comparison**.
- Attach **evals** to run tests, run them on completed executions, and view **eval summaries**, **explanation summaries**, and **analytics**.
- Export results (e.g. CSV), **rerun** calls or test executions, and use **agent prompt optimiser** (and related analysis) to improve agent behaviour from simulation outcomes.
