## Overview

Prism supports full SSE streaming, identical to the OpenAI streaming format. Set `"stream": true` in your request and receive response chunks in real-time.

Streaming works across all providers — Prism translates each provider's native streaming format to the standard OpenAI SSE format.

---

## Basic Streaming

<CodeGroup>
```bash cURL
curl -N $PRISM_BASE_URL/v1/chat/completions \
  -H "Authorization: Bearer $PRISM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o-mini",
    "messages": [{"role": "user", "content": "Write a haiku about code"}],
    "stream": true
  }'
```

```python Python
from prism import Prism

client = Prism(
    api_key="sk-prism-your-key-here",
    base_url="https://prism.futureagi.com",
)

stream = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Write a haiku about code"}],
    stream=True,
)

for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="", flush=True)
print()
```

```typescript TypeScript
import { Prism } from '@futureagi/prism';

const client = new Prism({
  apiKey: 'sk-prism-your-key-here',
  baseUrl: 'https://prism.futureagi.com',
});

const stream = await client.chat.completions.create({
  model: 'gpt-4o-mini',
  messages: [{ role: 'user', content: 'Write a haiku about code' }],
  stream: true,
});

for await (const chunk of stream) {
  process.stdout.write(chunk.choices[0]?.delta?.content || '');
}
```
</CodeGroup>

## Stream Manager

For a cleaner streaming experience, use the `.stream()` method. It provides a managed context with a dedicated text stream and access to the full completion after streaming finishes.

<CodeGroup>
```python Python
from prism import Prism

client = Prism(
    api_key="sk-prism-your-key-here",
    base_url="https://prism.futureagi.com",
)

with client.chat.completions.stream(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Write a haiku about code"}],
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)

    # Access the full completion after streaming
    completion = stream.get_final_completion()
    print(f"\nTokens used: {completion.usage.total_tokens}")
```

```typescript TypeScript
import { Prism } from '@futureagi/prism';

const client = new Prism({
  apiKey: 'sk-prism-your-key-here',
  baseUrl: 'https://prism.futureagi.com',
});

const stream = await client.chat.completions.stream({
  model: 'gpt-4o-mini',
  messages: [{ role: 'user', content: 'Write a haiku about code' }],
});

for await (const text of stream.textStream()) {
  process.stdout.write(text);
}

const completion = stream.getFinalCompletion();
console.log(`\nTokens used: ${completion.usage?.total_tokens}`);
```
</CodeGroup>

## SSE Format

Prism streams events in the standard OpenAI format:

```
data: {"id":"chatcmpl-abc123","choices":[{"index":0,"delta":{"role":"assistant"},"finish_reason":null}]}

data: {"id":"chatcmpl-abc123","choices":[{"index":0,"delta":{"content":"Silent"},"finish_reason":null}]}

data: {"id":"chatcmpl-abc123","choices":[{"index":0,"delta":{"content":" code"},"finish_reason":null}]}

...

data: {"id":"chatcmpl-abc123","choices":[{"index":0,"delta":{},"finish_reason":"stop"}]}

data: [DONE]
```

## Streaming with Guardrails

Guardrails run on streaming responses too. Post-processing guardrails accumulate the streamed chunks and run safety checks on the assembled output.

If a guardrail triggers during a stream:

- In **Enforce** mode: the stream is terminated with an error event
- In **Monitor** mode: the stream completes normally, and a warning is logged

<Note>
Pre-processing guardrails (PII detection, prompt injection) run on the request before streaming begins. If they trigger in enforce mode, the stream never starts — you get a standard `403` error response.
</Note>

---

## Streaming with Caching

Streaming requests bypass the cache entirely — they are not looked up in or stored to the cache. If you need caching for a frequently repeated prompt, send it as a non-streaming request.

---

## Cross-Provider Streaming

Prism translates streaming formats across providers transparently:

- **Anthropic** uses a different SSE event structure — Prism translates it to the OpenAI format
- **Google Gemini** has its own streaming protocol — Prism normalizes it
- **AWS Bedrock** uses chunked transfer encoding — Prism converts to SSE

Your code always receives the standard OpenAI SSE format regardless of which provider serves the request.
