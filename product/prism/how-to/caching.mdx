## Overview

Prism caches LLM responses to avoid redundant API calls. When a matching request is found in the cache, Prism returns the cached response instantly — no provider call needed.

Every response includes the `X-Prism-Cache` header indicating the cache status:

| Value | Meaning |
|-------|---------|
| `miss` | No cache hit — request was sent to the provider |
| `hit_exact` | Exact match found in cache |
| `hit_semantic` | Semantically similar request found in cache |

---

## Cache Modes

### Exact Match

Caches responses based on an exact match of the request parameters (model, messages, temperature, etc.). Two requests must be identical to produce a cache hit.

### Semantic Cache

Uses embedding similarity to match semantically equivalent requests. For example, "What's the capital of France?" and "Tell me France's capital city" would produce a semantic cache hit even though the text differs.

## Configuration

Configure caching in your org config:

```json
{
  "cache": {
    "enabled": true,
    "mode": "exact",
    "ttl_seconds": 300,
    "max_entries": 1000
  }
}
```

| Setting | Default | Description |
|---------|---------|-------------|
| `enabled` | `true` | Enable or disable caching |
| `mode` | `exact` | `exact` or `semantic` |
| `ttl_seconds` | 300 | How long cached responses are valid (5 minutes) |
| `max_entries` | 1000 | Maximum number of cached entries |

### Via Python SDK

You can configure caching at the client level using `GatewayConfig`. All requests made through this client will inherit the cache settings.

```python
from prism import Prism, GatewayConfig, CacheConfig

client = Prism(
    api_key="sk-prism-your-key",
    base_url="https://prism.futureagi.com",
    config=GatewayConfig(
        cache=CacheConfig(
            enabled=True,
            strategy="exact",  # or "semantic"
            ttl=300,
            namespace="prod",
        ),
    ),
)

# All requests through this client will use caching
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "What is photosynthesis?"}],
)
```

### Via TypeScript SDK

```typescript
import { Prism } from '@futureagi/prism';

const client = new Prism({
  apiKey: 'sk-prism-your-key',
  baseUrl: 'https://prism.futureagi.com',
  config: {
    cache: {
      enabled: true,
      strategy: 'exact',
      ttl: 300,
      namespace: 'prod',
    },
  },
});
```

## Per-Request Cache Control

Override cache behavior on individual requests using headers:

```bash
# Force a cache refresh (bypass cache, but store the new response)
curl $PRISM_BASE_URL/v1/chat/completions \
  -H "Authorization: Bearer $PRISM_API_KEY" \
  -H "x-prism-cache-force-refresh: true" \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4o-mini", "messages": [...]}'

# Disable caching entirely for this request
curl $PRISM_BASE_URL/v1/chat/completions \
  -H "Authorization: Bearer $PRISM_API_KEY" \
  -H "Cache-Control: no-store" \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4o-mini", "messages": [...]}'

# Set a custom TTL for this request (10 minutes)
curl $PRISM_BASE_URL/v1/chat/completions \
  -H "Authorization: Bearer $PRISM_API_KEY" \
  -H "x-prism-cache-ttl: 600" \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4o-mini", "messages": [...]}'
```

| Header | Description |
|--------|-------------|
| `x-prism-cache-force-refresh` | Set to `true` to bypass cache and refresh the entry |
| `Cache-Control: no-store` | Skip caching entirely for this request |
| `x-prism-cache-ttl` | Override the cache TTL in seconds |

### Via Python SDK

Use `extra_headers` to override cache behavior on individual requests:

```python
# Force cache refresh on a specific request
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "What is photosynthesis?"}],
    extra_headers={"x-prism-cache-force-refresh": "true"},
)

# Disable caching for a specific request
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "What is photosynthesis?"}],
    extra_headers={"Cache-Control": "no-store"},
)

# Set a custom TTL for this request (10 minutes)
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "What is photosynthesis?"}],
    extra_headers={"x-prism-cache-ttl": "600"},
)
```

### Via TypeScript SDK

Use `headers` to override cache behavior on individual requests:

```typescript
// Force cache refresh on a specific request
const response = await client.chat.completions.create(
  {
    model: 'gpt-4o-mini',
    messages: [{ role: 'user', content: 'What is photosynthesis?' }],
  },
  { headers: { 'x-prism-cache-force-refresh': 'true' } },
);

// Disable caching for a specific request
const response2 = await client.chat.completions.create(
  {
    model: 'gpt-4o-mini',
    messages: [{ role: 'user', content: 'What is photosynthesis?' }],
  },
  { headers: { 'Cache-Control': 'no-store' } },
);
```

## When Caching Helps

Caching is most effective for:

- **Repeated queries** — FAQ bots, common customer questions, template-based prompts
- **Development and testing** — Avoid burning API credits on the same test prompts
- **High-traffic endpoints** — Reduce provider costs for popular queries

<Note>
Streaming requests are cached too. The full assembled response is stored, and on a cache hit, the cached response is returned as a non-streaming response.
</Note>

---

## Cost Savings

When a cache hit occurs, the `X-Prism-Cost` header returns `0` — no provider API call was made. Track your cache hit rate in the Prism dashboard under **Analytics > Cache**.
