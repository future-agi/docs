## What Are Providers?

Providers are the LLM services that Prism routes your requests to. Each provider has its own API format, authentication, and model catalog. Prism translates between its unified OpenAI-compatible API and each provider's native format automatically.

---

## Supported Providers

<Note>
Any provider with an OpenAI-compatible API works with Prism out of the box — just set `api_format: "openai"`.
</Note>

| Provider | API Format | Models |
|----------|-----------|--------|
| **OpenAI** | `openai` | GPT-4o, GPT-4o-mini, GPT-3.5 Turbo, o1, o3-mini |
| **Anthropic** | `anthropic` | Claude Sonnet 4, Claude Haiku 4.5 |
| **Google Gemini** | `gemini` | Gemini 2.0 Flash, Gemini 2.5 Pro |
| **AWS Bedrock** | `bedrock` | All Bedrock-hosted models |
| **Azure OpenAI** | `azure` | Azure-hosted OpenAI models |
| **Cohere** | `cohere` | Command R, Command R+, Embed |
| **Groq** | `openai` | LLaMA, Mixtral (OpenAI-compatible) |
| **Together AI** | `openai` | LLaMA, Mixtral, Code Llama |
| **Fireworks** | `openai` | LLaMA, Mixtral, FireFunction |
| **Mistral AI** | `openai` | Mistral Large, Mistral Small |
| **DeepInfra** | `openai` | Open-source models |
| **Perplexity** | `openai` | Sonar models |
| **xAI** | `openai` | Grok |
| **OpenRouter** | `openai` | Multi-model aggregator |
| **Ollama** | `openai` | Self-hosted open-source models |
| **vLLM** | `openai` | Self-hosted with vLLM server |
| **LM Studio** | `openai` | Local models |

## Adding a Provider

### Via Dashboard

1. Go to [app.futureagi.com](https://app.futureagi.com) and open your project
2. Navigate to **Prism > Providers** in the sidebar
3. Click **Add Provider**
4. Select the provider type (e.g., OpenAI, Anthropic)
5. Enter your API key and select the models you want to enable
6. Click **Save**

<Tip>
The dashboard is the quickest way to add and manage providers. You can see all configured providers, their enabled models, and health status at a glance.
</Tip>

### Via Python SDK

Install the SDK:

```bash
pip install prism-ai
```

Add a provider by creating an org config with your provider settings:

```python
from prism import Prism

client = Prism(
    api_key="sk-prism-your-key",
    base_url="https://prism.futureagi.com",
    control_plane_url="https://api.futureagi.com",
)

# Add OpenAI and Anthropic as providers
config = client.org_configs.create(
    org_id="your-org-id",
    config={
        "providers": {
            "openai": {
                "api_key": "sk-...",
                "models": ["gpt-4o", "gpt-4o-mini"]
            },
            "anthropic": {
                "api_key": "sk-ant-...",
                "models": ["claude-sonnet-4-20250514", "claude-haiku-4-5-20251001"]
            }
        }
    }
)
```

### Via TypeScript SDK

Install the SDK:

```bash
npm install @futureagi/prism
```

Add a provider by creating an org config:

```typescript
import { Prism } from "@futureagi/prism";

const client = new Prism({
  apiKey: "sk-prism-your-key",
  baseUrl: "https://prism.futureagi.com",
  controlPlaneUrl: "https://api.futureagi.com",
});

const config = await client.orgConfigs.create({
  orgId: "your-org-id",
  config: {
    providers: {
      openai: {
        api_key: "sk-...",
        models: ["gpt-4o", "gpt-4o-mini"],
      },
      anthropic: {
        api_key: "sk-ant-...",
        models: ["claude-sonnet-4-20250514", "claude-haiku-4-5-20251001"],
      },
    },
  },
});
```

<Warning>
Provider API keys are stored encrypted. They are never exposed in API responses or the dashboard — you'll only see a masked version (e.g., `sk-...7x3Q`).
</Warning>

## Switching Providers at Request Time

Once providers are configured, switch between them by changing the model name in your request. Prism uses a unified SDK that routes to the right provider automatically.

### Python

```python
from prism import Prism

client = Prism(
    api_key="sk-prism-your-key",
    base_url="https://prism.futureagi.com",
)

# Uses OpenAI
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Hello"}],
)

# Uses Anthropic — same code, just change the model
response = client.chat.completions.create(
    model="anthropic/claude-haiku-4-5",
    messages=[{"role": "user", "content": "Hello"}],
)

# Uses Gemini
response = client.chat.completions.create(
    model="gemini/gemini-2.0-flash",
    messages=[{"role": "user", "content": "Hello"}],
)
```

### TypeScript

```typescript
import { Prism } from "@futureagi/prism";

const client = new Prism({
  apiKey: "sk-prism-your-key",
  baseUrl: "https://prism.futureagi.com",
});

// Uses OpenAI
const response = await client.chat.completions.create({
  model: "gpt-4o-mini",
  messages: [{ role: "user", content: "Hello" }],
});

// Uses Anthropic — same code, just change the model
const anthropicResponse = await client.chat.completions.create({
  model: "anthropic/claude-haiku-4-5",
  messages: [{ role: "user", content: "Hello" }],
});

// Uses Gemini
const geminiResponse = await client.chat.completions.create({
  model: "gemini/gemini-2.0-flash",
  messages: [{ role: "user", content: "Hello" }],
});
```

No code changes needed beyond the model name. The response is always in OpenAI format regardless of which provider served it.

## Custom & Self-Hosted LLM Providers

Prism is a hosted service, but the LLM providers you connect to it can be anywhere — including your own infrastructure. If you run models with Ollama, vLLM, or LM Studio, you can route Prism requests to them by specifying a custom base URL in the provider config.

### Via Dashboard

1. Navigate to **Prism > Providers** and click **Add Provider**
2. Select **Custom / OpenAI-compatible** as the provider type
3. Enter your self-hosted endpoint URL (e.g., `http://your-server:11434` for Ollama)
4. Add the model names you want to expose
5. Click **Save**

### Via Python SDK

```python
from prism import Prism

client = Prism(
    api_key="sk-prism-your-key",
    base_url="https://prism.futureagi.com",
    control_plane_url="https://api.futureagi.com",
)

config = client.org_configs.create(
    org_id="your-org-id",
    config={
        "providers": {
            "ollama": {
                "base_url": "http://your-server:11434",
                "api_format": "openai",
                "models": ["llama3.1", "codellama"]
            },
            "vllm": {
                "base_url": "http://your-vllm-server:8000",
                "api_format": "openai",
                "models": ["meta-llama/Llama-3.1-70B"]
            }
        }
    }
)
```

<Note>
Your self-hosted endpoint must be reachable from Prism's hosted gateway. Make sure your firewall and network settings allow inbound connections from Prism.
</Note>

## Provider Health

Prism monitors provider health automatically. If a provider starts returning errors or becomes slow, the gateway can:

- **Failover** to an alternative provider (if routing is configured)
- **Circuit break** to temporarily stop sending requests to the unhealthy provider
- **Alert** you via email or webhook

Check provider health status in the dashboard at [app.futureagi.com](https://app.futureagi.com) under **Prism > Providers**. Each provider shows its current status, recent error rates, and latency metrics.
