## Overview

Prism tracks the cost of every LLM request automatically, giving you full visibility into your AI spend. Use cost tracking to:

- **Monitor spend** — See exactly how much each request, model, and provider costs
- **Set budgets** — Prevent runaway costs with configurable spending limits
- **Attribute costs** — Break down spend by team, feature, or user with custom metadata
- **Get alerts** — Receive email notifications when spend crosses thresholds

---

## Per-Request Cost Tracking

Prism calculates the cost of every LLM request automatically. The cost is returned in the `X-Prism-Cost` response header (in USD) and logged for analytics.

```bash
curl -s -D - $PRISM_BASE_URL/v1/chat/completions \
  -H "Authorization: Bearer $PRISM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4o-mini", "messages": [{"role": "user", "content": "Hello"}]}' \
  2>&1 | grep "X-Prism-Cost"

# X-Prism-Cost: 0.000045
```

You can also access per-request cost through the Prism Python SDK:

```python
from prism import Prism

client = Prism(
    api_key="sk-prism-your-key",
    base_url="https://gateway.futureagi.com",
)

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Hello"}],
)

# Access cost from the response metadata
if hasattr(response, 'prism') and response.prism:
    print(f"Cost: ${response.prism.cost}")

# Access cumulative cost across all requests made by this client
print(f"Total cost: ${client.current_cost:.6f}")
```

### How Cost Is Calculated

```
cost = (input_tokens × input_price_per_token) + (output_tokens × output_price_per_token)
```

Prism maintains a model pricing database that's kept up to date with provider pricing. Cache hits return `X-Prism-Cost: 0`.

## Cost Analytics

### Via Dashboard

The [Prism analytics dashboard](https://app.futureagi.com/dashboard/gateway/analytics) provides detailed cost breakdowns:

- **Total spend** over any time period
- **Cost by model** — see which models cost the most
- **Cost by provider** — compare provider spend
- **Cost by API key** — attribute costs to specific keys/applications
- **Cost timeseries** — track spend trends over time

### Via Python SDK

Use the Prism Python SDK to query analytics programmatically:

```python
from prism import Prism

client = Prism(
    api_key="sk-prism-your-key",
    base_url="https://gateway.futureagi.com",
    control_plane_url="https://api.futureagi.com",
)

# Get a high-level overview of spend and usage
overview = client.analytics.overview(start_date="2025-01-01", end_date="2025-01-31")

# Break down costs by model, provider, or API key
costs = client.analytics.cost_breakdown(group_by="model")

# Retrieve latency percentiles
latency = client.analytics.latency_stats(percentiles=[50, 95, 99])

# Compare cost and performance across models
model_cmp = client.analytics.model_comparison(models=["gpt-4o", "claude-sonnet-4-6"])
```

## Budgets

Set spending limits to prevent runaway costs. Budgets can be configured at the org level.

### Via Dashboard

Navigate to **Settings → Budgets** in the [Prism dashboard](https://app.futureagi.com/dashboard/gateway/settings) to create and manage budget limits. You can set the maximum spend, budget period, and alert threshold directly from the UI.

### Via Python SDK

Budgets are configured as part of the org config. You can set the budget via the dashboard or through the org config API:

```python
from prism import Prism

client = Prism(
    api_key="sk-prism-your-key",
    base_url="https://gateway.futureagi.com",
    control_plane_url="https://api.futureagi.com",
)

config = client.org_configs.create(
    org_id="your-org-id",
    config={
        "budgets": {
            "limit": 100.00,
            "period": "monthly",
            "alert_threshold_percent": 80,
        }
    },
)
```

| Setting | Description |
|---------|-------------|
| `budget_limit` | Maximum spend in USD for the period |
| `budget_period` | `daily`, `weekly`, or `monthly` |
| `alert_threshold_percent` | Send an alert when spend reaches this percentage |

When the budget is exceeded, new requests are blocked with an error response until the next period begins. An email alert is also sent to configured recipients when spend crosses the `alert_threshold_percent`.

## Email Alerts

Prism sends email alerts when key cost and reliability events occur — such as budgets being exceeded, error spikes, latency spikes, or guardrail triggers. Configure alert rules to notify your team proactively before issues escalate.

### Via Dashboard

Navigate to **Settings → Alerts** in the [Prism dashboard](https://app.futureagi.com/dashboard/gateway/email-alerts) to create alert rules, choose event types, and manage recipients — all without writing code.

### Via Python SDK

```python
from prism import Prism

client = Prism(
    api_key="sk-prism-your-key",
    base_url="https://gateway.futureagi.com",
    control_plane_url="https://api.futureagi.com",
)

alert = client.alerts.create(
    name="Budget warning",
    condition="cost > 80",
    recipients=["team@example.com"],
    severity="high",
)
```

| Event Type | Triggers when |
|------------|---------------|
| `budget_exceeded` | Spend exceeds the configured budget limit |
| `error_spike` | Error rate spikes above normal |
| `latency_spike` | Average latency spikes above normal |
| `guardrail_triggered` | A guardrail blocks or flags a request |

<Tip>
Configure a cooldown on your alerts to prevent alert flooding. During cooldown, duplicate alerts for the same event type are suppressed.
</Tip>

## Cost Attribution with Metadata

Use custom metadata headers to attribute costs to specific features, users, or teams:

```bash
curl $PRISM_BASE_URL/v1/chat/completions \
  -H "Authorization: Bearer $PRISM_API_KEY" \
  -H 'x-prism-metadata: {"team": "search", "feature": "autocomplete", "user_tier": "premium"}' \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4o-mini", "messages": [...]}'
```

You can also pass metadata via the Prism Python SDK using extra headers:

```python
from prism import Prism

client = Prism(
    api_key="sk-prism-your-key",
    base_url="https://gateway.futureagi.com",
)

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Hello"}],
    extra_headers={
        "x-prism-metadata": '{"team": "search", "feature": "autocomplete", "user_tier": "premium"}'
    },
)
```

Filter analytics by metadata fields in the dashboard to get cost breakdowns by team, feature, or any custom dimension.
