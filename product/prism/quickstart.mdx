## Prerequisites

- A Future AGI account with Prism enabled
- A Prism API key (starts with `sk-prism-`) — generate one from the [dashboard](https://app.futureagi.com)
- At least one LLM provider configured (e.g., OpenAI, Anthropic)

---

## 0. Install the SDK

<CodeGroup>
```bash Python
pip install prism-ai
```

```bash TypeScript
npm install @futureagi/prism
```
</CodeGroup>

---

## 1. Set Your Credentials

```bash
export PRISM_API_KEY="sk-prism-your-key-here"
export PRISM_BASE_URL="https://prism.futureagi.com"
```

---

## 2. Make Your First Request

<CodeGroup>
```bash cURL
curl $PRISM_BASE_URL/v1/chat/completions \
  -H "Authorization: Bearer $PRISM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o-mini",
    "messages": [
      {"role": "user", "content": "What is an AI gateway?"}
    ]
  }'
```

```python Python
from prism import Prism

client = Prism(
    api_key="sk-prism-your-key-here",
    base_url="https://prism.futureagi.com",
)

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "What is an AI gateway?"}],
)

print(response.choices[0].message.content)
```

```typescript TypeScript
import { Prism } from '@futureagi/prism';

const client = new Prism({
  apiKey: 'sk-prism-your-key-here',
  baseUrl: 'https://prism.futureagi.com',
});

const response = await client.chat.completions.create({
  model: 'gpt-4o-mini',
  messages: [{ role: 'user', content: 'What is an AI gateway?' }],
});

console.log(response.choices[0].message.content);
```
</CodeGroup>

<Tip>
The Prism SDK is fully OpenAI-compatible. If you already use the OpenAI SDK, you can keep using it — just point `base_url` at Prism and swap your API key. The Prism SDK adds convenience features like built-in metadata access, but either approach works.
</Tip>

## 3. Check the Response Headers

Every response includes Prism metadata. Inspect them with cURL:

```bash
curl -s -D - $PRISM_BASE_URL/v1/chat/completions \
  -H "Authorization: Bearer $PRISM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o-mini",
    "messages": [{"role": "user", "content": "Hello"}],
    "max_tokens": 10
  }' 2>&1 | grep "X-Prism"
```

```
X-Prism-Request-Id: 01KJQXDQ9W5PMGB4S2KAZS5M3V
X-Prism-Cost: 0.000003
X-Prism-Cache: miss
X-Prism-Latency-Ms: 1376
X-Prism-Model-Used: gpt-4o-mini-2024-07-18
X-Prism-Provider: openai
```

With the Prism SDK, metadata is available directly on the response object:

```python Python
# Prism metadata is available on the response object
if hasattr(response, 'prism') and response.prism:
    print(f"Cost: ${response.prism.cost}")
    print(f"Provider: {response.prism.provider}")
    print(f"Latency: {response.prism.latency_ms}ms")
```

| Header | What it tells you |
|--------|-------------------|
| `X-Prism-Request-Id` | Unique ID for log correlation in the dashboard |
| `X-Prism-Cost` | Cost of this request in USD |
| `X-Prism-Cache` | Whether the response was cached (`miss`, `hit`, `skip`) |
| `X-Prism-Latency-Ms` | Total round-trip latency |
| `X-Prism-Model-Used` | Actual model that served the request |
| `X-Prism-Provider` | Which provider handled it |

## 4. Try Streaming

Prism streams Server-Sent Events (SSE) in real-time, identical to the OpenAI format:

<CodeGroup>
```bash cURL
curl -N $PRISM_BASE_URL/v1/chat/completions \
  -H "Authorization: Bearer $PRISM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o-mini",
    "messages": [{"role": "user", "content": "Count from 1 to 5"}],
    "stream": true
  }'
```

```python Python
from prism import Prism

client = Prism(
    api_key="sk-prism-your-key-here",
    base_url="https://prism.futureagi.com",
)

# Approach 1: stream=True
stream = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Count from 1 to 5"}],
    stream=True,
)

for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="", flush=True)

# Approach 2: .stream() context manager (recommended)
with client.chat.completions.stream(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Count from 1 to 5"}],
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)
```

```typescript TypeScript
import { Prism } from '@futureagi/prism';

const client = new Prism({
  apiKey: 'sk-prism-your-key-here',
  baseUrl: 'https://prism.futureagi.com',
});

// Approach 1: stream: true
const stream = await client.chat.completions.create({
  model: 'gpt-4o-mini',
  messages: [{ role: 'user', content: 'Count from 1 to 5' }],
  stream: true,
});

for await (const chunk of stream) {
  process.stdout.write(chunk.choices[0]?.delta?.content || '');
}

// Approach 2: .stream() (recommended)
const runner = await client.chat.completions.stream({
  model: 'gpt-4o-mini',
  messages: [{ role: 'user', content: 'Count from 1 to 5' }],
});

for await (const text of runner.textStream()) {
  process.stdout.write(text);
}
```
</CodeGroup>

## 5. Switch Providers

The same API works for every provider — just change the model name:

<CodeGroup>
```bash cURL
# OpenAI
curl $PRISM_BASE_URL/v1/chat/completions \
  -H "Authorization: Bearer $PRISM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4o-mini", "messages": [{"role": "user", "content": "Hello"}]}'

# Anthropic (same API format — Prism translates automatically)
curl $PRISM_BASE_URL/v1/chat/completions \
  -H "Authorization: Bearer $PRISM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model": "anthropic/claude-haiku-4-5", "messages": [{"role": "user", "content": "Hello"}]}'

# Google Gemini
curl $PRISM_BASE_URL/v1/chat/completions \
  -H "Authorization: Bearer $PRISM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model": "gemini/gemini-2.0-flash", "messages": [{"role": "user", "content": "Hello"}]}'
```

```python Python
from prism import Prism

client = Prism(
    api_key="sk-prism-your-key-here",
    base_url="https://prism.futureagi.com",
)

# OpenAI
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Hello"}],
)

# Anthropic (same API format — Prism translates automatically)
response = client.chat.completions.create(
    model="anthropic/claude-haiku-4-5",
    messages=[{"role": "user", "content": "Hello"}],
)

# Google Gemini
response = client.chat.completions.create(
    model="gemini/gemini-2.0-flash",
    messages=[{"role": "user", "content": "Hello"}],
)
```

```typescript TypeScript
import { Prism } from '@futureagi/prism';

const client = new Prism({
  apiKey: 'sk-prism-your-key-here',
  baseUrl: 'https://prism.futureagi.com',
});

// OpenAI
const r1 = await client.chat.completions.create({
  model: 'gpt-4o-mini',
  messages: [{ role: 'user', content: 'Hello' }],
});

// Anthropic (same API format — Prism translates automatically)
const r2 = await client.chat.completions.create({
  model: 'anthropic/claude-haiku-4-5',
  messages: [{ role: 'user', content: 'Hello' }],
});

// Google Gemini
const r3 = await client.chat.completions.create({
  model: 'gemini/gemini-2.0-flash',
  messages: [{ role: 'user', content: 'Hello' }],
});
```
</CodeGroup>

No code changes needed. Prism translates the request to each provider's native format and normalizes the response back to OpenAI format.
