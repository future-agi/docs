## About

LLMs in production behave differently from LLMs in development. Prompts that worked in testing can produce hallucinations, inconsistencies, or slow responses under real traffic. Without observability, these issues go unnoticed until a user reports them.

Observability is the ability to understand what an LLM application is doing, why it produced a specific output, and whether that output is correct, safe, and fast enough. Future AGI Observe provides this through tracing, automated evaluation, session analysis, historical trend tracking, and real-time alerting.

---

## Why it matters

- **LLMs are non-deterministic**: The same input can produce different outputs across calls. Observability captures every input-output pair so behavior can be tracked and compared.
- **Failures are silent**: An LLM does not throw an error when it hallucinates. Without evaluation and monitoring, bad outputs reach users undetected.
- **Multi-turn context drifts**: Chatbots and agents make decisions across multiple turns. Session-level observability reveals where conversations break down.
- **Cost and latency vary**: Token usage and response time depend on prompt length, model choice, and provider load. Observability surfaces these metrics for optimization.

---

## Core components

### 1. Tracing and debugging

Every LLM call is captured as a trace with a unique ID. Each trace contains spans that record inputs, outputs, execution time, token counts, and errors.

- **Trace identification**: Every request gets a unique trace ID for tracking and debugging.
- **Response auditing**: Input queries, generated responses, and execution times are logged automatically.
- **Error detection**: Failed completions, latency spikes, and incomplete outputs are highlighted.

> **Use case**: A chatbot generates a misleading response. The trace log shows the exact input, model parameters, and output, making it possible to pinpoint why the hallucination occurred.

### 2. Session-based observability

Multi-turn interactions are grouped into sessions using `session.id`. This connects related traces into a single conversation view.

- **Session grouping**: Cluster multiple interactions within a conversation or task.
- **Conversation analysis**: Evaluate how the LLM performs across a sequence of turns.
- **Performance trends**: Track whether quality stays consistent throughout a session.

> **Use case**: A virtual assistant handling customer queries must maintain relevance over multiple turns. Session observability reveals where the conversation drifts or breaks.

### 3. Automated evaluation and scoring

Observe runs evaluations on traces using predefined or custom templates. Scores are attached directly to spans.

- **Evaluation templates**: Predefined models for coherence, completeness, toxicity, groundedness, and more.
- **Scoring system**: Quantitative metrics assess response quality automatically.
- **Pass/fail flags**: Responses that fall below a threshold are flagged.
- **Real-time evaluation**: Scoring happens as responses are generated, not after the fact.
- **Custom criteria**: Define organization-specific metrics tailored to the use case.

> **Use case**: A content generation model produces AI-written summaries. Observe automatically scores each summary for accuracy, coherence, and relevance as it is generated.

### 4. Historical trend analysis

Observability is not only about real-time monitoring. Tracking model behavior over time reveals regressions and improvements.

- **Performance trends**: Compare past and present behavior to measure progress.
- **Cross-model comparisons**: Analyze different model versions to assess upgrades.
- **Statistical insights**: Standard deviation, percentiles, and response distributions detect long-term anomalies.

> **Use case**: A team updates its legal AI assistant. Historical data shows whether the new version improves or worsens accuracy compared to the previous one.

### 5. Automated issue detection and alerts

Monitors evaluate metrics on a schedule and send notifications when thresholds are breached.

- **Live monitoring**: Track token consumption, processing delays, and response failures in real time.
- **Threshold-based alerts**: Get notified by email or Slack when error rates or latency exceed limits.
- **Workflow automation**: Problematic interactions are flagged and logged for investigation.

> **Use case**: A customer service AI starts generating unexpected responses. Observe triggers an alert, allowing the team to investigate before more users are affected.
