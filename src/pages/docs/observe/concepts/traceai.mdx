## About

OpenTelemetry provides the transport protocol (OTLP) and the span model, but it has no concept of LLM calls, prompt templates, or tool invocations. traceAI fills that gap. It is an open-source package that adds AI-specific semantic conventions, framework instrumentors, and helper utilities on top of OpenTelemetry. traceAI works natively with Future AGI but is compatible with any OpenTelemetry backend.

---

## What traceAI adds

- **Semantic conventions**: Standardized attribute keys (`gen_ai.input.messages`, `gen_ai.span.kind`, `session.id`, etc.) that Future AGI recognizes for filtering, search, and dashboard rendering.
- **Framework instrumentors**: One-line setup for 20+ LLM providers and frameworks. Install the package, call `.instrument()`, and every LLM call is traced automatically.
- **`FITracer`**: A wrapper around the standard OTel tracer with typed decorators (`@tracer.chain`, `@tracer.agent`, `@tracer.tool`), `set_input()`/`set_output()`, and automatic context injection.
- **Context helpers**: `using_session`, `using_user`, `using_metadata`, `using_tags`, `using_prompt_template` propagate attributes to all child spans in a block.
- **`TraceConfig`**: Privacy controls to mask inputs, outputs, images, or embeddings before export.
