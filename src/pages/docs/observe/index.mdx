## About

Observability is Future AGI's production monitoring layer. It automatically captures every LLM call your application makes, groups interactions into sessions, scores outputs using evaluations, and surfaces issues through alerts. You get a continuous, real-time view of quality, cost, and latency across all user interactions.

Production LLM applications change over time: models get updated, user inputs shift, and edge cases emerge that testing never covered. Observability gives you the visibility to catch these changes as they happen. When quality drops, cost spikes, or responses start failing your criteria, you see it in the dashboard and get notified through alerts before it affects users at scale.

## How Observability Connects to Other Features

- **Prototype**: After you promote a winning version in Prototype, its traces continue flowing into Observe so you can monitor production performance against the same quality criteria. [Learn more](/docs/prototype)
- **Evaluation**: Observability uses the same built-in eval templates to score production traces automatically. Any eval you configured in Prototype or Datasets runs the same way here. [Learn more](/docs/evaluation)
- **Alerts**: Observability feeds into the alerting system so you are notified when quality, cost, or latency crosses a threshold in production. [Learn more](/docs/observe/features/alerts)
