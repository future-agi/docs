## What is this page?

A **dashboard** is a custom view of your project data made up of widgets. Each widget runs a query and displays the result as a chart. Use dashboards when you want to combine specific metrics into one place — for example, a single screen with cost, latency, and token usage side by side.

For pre-built, zero-config charts, use [Charts](/docs/observe/features/charts) instead. Dashboards are for when you need a custom layout.

<img src="/images/docs/observe/dashboard-overview.png" alt="Dashboards list — Create dashboard to monitor" style={{ borderRadius: '5px' }} />

---

## When would you use this?

- **One screen for everything you care about** — Combine cost, latency, error rate, and eval scores into one shared view instead of jumping between pages.
- **Monitoring a deployment** — Build a dashboard with the metrics you'll watch after a release.
- **A team-wide view** — Dashboards are shared across the project, so anyone on the team can open the same view.
- **Side-by-side comparison** — Place multiple widgets together to spot correlations (e.g. latency spike + error rate up at the same time).

## The dashboards list

Open **Dashboards** from the left sidebar to see all dashboards in your workspace.

Each row shows: dashboard name · widget count · how long ago it was modified · who created it.

Top of the page:
- **Search** — find a dashboard by name.
- **Created by anyone** filter — narrow to dashboards you or a teammate created.
- **View Docs** — link to documentation.
- **+ Create Dashboard** — make a new dashboard.

---

## Opening a dashboard

Click any row to open that dashboard. Widgets render inline with their charts.

<img src="/images/docs/observe/dashboard-populated.png" alt="A populated dashboard with cost, latency, token, and Untitled widget charts" style={{ borderRadius: '5px' }} />

The header bar has:
- **Breadcrumb** — `Dashboards › <dashboard name>`.
- **Time range buttons** — Today · Yesterday · 7D · 30D · 3M · 6M · 12M · Default. Applies to all widgets on the dashboard.
- **`...`** menu, **Share** icon for dashboard-level actions.

## Adding a widget

Click **+ Add Widget** on a dashboard to open the widget builder.

<img src="/images/docs/observe/dashboard-add-widget.png" alt="Widget builder — date range buttons, chart type, Query and Chart tabs, Metric, Filter, Breakdown" style={{ borderRadius: '5px' }} />

The builder has two tabs (right panel):

### Query tab

Define what data to plot:

- **Metric** — pick what to measure (e.g. cost, latency, token count, eval scores).
- **Filter** — narrow the query to specific spans.
- **Breakdown** — group the metric by a span attribute (model, user, etc.).

<Note>
  **Known limitation (LiveKit users)**: Some trace attributes that LiveKit sets — and that are visible in the [Tracing](/docs/observe/features/llm-tracing) view — are not yet surfaced as **Filter** or **Breakdown** options in Dashboard widgets. A fix is in flight on the platform side (tracked as [TH-4660](https://github.com/future-agi/future-agi/pull/124)). Until it lands, fall back to the Tracing view for filtering by LiveKit-specific attributes.
</Note>


### Chart tab

Pick the visualization (top-right of the header has the chart-type selector — defaults to **Line**).

### Time controls (top of the builder)

- **Time range buttons**: Custom · 30 mins · 6 hrs · Today · Yesterday · 7D · 30D · 3M · 6M · 12M.
- **Granularity dropdown** (default **Day**) — minute/hour/day/week/month bucket size for the time axis.

When all required fields are filled, a preview renders. Click **Save** (top-right) to add the widget to the dashboard.

---

## Creating a dashboard

Click **+ Create Dashboard** on the dashboards list. A new dashboard is created immediately (named "Untitled") and you're dropped into the empty layout. From there:

1. Add widgets via **+ Add Widget**.
2. Drag and resize widgets to arrange the layout.
3. Click into the title to rename, or click "Add description" below it.

<Note>
  Dashboards are scoped to the workspace. Anyone on your team with access to the workspace can view and edit them.
</Note>
