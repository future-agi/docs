## What is this page?

Voice agents are hard to debug — conversations are real-time, multi-turn, and you usually only hear about problems from a user complaint. **Voice Observability** fixes that by pulling call logs from your voice provider into Future AGI automatically. No SDK or code changes needed. Connect a provider with its API key and assistant ID, and every call shows up in Future AGI as a trace with its transcript, recording, latency pipeline, talk ratio, words per minute, and cost.

From there you can run [evals](/docs/observe/features/evals), set [alerts](/docs/observe/features/alerts), filter, and search — the same way you do with text traces.

## When would you use this?

- **Visibility into every call** — See all conversations for a voice agent in one project, no SDK setup required.
- **Quality scoring on calls** — Run evals (relevance, tone, adherence, etc.) on conversation transcripts.
- **Alerts on voice metrics** — Get notified when latency, error rate, or call volume crosses a threshold.
- **Transcripts and recordings for debugging** — Listen to the recording and read the transcript right inside the trace view.
- **Multi-provider** — Vapi and others are supported, so you can monitor regardless of provider.

## Step 1 — Create an Agent Definition

Voice observability is set up by creating an **Agent Definition**. Open **Agent Definition** under **Simulate** in the left sidebar to see all your agents.

<img src="/images/docs/observe/voice-agent-definitions.png" alt="Agent Definitions list with Voice Inbound and Voice Outbound agents" style={{ borderRadius: '5px' }} />

The list shows every agent you've configured, with these columns:

| Column | What it shows |
|---|---|
| **Agent Name** | Your name for the agent. |
| **Type** | `Voice Inbound`, `Voice Outbound`, or `Chat`. |
| **Provider** | The voice provider (e.g. Vapi, Others). |
| **Contact Number** | The phone number associated with the voice agent. |
| **Languages** | Languages the agent supports (e.g. English, Hindi). |
| **Version** | The agent definition version. |

Click **+ Create agent definition** to make a new one.

### The create flow

The create form is a 3-step wizard: **Basic Info → Configuration → Behaviour**.

<img src="/images/docs/observe/voice-create-form.png" alt="Create new agent definition — Basic Info step with Agent type, Agent name, language fields" style={{ borderRadius: '5px' }} />

In **Basic Info** you pick the **Agent type** (Voice Inbound / Voice Outbound / Chat), give the agent a name, and choose the languages it speaks. Click **Next** to move into **Configuration** — that's where the provider details (API key, assistant ID) and observability toggle live for voice agents.

<Note>
  For voice agents, the **API key** and **Assistant ID** from your provider are required. Get them from your provider's dashboard ([Vapi](https://dashboard.vapi.ai), [Retell](https://www.retellai.com/)).
</Note>

## Step 2 — Find your voice project

When you create a voice agent, Future AGI also creates a Tracing **project** with the same name. Open **Tracing** in the left sidebar — your voice agent shows up alongside your text projects.

<img src="/images/docs/observe/voice-projects-list.png" alt="Tracing projects list including voice projects" style={{ borderRadius: '5px' }} />

Click the project to open its call logs.

---

## Step 3 — The call log table

The voice project's Tracing page shows one row per call, with **voice-specific columns** (different from a text project):

<img src="/images/docs/observe/voice-tracing-overview.png" alt="Voice project Tracing — Call Details, Status, Duration, Avg Latency, Turn Count, Tokens" style={{ borderRadius: '5px' }} />

| Column | What it shows |
|---|---|
| **Call Details** | Date and time of the call. |
| **Status** | `Completed`, in progress, etc. |
| **Duration** | Total call length (e.g. 0:38, 1:55, 3:01). |
| **Avg Latency** | Average response latency for the agent during the call (in ms). |
| **Turn Count** | Number of back-and-forth exchanges in the call. |
| **Tokens** | Total tokens used by the LLM driving the agent. |

A **Primary Graph** at the top plots latency / traffic over time, the same way the text Tracing page does. Date range, Filter, Display, and Add Evals work the same too.

## Step 4 — The call detail drawer

Click any call row to open the detail drawer. This is where Voice Observability really earns its keep.

<img src="/images/docs/observe/voice-call-detail.png" alt="Call detail drawer — transcript, recording, call metadata, Call Analytics with talk ratio, latency pipeline, cost breakdown" style={{ borderRadius: '5px' }} />

The drawer is split into a **transcript view** on the left and a **call metadata + analytics panel** on the right.

### Transcript (left)

Shows the recording at the top (with a waveform you can play) and the full transcript below it — every turn from both speakers, in order. You can search the transcript and filter by speaker (All / Assistant / Customer).

### Call metadata header (right)

Quick facts about the call:
- **Type** — `inbound` or `outbound`.
- **Status** — `completed`, etc.
- **Duration** — total call length.
- **Avg Latency** — average agent response latency.
- **Phone** — the phone number on the call.
- **Provider** — e.g. `vapi`.
- **When** — start timestamp.
- **Ended** — how the call ended (e.g. `customer-ended-call`).

### Tabs

- **Call Analytics** — Duration, Turns, Latency, User/AI talk ratio, Words, Silence, TTFW (time to first word), User INT and AI INT (interruption counts), User WPM and Agent WPM (words per minute). Below those: a **Talk Ratio** bar (% time each side spoke), a **Latency Pipeline** breakdown (Endpointing / Transcribe / Inference / Synthesizer with totals), and **Cost Breakdown** by component.
- **Evals** — eval scores attached to this call.
- **Messages** — raw message log.
- **Logs** — raw call logs.

---

## Updating or disabling observability

Open an agent's edit form to change provider details or toggle observability on/off. If you disable observability, the API key and Assistant ID become optional. If you re-enable it, they're required again.

---

## Supported providers

- [Vapi](https://dashboard.vapi.ai)
- [Retell](https://www.retellai.com/)
- Others (Future AGI accepts call data from additional providers — check with your contact at Future AGI for current support).
