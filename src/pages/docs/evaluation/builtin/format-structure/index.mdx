## Format & structure

Evals in this category check **format and structure**: valid email or JSON, JSON schema validation, code detection, length, valid links, regex patterns, keyword/pattern containment, and style rules (e.g. no apologies, no LLM reference).

| Eval | Description | Inputs | Output | Use case |
| --- | --- | --- | --- | --- |
| Is Email | Checks if the input text is a valid email address. | `text` | Pass/Fail | Validating email format. |
| Is JSON | Determines whether text conforms to valid JSON format. | `text` | Pass/Fail | API or structured output validation. |
| Length Evals | Validates structure and length of text against requirements. | `text` (+ config e.g. min/max) | Pass/Fail | Length or format constraints. |
| Valid Links | Ensures generated content contains valid hyperlinks. | `text` or `output` | Pass/Fail | Link validity in generated text. |
| No Apologies | Checks that the response does not contain unnecessary apologies. | `output` | Pass/Fail | Style or brand guidelines. |
| No LLM Reference | Checks that the response does not reference OpenAI or identify as an OpenAI product. | `output` | Pass/Fail | White-label or unbranded output. |
| Is Code | Checks if the text contains valid programming code. | `text` | Pass/Fail | Code generation or extraction. |
| JSON Schema Validation | Verifies JSON data against specified schema and validation rules. | `text` or JSON field | Pass/Fail | Strict JSON structure checks. |
| Regex | Checks if text matches a specified regular expression pattern. | `text` (+ pattern config) | Pass/Fail | Pattern or format validation. |
| Contain | Validates presence or absence of specific keywords or patterns in text. | `text` (+ keywords/config) | Pass/Fail | Required or forbidden phrases. |
