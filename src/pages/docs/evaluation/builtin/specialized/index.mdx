## Specialized

Evals in this category are **specialized or meta**: eval ranking, input–output or image-instruction scoring, function calling, Text-to-SQL, translation, score/perplexity/deterministic evals, custom code, API call validation, aggregated metrics, and agent-as-judge.

| Eval | Description | Inputs | Output | Use case |
| --- | --- | --- | --- | --- |
| Eval Ranking | Ranks contexts by relevance and suitability for the given input. | `input`, `context` | Score | Retrieval or context ranking. |
| Eval Output | Scores alignment between input and output based on specified criteria. | `input`, `output` (optional criteria) | Score | Generic input–output alignment. |
| LLM Function Calling | Evaluates accuracy of function calls made by the LLM (tool name and parameters). | (tool call + input/output) | Pass/Fail or score | Tool use and agents. |
| Text to SQL | Evaluates accuracy and quality of SQL generated from natural language. | `input`, `output` | Pass/Fail | Text-to-SQL systems. |
| Translation Accuracy | Evaluates translation quality: semantic accuracy, cultural fit, preservation of meaning. | `input`, `output` (source/target) | Score | Machine translation. |
| Score Eval | Scores linkage between instructions, input images, and output images. | Config-driven (rule_prompt, inputs) | Score | Image generation or editing. |
| Prompt Perplexity | Measures how well the model predicts tokens in a given input prompt. | `input` | Score | Prompt analysis or optimisation. |
| Deterministic Eval | Evaluates whether output follows specific rules or patterns (text, images, conversations). | Config-driven (rule_prompt, input keys) | Choices or Pass/Fail | Rule-based or format checks. |
| Custom Code | Runs custom Python code to assess specific evaluation criteria. | User-defined keys | Pass/Fail or score | Fully custom logic. |
| API Call | Assesses validity and correctness of responses from external APIs. | API request/response fields | Pass/Fail or score | API or integration testing. |
| Aggregated Metric | Combines multiple evaluation metrics into a single normalised score. | Outputs of other evals (e.g. BLEU, ROUGE) | Score | Combined quality scores. |
| Agent as a Judge | Uses AI agents with customisable prompts to evaluate content. | Config-driven (prompt, inputs) | Pass/Fail or score | LLM-as-judge evaluations. |
