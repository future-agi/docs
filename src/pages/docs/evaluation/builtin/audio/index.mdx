## Audio

Evals in this category work on **audio** inputs. Use them to assess perceptual quality, transcription accuracy, or whether a description matches the audio content.

| Eval | Description | Inputs | Output | Use case |
| --- | --- | --- | --- | --- |
| Audio Quality | Evaluates the perceptual quality of audio (clarity, noise, listenability) using an LLM. | `input_audio` | Score | Assessing voice or audio output quality; pre/post processing checks. |
| Audio Transcription | Evaluates transcription accuracy or quality of speech-to-text output. | (provider-specific) | Score / Pass-Fail | Validating ASR or captioning pipelines. |
| Eval Audio Description | Evaluates if a given description matches the content of the audio. | `input_audio`, description/criteria | Pass/Fail or score | Checking alignment between audio and its description. |
