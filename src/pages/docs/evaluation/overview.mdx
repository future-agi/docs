## What it is

**Evaluation** is the quality-measurement layer across Future AGI. Without it, you don’t know whether a prompt or agent change actually improved things—you’re left guessing from a few examples or manual review, and results aren’t comparable across runs or experiments. That’s why we built evaluation: so you can define *what* to measure (task completion, tone, safety, factual accuracy, or custom metrics) once, then run the same checks everywhere your outputs live. You use **eval templates** to define the evaluation type and **eval configs** to choose model, mapping, and where they run. Evaluations run on **datasets**, **simulations** (run tests and call executions), **experiments**, **playground** runs, and **replay** sessions. You get scores, pass/fail, and optional explanations per row or per call, plus aggregated summaries and analytics, so you can compare runs, catch regressions, and feed results into optimisation (e.g. Fix My Agent, prompt optimiser, experiment comparison) instead of relying on gut feel.

<Note>
Use the same **eval templates** and **configs** across datasets, simulations, and experiments so results are comparable and you don’t redefine evals for each context.
</Note>

## Purpose

- **Measure quality consistently** — Apply the same eval definitions (templates and configs) across datasets, simulations, and experiments so results are comparable.
- **Compare prompts and agents** — Run evals before and after changes to see whether quality improved or regressed.
- **Support optimization** — Feed eval results into prompt optimisation, Fix My Agent, and experiment analysis so you can iterate on quality.
- **Scale evaluation** — Run evals over many rows (dataset) or many calls (simulation) and view aggregated summaries, KPIs, and per-row or per-call breakdowns.
