## What is this page?

An **eval task** is an automated quality check that runs across your traces. You pick what to score (hallucination, tone, bias, toxicity, custom rubrics), narrow which spans it applies to, and choose whether to run it on past data or on every new trace as it arrives. Scores are attached back to each span — visible in the [Tracing](/docs/observe-two/features/llm-tracing) view and chartable in [Charts](/docs/observe-two/features/charts) and [Dashboards](/docs/observe-two/features/dashboard).

## When would you use this?

- **Score production output quality** — Run evals on real traffic to catch hallucinations, bias, or unsafe content.
- **Catch regressions automatically** — Set a continuous eval task so new traces are scored without you doing anything.
- **Spot-check an incident** — Filter to the time window or session that broke and run evals on just those spans.
- **Budget-aware quality monitoring** — Use sampling so you score a representative subset, not every span.
- **Multiple checks on every trace** — Attach several evals to one task so each span gets scored on safety, accuracy, and tone in a single pass.

## The Evals & Tasks list

The Evals & Tasks page (top-tab next to Tracing/Charts/Alerts inside an Observe project) shows every eval task in this project.

| Column | What it shows |
|---|---|
| **Task Name** | The name you gave the task. |
| **Eval Metrics** | The eval(s) attached to the task. |
| **Filters** | The filters narrowing which spans the task runs on (e.g. date range). |
| **Sampling** | Percentage of matching spans being evaluated. |
| **Status** | Current task status (e.g. Completed, Running). |
| **Created** | When the task was created. |
| **Last Run** | When the task last ran. |

Click **+ Create Task** to make a new one.

## Creating an eval task

The Create Task page is split into a configuration form (left) and a **Live Preview** (right) that shows which rows are currently matching your filters as you build.

<img src="/images/docs/observe-two/evals-create.png" alt="Create Task form with Basic Info, Run evaluations on, Evaluations, Filters, Scheduling sections; Live Preview on right" style={{ borderRadius: '5px' }} />

### Basic Info

- **Task Name** — give the task a memorable name.
- **Project** — defaults to the project you came from.

### Run evaluations on

Pick the level the eval scores at:
- **Spans** (default) — score each individual span.
- **Traces** — score each trace as a whole.
- **Sessions** — score each session as a whole.

### Evaluations

Click **+ Add Evaluation** to attach one or more evals to the task. Each one runs on every matching row. For evals that need a specific input (e.g. Bias Detection), set the input key to the span attribute that holds the field you want evaluated. See [built-in evals](/docs/evaluation/builtin) for the full list.

### Filters

Click **+ Add filter** to narrow which spans the task processes. Common filters: span type, date range, model, session ID, custom span attributes.

### Scheduling

Choose **When to run**:
- **Historical data** — evaluate existing traces from a past time window. The task completes after one pass.
- **New incoming data** — evaluate every new trace as it arrives. The task stays active.

### Live Preview

As you change filters, the right panel updates to show how many rows match and a sample of their column values. Use this to confirm you're scoring the right spans before you save.

### Buttons

- **Cancel** — discard.
- **Test** — dry-run the configuration.
- **Create Task** — save and start the task.

## Where eval results show up

Once a task runs, scores appear in three places:

1. **[Tracing](/docs/observe-two/features/llm-tracing) grid** — the eval name appears as a column, with the score per span.
2. **Trace detail panel → Evals tab** — the full eval result for the selected span (score, output, explanation, errors).
3. **[Charts](/docs/observe-two/features/charts) → Evaluation Metrics** — each eval gets its own time-series chart so you can watch quality trends.

You can also use eval scores as the metric in an [Alert](/docs/observe-two/features/alerts) to get notified when quality drops.
