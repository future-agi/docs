## About

Voice agents are hard to debug. Conversations happen in real time, across multiple turns, and when something goes wrong you usually find out from a user complaint, not a log. **Voice observability** fixes this by pulling call logs from your voice provider into Observe automatically. No SDK or code changes needed. Connect a provider (Vapi or Retell) using its API key and assistant ID, and every call shows up as a trace with its transcript, recording URLs, cost, and duration. From there you can run [evaluations](/docs/observe-two/features/evals), set [alerts](/docs/observe-two/features/alerts), search, filter, and export — the same way you would with any other trace.

## When to use

- **Visibility into voice agent calls**: See all conversations for a voice agent in one project without adding SDK instrumentation.
- **Evaluate voice conversations**: Run evals (quality, bias, adherence) on conversation spans from voice calls.
- **Alerts on voice metrics**: Set monitors on voice project metrics and get notified when something degrades.
- **Transcripts and recordings for debugging**: Access transcript and recording URLs from the trace view.
- **Multiple voice providers**: Support for Vapi and Retell so you can monitor agents regardless of provider.

## How to

<Steps>
  <Step title="Get provider credentials">
    From your voice provider's dashboard, obtain:
    - **API key**
    - **Assistant ID** (or agent ID)

    These are required when observability is enabled. Supported providers: [Vapi](https://dashboard.vapi.ai), [Retell](https://www.retellai.com/).
  </Step>

  <Step title="Create an agent definition with observability">

    {/* SCREENSHOT NEEDED — [Click Create agent definition → fill in the form →](https://app.futureagi.com/dashboard/observe/f6ecb8b6-2566-4886-918d-662144d26931) — capture: the filled-in create agent definition form */}
    <img src="/screenshot/product/observe/voice/agent_definition_filled.png" alt="Create agent definition form" style={{ borderRadius: '5px' }} />

    Check **Enable Observability**. The API key and Assistant ID are required only if observability is enabled.

    {/* SCREENSHOT NEEDED — [Create agent definition form → Enable Observability checkbox →] — capture: the form with Enable Observability checked */}
    <img src="/screenshot/product/observe/voice/agent_definition_details.jpeg" alt="Enable observability option" style={{ borderRadius: '5px' }} />

  <Step title="View call logs in Observe">
    Open the **Projects** tab. A project with the same name as your agent lists all call logs.
