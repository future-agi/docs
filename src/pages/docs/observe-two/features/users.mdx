## What is this page?

The **Users** view in Observe groups your AI activity by individual end users. Each row is one user (identified by a `user.id` you set on your spans), with quick stats: when they were first active, when they were last active, how many traces and sessions they have.

Click any user to open their detail page — a focused view of just their traces and sessions.

<img src="/images/docs/observe-two/users-overview.png" alt="Users tab overview" style={{ borderRadius: '5px' }} />

---

## When would you use this?

- **A user reports a bug** — Find them by ID, open their detail page, walk through every request they made.
- **You want to see who's most active** — Sort by trace count or session count to find power users.
- **A user complains about cost** — Open their detail page and see the traces driving usage.
- **Engagement check** — Compare First Active vs. Last Active dates to see who stuck around vs. dropped off.

---

## The users list

The **Users** tab shows one row per end user. Columns confirmed from the live platform:

| Column | What it tells you |
|---|---|
| **User ID** | The identifier set via `user.id` on your spans. |
| **First Active** | When this user's first request arrived. |
| **Last Active** | When their most recent request arrived. |
| **No. of Traces** | Total requests from this user. |
| **No. of Sessions** | How many distinct conversations this user has had. |
| **Actions** | A per-row dropdown for managing the user. |

A **User Metrics** chart at the top plots latency and traffic over the selected time window.

<img src="/images/docs/observe-two/users-filter.png" alt="Filter panel with AI search bar, Basic and Query tabs, and property list" style={{ borderRadius: '5px' }} />

- **AI search** — Describe what you want in plain English.
- **Basic mode** — Pick a property, condition, value.
- **Query mode** — Write a filter expression directly.
- **Property categories**: All · System · Evals · Annotations · Attributes.

## Display settings

Click **Display** to control the layout.

<img src="/images/docs/observe-two/users-display.png" alt="Display panel with sections for Rows, Columns, Group, Graph, Settings" style={{ borderRadius: '5px' }} />

## User detail page

Click any user row to open their detail page — focused on just that user's activity.

<img src="/images/docs/observe-two/users-detail.png" alt="User detail page showing Trace tab with this user's requests" style={{ borderRadius: '5px' }} />

The page shows:

- **Header** — the User ID and "Activity across all projects" subtitle. Use the `← Back` button to return to the user list.
- **Two tabs**: **Sessions** and **Trace**.
- **Date range picker** — narrow the user's activity to a time window (independent of the main Users page filter).
- **Filter, Display, Add Evals** — the same filter and display controls as on the main pages.

### Trace tab

Lists every individual request from this user. Same columns as the main Tracing page (Trace Name, Input, Output, Timestamp, Status, Latency, Tokens). Click any row to open the [Trace detail panel](/docs/observe-two/features/llm-tracing#opening-a-trace-the-detail-panel).

### Sessions tab

Lists every conversation this user had — same columns as the main [Sessions](/docs/observe-two/features/session) page.

## How to associate spans with an end user

For a span to count under a user in the dashboard, set a `user.id` attribute on it.

<Steps>
  <Step title="Set user.id on a span directly">
    Set **`user.id`** (required). Optionally set **`user.id.type`** (email, phone, uuid, custom), **`user.id.hash`**, and **`user.metadata`** (JSON):

    <CodeGroup>
    ```python Python
    from fi_instrumentation import register, FITracer
    from fi_instrumentation.fi_types import ProjectType
    from opentelemetry.trace import Status, StatusCode

    trace_provider = register(
        project_type=ProjectType.OBSERVE,
        project_name="PROJECT_NAME",
    )
    tracer = FITracer(trace_provider.get_tracer(__name__))

    with tracer.start_as_current_span("SPAN_NAME") as span:
        span.set_status(Status(StatusCode.OK))
        span.set_attribute("user.id", "vivek.gupta@example.com")
        span.set_attribute("user.id.type", "email")  # email | phone | uuid | custom
        span.set_attribute("user.id.hash", "<hash_for_the_user.id>")  # optional
        span.set_attribute("user.metadata", {})  # optional
        span.set_attribute("input.value", "input")
        span.set_attribute("output.value", "output")
    ```
    ```typescript JS/TS
    const { register, ProjectType } = require("@traceai/fi-core");

    const traceProvider = register({
        projectType: ProjectType.OBSERVE,
        projectName: "FUTURE_AGI"
    });
    const tracer = traceProvider.getTracer("manual-instrumentation-example");

    tracer.startActiveSpan("SPAN_NAME", {}, (span) => {
        span.setAttribute("user.id", "vivek.gupta@example.com");
        span.setAttribute("user.id.type", "email");
        span.end();
    });
    ```
    </CodeGroup>
  </Step>

  <Step title="Set user for many spans using context">
    To tag every span in a block with the same user, use `using_attributes` and pass `user_id` (and optionally `session_id`):

    <CodeGroup>
    ```python Python
    from fi_instrumentation import using_attributes

    with using_attributes(user_id="newuser@example.com", session_id="new-session"):
        response = client.chat.completions.create(
            model="gpt-3.5-turbo",
            messages=[{"role": "user", "content": "Write a haiku."}],
            max_tokens=20,
        )
    ```
    </CodeGroup>
  </Step>
</Steps>

<Note>
  End users are unique per project and organization by `(user_id, user_id_type)`. Sending the same `user.id` (and type) on spans in the same Observe project ties those spans to one end user.
</Note>
