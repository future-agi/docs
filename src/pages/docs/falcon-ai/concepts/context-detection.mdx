## About

Asking "why is this score so low?" only makes sense if Falcon AI knows which evaluation is on screen. Context detection makes this work. Every time a page changes, the frontend extracts the page type, entity type, and entity ID from the URL and sends it to Falcon AI as part of each message. This means Falcon AI always knows what is being viewed without the user having to specify it.

---

## How it works

The frontend maintains a context object that updates on every route change:

- **page**: The domain of the current page (e.g., `datasets`, `evaluations`, `tracing`, `agents`, `experiments`, `prompts`, `admin`).
- **entity_type**: The type of entity being viewed, if any (e.g., `dataset`, `evaluation`, `trace`, `experiment`, `agent`, `scenario`, `test_run`).
- **entity_id**: The specific entity ID extracted from the URL.
- **path**: The full URL path.

If no matching page is found, context defaults to `general`.

## Manual context

Beyond automatic detection, entities can be attached manually:

- Click **+ Add context** in the input area to search and attach specific entities.
- Up to 5 entities can be attached at a time.
- Context chips appear above the input with an X to remove.
- Manual context is sent alongside automatic page context in each message.
