## About

Loading every tool into the LLM context at once wastes context window space and slows down responses. The mode system solves this by detecting which domain a request belongs to and loading only the relevant tools. Detection is deterministic (no LLM call) and happens in milliseconds. If a request spans multiple domains, all tools are loaded automatically.

---

## How it works

Every request goes through three detection steps in order:

1. **Keyword matching** (priority 1): The message is scanned for domain-specific keywords. If keywords from multiple domains are found, general mode (all tools) is used.
2. **Page context** (priority 2): If no keywords match, the current dashboard page determines the mode.
3. **Default**: Falls back to general mode.

Once a mode is detected, three layers of tools are loaded:

- **Core tools** (always loaded): `whoami`, `list_workspaces`, `search`, `read_schema`, `read_taxonomy`, `search_docs`
- **Common tools** (always loaded): Cross-cutting tools like `list_datasets`, `search_traces`, `list_evaluations`, `save_memory`, etc.
- **Mode-specific tools**: Tools from the categories defined for that mode.

---

## Modes

| Mode | Tool categories | Activates when |
|------|----------------|----------------|
| `general` | All categories | Multiple domains mentioned, or no match |
| `datasets` | context, datasets, annotations | Message mentions "dataset", "rows", "columns", "synthetic" |
| `evaluations` | context, evaluations | Message mentions "eval", "score", "faithfulness", "hallucination" |
| `tracing` | context, tracing | Message mentions "trace", "span", "latency", "error rate" |
| `experiments` | context, experiments | Message mentions "experiment", "a/b test", "variant" |
| `agents` | context, agents, simulation | Message mentions "agent", "simulation", "scenario" |
| `prompts` | context, prompts, optimization | Message mentions "prompt engineering", "prompt version" |
| `admin` | context, users, usage | Message mentions "user", "api key", "cost", "billing" |

---

## Page-to-mode mapping

When keyword detection does not match, the current dashboard page determines the mode:

| Dashboard page | Mode |
|---------------|------|
| `/dashboard/data`, `/dashboard/develop` | datasets |
| `/dashboard/evaluations`, `/dashboard/evals` | evaluations |
| `/dashboard/observe` | tracing |
| `/dashboard/experiments` | experiments |
| `/dashboard/simulate` | agents |
| `/dashboard/workbench` | prompts |
| `/dashboard/settings`, `/dashboard/users` | admin |
