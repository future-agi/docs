## About

Future AGI is fully open-source. Self-hosting runs the entire stack on your machines вЂ” all traces, datasets, evaluations, and model calls stay within your network. Backend is Django, frontend is React + Vite, LLM gateway is Go.

Not sure if you need this? The hosted version at [app.futureagi.com](https://app.futureagi.com) is easier to operate. Self-host when you need **data residency**, **air-gapped environments**, **cost control at scale**, or **deep customization**.

## Quick start

```bash
git clone https://github.com/future-agi/future-agi.git
cd future-agi
cp .env.example .env
docker compose up
```

First boot builds from source (~10вЂ“15 min). After `Application startup complete`:

| Service | URL |
|---|---|
| Frontend | http://localhost:3031 |
| Backend API | http://localhost:8000 |
| PeerDB UI | http://localhost:3001 вЂ” `peerdb` / `peerdb` |

| Layer | Service | Purpose |
|---|---|---|
| App | `frontend` | React SPA served by nginx |
| App | `backend` | Django REST + gRPC + WebSocket API |
| App | `worker` | Temporal worker вЂ” evals, agent loops, data jobs |
| App | `gateway` | Go LLM proxy вЂ” routing, retries, rate limits, logging |
| App | `serving` | Embeddings and small model inference |
| App | `code-executor` | nsjail-sandboxed eval code runner (`privileged: true` required) |
| Data | `postgres` | Primary DB вЂ” users, traces, datasets, evals, prompts |
| Data | `clickhouse` | Analytics DB вЂ” replicated from Postgres via PeerDB |
| Data | `redis` | Cache, rate limits, WebSocket pub/sub |
| Data | `minio` | S3-compatible object storage (swap for S3 in prod) |
