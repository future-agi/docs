## About

Every time your application calls a language model, Future AGI records the request and response as a trace. A trace captures what was sent, what came back, how long it took, how many tokens were used, and what it cost. On its own, a trace is a snapshot of a single generation.

Linked traces connect each recorded generation back to the specific prompt template and version that produced it. Once that association exists, the Prompt Workbench aggregates trace data per prompt version and surfaces it alongside the prompt itself. Instead of searching through individual traces, you see a consolidated view: how many times a prompt was called, its typical latency and cost, and how those metrics shift as you iterate.

The connection is established at logging time. When your application sends a generation request, it includes the prompt template or template ID in the trace payload. Future AGI uses that to attribute the trace to the correct prompt and version in the Workbench.

---

## How linking works

The connection is established when your application includes the prompt template or template ID in the generation request it sends to Future AGI. The platform matches the template reference in the trace to the corresponding prompt in the Workbench and groups all traces for that prompt together.

Once linked, the Workbench aggregates metrics per prompt version: median latency, input and output token counts, cost per generation, and total call count. You can view these on the **Metrics** tab of any prompt.

---

## Why it matters

Without linking, you can see that your application is slow or expensive, but you cannot tell which prompt is responsible. With linking, every metric is tied to a specific prompt and version. This makes it possible to:

- Attribute a latency increase to a specific prompt change
- Compare cost across prompt versions without building separate dashboards
- Know which prompts are actively being used and which are stale
- Validate that a prompt change improved real-world performance, not just test results

---

## Linked traces vs raw traces

| | Raw traces | Linked traces |
|---|---|---|
| **What you see** | Application-level metrics | Metrics per prompt and version |
| **Attribution** | Anonymous API calls | Tied to a specific template and version |
| **Where to view** | Observe / tracing dashboard | Prompt Workbench Metrics tab |
| **Setup required** | SDK instrumentation | SDK instrumentation + template reference in request |
