# Delete a scenario

Soft-deletes a scenario by marking it as deleted. The scenario data is retained but excluded from future listings.

<ApiPlayground
  method="DELETE"
  endpoint="/simulate/scenarios/{scenario_id}/delete/"
  baseUrl="https://api.futureagi.com"
  parameters={[{"name": "scenario_id", "in": "path", "required": true, "description": "The UUID of the scenario to delete.", "type": "string"}]}
/>

## Authentication

All requests require authentication via API keys:

| Header | Description |
|--------|-------------|
| `X-Api-Key` | Your API key |
| `X-Secret-Key` | Your secret key |

## Parameters

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `scenario_id` | string (UUID) | Yes | The UUID of the scenario to delete. |

## Responses

### 200

Scenario deleted successfully.

- **message**: string
  Example: `Scenario deleted successfully`

### 401

Unauthorized. Authentication credentials were not provided or are invalid.

### 404

Not Found. The scenario with the specified ID could not be found.

### 500

Internal Server Error. An unexpected error occurred.

## Code Examples

### cURL

```bash
curl -X DELETE "https://api.futureagi.com/simulate/scenarios/{scenario_id}/delete/" \
  -H "X-Api-Key: YOUR_API_KEY" \
  -H "X-Secret-Key: YOUR_SECRET_KEY"
```

### Python

```python
import requests

url = "https://api.futureagi.com/simulate/scenarios/{scenario_id}/delete/"
headers = {
    "X-Api-Key": "YOUR_API_KEY",
    "X-Secret-Key": "YOUR_SECRET_KEY"
}

response = requests.delete(url, headers=headers)
print(response.json())
```

### JavaScript

```javascript
const response = await fetch("https://api.futureagi.com/simulate/scenarios/{scenario_id}/delete/", {
  method: "DELETE",
  headers: {
    "X-Api-Key": "YOUR_API_KEY",
    "X-Secret-Key": "YOUR_SECRET_KEY"
```
