---
title: "Create Scenario"
description: "Creates a new scenario from a dataset, a script, or a conversation graph. Scenario creation is processed asynchronously in the background, and the response returns immediately with a processing status."
---

<ApiSection title="Authentication">
  <ParamField name="X-Api-Key" type="string" required>
    Your Future AGI API key used to authenticate requests. You can find and manage your API keys in the [Dashboard](https://app.futureagi.com) under Settings.
  </ParamField>
  <ParamField name="X-Secret-Key" type="string" required>
    Your Future AGI secret key, used alongside the API key for request authentication. This is generated when you create an API key in the [Dashboard](https://app.futureagi.com).
  </ParamField>
</ApiSection>

<ApiSection title="Request body">
  <ParamField body="name" type="string" required>
    A human-readable name for the scenario. Maximum 255 characters. Cannot be blank. Choose a descriptive name that reflects the test scenario, such as `"billing-inquiry-refund"` or `"onboarding-flow-happy-path"`.
  </ParamField>

  <ParamField body="description" type="string" optional>
    An optional description providing context about the scenario's purpose, the test cases it covers, or any special conditions.
  </ParamField>

  <ParamField body="kind" type="string" optional>
    The type of scenario to create, which determines how test data is sourced:
    - `"dataset"` — Creates the scenario from an existing dataset. Requires `datasetId`. This is the default.
    - `"script"` — Creates the scenario from a script file. Requires `scriptUrl`.
    - `"graph"` — Creates the scenario from a conversation graph structure. Requires `graph` (or `generateGraph`).
  </ParamField>

  <ParamField body="agentDefinitionId" type="string" optional>
    The UUID of the agent definition this scenario is designed to test. Required when `generateGraph` is `true` or when `sourceType` is `"agent_definition"`. The agent's configuration is used to generate appropriate test scenarios.
  </ParamField>

  <ParamField body="agentDefinitionVersionId" type="string" optional>
    The UUID of a specific agent version to use for scenario generation. When omitted, the latest or active version is used.
  </ParamField>

  <ParamField body="datasetId" type="string" optional>
    The UUID of the source dataset to build the scenario from. Required when `kind` is `"dataset"`. The dataset's rows become the test cases for the scenario.
  </ParamField>

  <ParamField body="scriptUrl" type="string" optional>
    A URL pointing to the script file for script-based scenarios. Required when `kind` is `"script"`. The script defines the conversation flow and expected behaviors.
  </ParamField>

  <ParamField body="noOfRows" type="integer" optional>
    The number of test case rows to generate for the scenario. Accepts values between `10` and `100`. Defaults to `20`. Higher values provide more comprehensive testing coverage but take longer to generate.
  </ParamField>

  <ParamField body="addPersonaAutomatically" type="boolean" optional>
    When set to `true`, the system automatically assigns diverse personas to the generated test cases. This helps test the agent's behavior across different customer profiles and communication styles. Defaults to `false`.
  </ParamField>

  <ParamField body="graph" type="object" optional>
    A conversation graph structure that defines the flow of the simulated conversation. Required when `kind` is `"graph"` and `generateGraph` is `false`. The graph specifies decision nodes, branches, and expected outcomes.
  </ParamField>

  <ParamField body="generateGraph" type="boolean" optional>
    When set to `true`, the system automatically generates a conversation graph based on the agent definition's configuration and prompt. Requires `agentDefinitionId` to be provided. Defaults to `false`.
  </ParamField>

  <ParamField body="personas" type="array of string" optional>
    A list of persona UUIDs to include in the scenario. Personas define simulated customer profiles with specific traits, communication styles, and behaviors. You can create personas via the Personas API.
  </ParamField>

  <ParamField body="sourceType" type="string" optional>
    Determines the source used for AI-powered scenario generation:
    - `"agent_definition"` — Uses the agent definition's configuration and prompt as context. This is the default.
    - `"prompt"` — Uses a specific prompt template and version as context. Requires `promptTemplateId` and `promptVersionId`.
  </ParamField>

  <ParamField body="promptTemplateId" type="string" optional>
    The UUID of the prompt template to use as the generation source. Required when `sourceType` is `"prompt"`.
  </ParamField>

  <ParamField body="promptVersionId" type="string" optional>
    The UUID of the specific prompt version to use. Required when `sourceType` is `"prompt"`. Must belong to the specified `promptTemplateId`.
  </ParamField>

  <ParamField body="customColumns" type="array of object" optional>
    An array of custom column definitions to add to the scenario dataset (maximum 10). Each column object must include:
    - `name` (string) — The column name. Maximum 50 characters. Must be unique within the dataset.
    - `dataType` (string) — The data type: `text`, `boolean`, `integer`, `float`, `json`, `array`, `image`, `images`, `datetime`, `audio`, `document`, `others`, or `persona`.
    - `description` (string) — A description that guides AI data generation. Maximum 200 characters.
  </ParamField>

  <ParamField body="customInstruction" type="string" optional>
    A custom instruction that provides additional context or constraints for the AI-powered scenario generation. Use this to steer the generated test cases toward specific behaviors or edge cases.
  </ParamField>
</ApiSection>

<ApiSection title="Response" status={202} statusText="Accepted">
  <ResponseField name="message" type="string">
    A confirmation message indicating the scenario creation has been queued for background processing.
  </ResponseField>
  <ResponseField name="scenario" type="object">
    The newly created scenario object with its generated `id` and `name`. Additional fields will be populated once background processing completes.
  </ResponseField>
  <ResponseField name="status" type="string">
    The current processing status. Returns `"processing"` immediately after creation. The scenario transitions to `"completed"` once background generation finishes.
  </ResponseField>
</ApiSection>

<ApiSection title="Errors">
  <ParamField name="400" type="Bad Request">
    The request body failed validation. Common causes include a missing or blank `name`, a missing `datasetId` when `kind` is `"dataset"`, a missing `scriptUrl` when `kind` is `"script"`, or invalid custom column definitions.
  </ParamField>
  <ParamField name="401" type="Unauthorized">
    The request could not be authenticated. Verify that both `X-Api-Key` and `X-Secret-Key` headers are present and contain valid credentials.
  </ParamField>
  <ParamField name="500" type="Internal Server Error">
    An unexpected server-side error occurred during scenario creation. If this persists, contact support with the request details.
  </ParamField>
</ApiSection>
