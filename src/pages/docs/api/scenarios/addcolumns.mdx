# Add columns to a scenario

Adds new columns to a scenario's dataset. Column data is generated asynchronously using AI based on the existing rows and column descriptions.

<ApiPlayground

  endpoint="/simulate/scenarios/{scenario_id}/add-columns/"

  parameters={[{"name": "scenario_id", "in": "path", "required": true, "description": "The UUID of the scenario to add columns to.", "type": "string"}]}
  requestBody={{"columns": [{"name": "expected_outcome", "dataType": "text", "description": "The expected outcome of the conversation"}]}}
/>

## Authentication

All requests require authentication via API keys:

| Header | Description |
|--------|-------------|
| `X-Api-Key` | Your API key |
| `X-Secret-Key` | Your secret key |

## Parameters

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `scenario_id` | string (UUID) | Yes | The UUID of the scenario to add columns to. |

## Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `columns` | array of objects | Yes | Column definitions to add. Max 10 per request. |

Each column object:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Column name. Max 50 chars. Must be unique in the dataset. |
| `dataType` | string | Yes | Data type for the column. Must be one of: `text`, `boolean`, `integer`, `float`, `json`, `array`, `image`, `images`, `datetime`, `audio`, `document`, `others`, `persona`. |
| `description` | string | Yes | Guides AI generation. Max 200 chars. |

**Validations:** Min 1 column required. No duplicate names in request or dataset. Dataset must have at least one row.

### Example

```json
{
  "columns": [
    {
      "name": "expected_outcome",
      "dataType": "text",
      "description": "The expected outcome of the conversation"
    },
    {
      "name": "difficulty_level",
      "dataType": "text",
      "description": "How difficult the scenario is (easy, medium, hard)"
    }
  ]
}
```

## Responses

### 202

Accepted. Column generation has started in the background.

- **message**: string
  Example: `Started generating 2 new column(s) for scenario`
- **scenario_id**: string (UUID)
- **dataset_id**: string (UUID)
- **columns**: array of string
  List of column names being generated.

### 400

Bad Request. Possible reasons:
- No columns provided.
- More than 10 columns in a single request.
- Duplicate column names in the request.
- Column name already exists in the dataset.
- Dataset has no rows.
- Missing required fields in column definition.

### 401

Unauthorized. Authentication credentials were not provided or are invalid.

### 404

Not Found. The scenario with the specified ID could not be found.

### 500

Internal Server Error. An unexpected error occurred.

## Code Examples

### cURL

```bash
curl -X POST "https://api.futureagi.com/simulate/scenarios/{scenario_id}/add-columns/" \
  -H "X-Api-Key: YOUR_API_KEY" \
  -H "X-Secret-Key: YOUR_SECRET_KEY" \
  -H "Content-Type: application/json" \
  -d '{"columns": [{"name": "expected_outcome", "dataType": "text", "description": "The expected outcome of the conversation"}]}'
```

### Python

```python
import requests

url = "https://api.futureagi.com/simulate/scenarios/{scenario_id}/add-columns/"
headers = {
    "X-Api-Key": "YOUR_API_KEY",
    "X-Secret-Key": "YOUR_SECRET_KEY",
    "Content-Type": "application/json"
}
data = {
    "columns": [
        {
            "name": "expected_outcome",
            "dataType": "text",
            "description": "The expected outcome of the conversation"
        }
    ]
}

response = requests.post(url, headers=headers, json=data)
print(response.json())
```

### JavaScript

```javascript
const response = await fetch("https://api.futureagi.com/simulate/scenarios/{scenario_id}/add-columns/", {

        "dataType": "text",

const data = await response.json();

```
