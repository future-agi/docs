## Authentication

All requests require authentication via API keys:

| Header | Description |
|--------|-------------|
| `X-Api-Key` | Your API key |
| `X-Secret-Key` | Your secret key |

## Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `agentType` | string | Yes | `voice` or `text`. |
| `agentName` | string | Yes | Display name for the agent. |
| `provider` | string | Yes | Voice provider: `vapi`, `retell`, `eleven_labs`, or `others`. Required for voice agents. |
| `apiKey` | string | Conditional | Provider API key. Required for outbound agents or when `observabilityEnabled` is `true`. |
| `assistantId` | string | Conditional | Provider assistant ID. Required for outbound agents or when `observabilityEnabled` is `true`. |
| `description` | string | No | Description for the initial agent version. |
| `language` | string | Yes | [ISO 639-1](https://en.wikipedia.org/wiki/List_of_ISO_639_language_codes) language code, e.g. `en`. |
| `knowledgeBase` | string (UUID) | No | Knowledge base ID to link to the agent. |
| `countryCode` | string | No | Phone country code, e.g. `1` (USA), `91` (India). |
| `contactNumber` | string | Yes | Full phone number with country code, e.g. `+1xxxxxxxxxx`. Must be 10-12 digits. |
| `inbound` | boolean | No | `true` if the agent handles inbound calls. |
| `commitMessage` | string | No | Commit message for the initial version. |
| `observabilityEnabled` | boolean | No | Enable provider observability integration. |

## Responses

### 201

Agent definition created successfully.

- **message**: string
  Example: `Agent definition created successfully`
- **agent**: any

### 400

Invalid data provided.

- **error**: string
  Example: `Invalid data`
- **details**: object

### 401

Authentication credentials were not provided.

### 500

Internal server error.

- **error**: string
  Example: `Failed to create agent definition: [error details]`
