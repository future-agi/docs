title: "Create Agent Definition"
description: "Creates a new agent definition along with its first version. An agent definition represents a voice or text AI agent configured with a specific provider, language, and communication settings."
---

<ApiSection title="Authentication">
  <ParamField name="X-Api-Key" type="string" required>
    Your Future AGI API key used to authenticate requests. You can find and manage your API keys in the [Dashboard](https://app.futureagi.com) under Settings.
  </ParamField>
  <ParamField name="X-Secret-Key" type="string" required>
    Your Future AGI secret key, used alongside the API key for request authentication. This is generated when you create an API key in the [Dashboard](https://app.futureagi.com).
  </ParamField>
</ApiSection>

<ApiSection title="Request body">
  <ParamField body="agentType" type="string" required>
    The type of agent to create. Determines the communication channel and available features:
    - `"voice"` — A voice-based agent that communicates via phone calls. Requires a `provider`, `contactNumber`, and typically an `assistantId`.
    - `"text"` — A text-based agent that communicates via chat or messaging interfaces.
  </ParamField>

  <ParamField body="agentName" type="string" required>
    A human-readable display name for the agent definition. This name appears in the Dashboard and is used to identify the agent across the platform. Choose a descriptive name like `"customer-support-bot"` or `"sales-qualifier-agent"`.
  </ParamField>

  <ParamField body="provider" type="string" required>
    The external voice provider that powers the agent. This determines which service handles the actual voice calls:
    - `"vapi"` — VAPI voice AI platform
    - `"retell"` — Retell AI voice platform
    - `"eleven_labs"` — Eleven Labs voice synthesis
    - `"others"` — Custom or unsupported provider

    Required for voice agents. The provider selection affects which API key format and assistant ID format are expected.
  </ParamField>

  <ParamField body="apiKey" type="string" optional>
    The API key for authenticating with the external voice provider (e.g., your VAPI or Retell API key). Required when the agent handles outbound calls or when `observabilityEnabled` is set to `true`. This key is stored securely and used to make calls on your behalf.
  </ParamField>

  <ParamField body="assistantId" type="string" optional>
    The identifier of the assistant configured in your external provider's platform (e.g., your VAPI assistant ID). Required for outbound agents or when `observabilityEnabled` is `true`. This links the Future AGI agent definition to the specific assistant configuration in the provider's system.
  </ParamField>

  <ParamField body="description" type="string" optional>
    A description for the initial agent version. Use this to document the agent's purpose, behavior, or any initial configuration notes. This description is attached to the first version, not the agent definition itself.
  </ParamField>

  <ParamField body="language" type="string" required>
    The primary language of the agent, specified as an [ISO 639-1](https://en.wikipedia.org/wiki/List_of_ISO_639_language_codes) two-letter code. For example, `"en"` for English, `"es"` for Spanish, `"fr"` for French. This determines the language used for scenario generation and evaluation.
  </ParamField>

  <ParamField body="knowledgeBase" type="string" optional>
    The UUID of a knowledge base to link to the agent. When provided, the agent can reference this knowledge base during conversations, enabling it to answer questions using your organization's documents and data.
  </ParamField>

  <ParamField body="countryCode" type="string" optional>
    The phone country code without the `+` prefix. For example, `"1"` for the United States, `"91"` for India, `"44"` for the United Kingdom. Used in combination with `contactNumber` for phone-based voice agents.
  </ParamField>

  <ParamField body="contactNumber" type="string" required>
    The full phone number including the country code prefix, formatted as `+<country_code><number>`. For example, `"+14155551234"`. The number portion must be between 10 and 12 digits. This is the phone number the agent uses to make or receive calls.
  </ParamField>

  <ParamField body="inbound" type="boolean" optional>
    Set to `true` if the agent is configured to handle inbound calls (i.e., customers call the agent). When `false` or omitted, the agent is treated as outbound-only (the agent initiates calls to customers).
  </ParamField>

  <ParamField body="commitMessage" type="string" optional>
    A commit message describing the initial version of the agent. Works like a git commit message — use it to document what this version represents. For example, `"Initial setup with VAPI integration"`. If omitted, a default message is generated.
  </ParamField>

  <ParamField body="observabilityEnabled" type="boolean" optional>
    When set to `true`, enables observability integration with the external provider. This allows Future AGI to ingest call logs, transcripts, and metrics from the provider for monitoring and evaluation. Requires both `apiKey` and `assistantId` to be provided.
  </ParamField>
</ApiSection>

<ApiSection title="Response" status={201} statusText="Created">
  <ResponseField name="message" type="string">
    A confirmation message indicating the agent definition was created successfully.
  </ResponseField>
  <ResponseField name="agent" type="object">
    The newly created agent definition object, including its generated `id`, `agentName`, `agentType`, `provider`, `language`, and `created_at` timestamp.
  </ResponseField>
</ApiSection>
