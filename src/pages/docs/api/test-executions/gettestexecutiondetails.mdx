<ApiSection title="Path parameters">
  <ParamField path="test_execution_id" type="string" required>
    The unique identifier (UUID) of the test execution to retrieve. You can obtain this ID from the test executions list endpoint or from the response when creating a new test execution.
  </ParamField>

<ApiSection title="Query parameters">
  <ParamField query="search" type="string" optional>
    A search string to filter the call executions returned in the `calls` array. The search is applied against call execution fields such as scenario name, transcript content, and status. Only matching call executions are included in the paginated results.
  </ParamField>
  <ParamField query="page" type="integer" optional>
    The page number for paginating through the call executions list. Defaults to `1` if not specified. Each page returns a fixed number of call execution records. Use in combination with the response metadata to navigate through large result sets.
  </ParamField>
  <ParamField query="filters" type="string" optional>
    A JSON-encoded array of filter objects to narrow down the call executions. Each filter object defines a column, operator, and value to match against. For example: `[{"colId":"status","filterType":"text","type":"equals","filter":"completed"}]`. URL-encode the JSON string when passing it as a query parameter.
  </ParamField>
  <ParamField query="row_groups" type="string" optional>
    A JSON-encoded array of column IDs to group the call execution results by. When specified, the response organizes call executions into groups based on the specified column values. For example: `["scenario"]` groups results by scenario name. URL-encode the JSON string when passing it as a query parameter.
  </ParamField>
  <ParamField query="group_keys" type="string" optional>
    A JSON-encoded array of group key values used in conjunction with `row_groups` to drill into a specific group. When `row_groups` is set, provide the corresponding group key values to retrieve call executions within that group. URL-encode the JSON string when passing it as a query parameter.
  </ParamField>
</ApiSection>
