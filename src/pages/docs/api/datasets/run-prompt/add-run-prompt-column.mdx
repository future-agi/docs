# Add Run Prompt Column

Add a new run prompt column to a dataset. This creates a column that executes an LLM prompt for each row, populating cells with the model's responses. The prompt can reference other column values using template variables.

<ApiPlayground
  method="POST"
  endpoint="/model-hub/develops/add_run_prompt_column/"
  baseUrl="https://api.futureagi.com"
  parameters={[]}
  requestBody={{
    dataset_id: "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    name: "gpt_response",
    config: {
      model: "gpt-4o",
      messages: [
        { role: "system", content: "You are a helpful assistant." },
        { role: "user", content: "Summarize: {{input}}" }
      ],
      output_format: "string",
      temperature: 0.7,
      max_tokens: 1024
    }
  }}
/>

## Authentication

All requests require authentication via API keys:

| Header | Description |
|--------|-------------|
| `X-Api-Key` | Your API key |
| `X-Secret-Key` | Your secret key |

## Request Body

<ParamField body="dataset_id" type="string (UUID)" required>
  The UUID of the dataset to add the column to.
</ParamField>

<ParamField body="name" type="string" required>
  Name for the new column. Must be unique within the dataset. Minimum 1 character.
</ParamField>

<ParamField body="config" type="object" required>
  Prompt configuration object.
</ParamField>

### Config Object

<ParamField body="config.model" type="string" required>
  The model to use (e.g., `gpt-4o`, `gpt-4o-mini`, `claude-3-5-sonnet`).
</ParamField>

<ParamField body="config.messages" type="array" required>
  Array of message objects defining the prompt. Use `{{column_name}}` syntax to reference column values.
</ParamField>

<ParamField body="config.output_format" type="string">
  Output format. Values: `string` (default), `audio`, `json`.
</ParamField>

<ParamField body="config.temperature" type="number">
  Sampling temperature (0-2). Default: `1`.
</ParamField>

<ParamField body="config.max_tokens" type="integer">
  Maximum tokens in the response.
</ParamField>

<ParamField body="config.top_p" type="number">
  Nucleus sampling parameter (0-1). Default: `1`.
</ParamField>

<ParamField body="config.frequency_penalty" type="number">
  Frequency penalty (-2 to 2). Default: `0`.
</ParamField>

<ParamField body="config.presence_penalty" type="number">
  Presence penalty (-2 to 2). Default: `0`.
</ParamField>

<ParamField body="config.response_format" type="string">
  Response format constraint (e.g., `json_object`).
</ParamField>

<ParamField body="config.tool_choice" type="string">
  Tool selection strategy (e.g., `auto`, `none`, `required`).
</ParamField>

<ParamField body="config.tools" type="array">
  Array of tool definitions. Each tool has `id`, `name`, and `config`.
</ParamField>

<ParamField body="config.concurrency" type="integer">
  Number of concurrent requests. Controls how many rows are processed simultaneously.
</ParamField>

## Response

<ResponseField name="message" type="string">Confirmation message. Example: `Run prompt column added successfully`.</ResponseField>

### Example Response

```json
{
  "message": "Run prompt column added successfully"
}
```

## Responses

### 200

Column created and prompt execution started. Rows are processed asynchronously in the background.

### 400

Bad request. Possible reasons:

- **Missing dataset_id** — `dataset_id` is required.
- **Missing name** — Column `name` is required and must be at least 1 character.
- **Duplicate name** — A column with this name already exists in the dataset.
- **Invalid config** — The prompt configuration is invalid.

### 401

Authentication credentials were not provided or are invalid.

### 404

Dataset not found or does not belong to your organization.

### 500

Internal server error.
