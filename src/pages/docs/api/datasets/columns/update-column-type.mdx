<ApiSection title="Path parameters">
  <ParamField path="dataset_id" type="string" required>
    The universally unique identifier (UUID) of the dataset that contains the column whose type you want to change. This ID is returned when you create a dataset and can be found in the dataset listing or dataset detail endpoints.
  </ParamField>
  <ParamField path="column_id" type="string" required>
    The universally unique identifier (UUID) of the column to convert. This ID is returned when a column is created and can be retrieved from the column details endpoint. Only static columns (those without a source type) can have their data type changed.
  </ParamField>
</ApiSection>

<ApiSection title="Request body">
  <ParamField body="new_column_type" type="string" required>
    The target data type to convert the column to. This determines how existing cell values are interpreted and validated during conversion. Valid values: `text`, `number`, `boolean`, `json`, `image`, `audio`, `pdf`.
  </ParamField>

  <ParamField body="preview" type="boolean" optional>
    Controls whether to preview the conversion or execute it. When set to `true` (the default), the API returns a compatibility analysis showing which values can and cannot be converted, without making any changes. Set to `false` to execute the actual conversion, which runs asynchronously as a background task.
  </ParamField>

  <ParamField body="force_update" type="boolean" optional>
    When set to `true`, forces the type conversion even if some cell values cannot be successfully converted to the target type. Inconvertible values may be set to null or a default value. Defaults to `false`, which requires all values to be convertible before proceeding.
  </ParamField>
</ApiSection>

<ApiSection title="Response" status={200} statusText="OK">
  The response structure varies depending on whether the request is in preview mode or execution mode.

  <ResponseField name="message" type="string">
    A human-readable status message. Returns `success` for preview mode results, or `Column type conversion started` when the conversion is initiated in execution mode.
  </ResponseField>

  <ResponseField name="data" type="object">
    Preview data containing conversion compatibility analysis. Only present in preview mode responses.
  </ResponseField>
    <ApiCollapsible title="Show 4 properties">
      <ResponseField name="invalid_count" type="integer">
        The total number of cell values in the column that cannot be converted to the target data type. A count of zero indicates all values are compatible with the target type.
      </ResponseField>
      <ResponseField name="invalid_values" type="array">
        A sample list of cell values that cannot be converted to the target data type. Useful for identifying problematic data before executing the conversion. May be truncated for columns with many invalid values.
      </ResponseField>
      <ResponseField name="valid_conversion_samples" type="object">
        A mapping of sample original values to their converted equivalents, demonstrating how existing data will be transformed. Keys are the original string values and values are the converted representations in the target type.
      </ResponseField>
      <ResponseField name="new_data_type" type="string">
        The target data type that was evaluated in the preview. Matches the `new_column_type` value from the request.
      </ResponseField>
    </ApiCollapsible>

  <ResponseField name="column_id" type="string">
    The UUID of the column being converted. Only present in execution mode responses.
  </ResponseField>

  <ResponseField name="new_data_type" type="string">
    The target data type the column is being converted to. Only present in execution mode responses.
  </ResponseField>

  <ResponseField name="status" type="string">
    The current status of the conversion task. Returns `RUNNING` when the conversion has been initiated. Only present in execution mode responses.
  </ResponseField>
</ApiSection>
