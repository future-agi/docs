<ApiSection title="Request body">
  Fields are grouped by simulation type. Voice-only and text-only fields are rejected by the API if sent with the wrong `simulationType`.

  ### Common fields

  <ParamField body="name" type="string" required>
    A human-readable name for the persona. Must be unique within your workspace and must not conflict with any system-level persona names. Case-insensitive comparison. Maximum 255 characters.
  </ParamField>
  <ParamField body="description" type="string" required>
    A non-empty description that explains the persona's role, behavior, and characteristics.
  </ParamField>
  <ParamField body="simulationType" type="string" optional>
    The type of simulation this persona is designed for. Accepted values: `voice`, `text`. Defaults to `voice`. Determines which type-specific fields are accepted.
  </ParamField>
  <ParamField body="gender" type="array of string" optional>
    Gender characteristics. Accepted values: `male`, `female`.
  </ParamField>
  <ParamField body="ageGroup" type="array of string" optional>
    Age range brackets. Accepted values: `18-25`, `25-32`, `32-40`, `40-50`, `50-60`, `60+`.
  </ParamField>
  <ParamField body="location" type="array of string" optional>
    Geographic locations. Accepted values: `United States`, `Canada`, `United Kingdom`, `Australia`, `India`.
  </ParamField>
  <ParamField body="profession" type="array of string" optional>
    Professional roles. Accepted values: `Student`, `Teacher`, `Engineer`, `Doctor`, `Nurse`, `Business Owner`, `Manager`, `Sales Representative`, `Customer Service`, `Technician`, `Consultant`, `Accountant`, `Marketing Professional`, `Retired`, `Homemaker`, `Freelancer`, `Other`.
  </ParamField>
  <ParamField body="personality" type="array of string" optional>
    Personality traits. Accepted values: `Friendly and cooperative`, `Professional and formal`, `Cautious and skeptical`, `Impatient and direct`, `Detail-oriented`, `Easy-going`, `Anxious`, `Confident`, `Analytical`, `Emotional`, `Reserved`, `Talkative`.
  </ParamField>
  <ParamField body="communicationStyle" type="array of string" optional>
    Communication style descriptors. Accepted values: `Direct and concise`, `Detailed and elaborate`, `Casual and friendly`, `Formal and polite`, `Technical`, `Simple and clear`, `Questioning`, `Assertive`, `Passive`, `Collaborative`.
  </ParamField>
  <ParamField body="multilingual" type="boolean" optional>
    Enables multi-language support. When `true`, the `language` field is required. Defaults to `false`.
  </ParamField>
  <ParamField body="language" type="array of string" optional>
    Languages the persona can communicate in. Accepted values: `English`, `Hindi`. Required when `multilingual` is `true`.
  </ParamField>
  <ParamField body="keywords" type="array of string" optional>
    Keywords or tags for search and categorization.
  </ParamField>
  <ParamField body="customProperties" type="object" optional>
    Key-value metadata. Both keys and values must be non-empty strings.
  </ParamField>
  <ParamField body="additionalInstruction" type="string" optional>
    Free-form behavioral instructions passed to the simulation engine.
  </ParamField>

  ### Voice-only fields
  These fields are only accepted when `simulationType` is `voice`. Sending them with `simulationType: "text"` returns a `400` error.

  <ParamField body="accent" type="array of string" optional>
    Accent options. Accepted values: `American`, `Australian`, `Indian`, `Canadian`, `Neutral`.
  </ParamField>
  <ParamField body="conversationSpeed" type="array of string" optional>
    Conversation speed multipliers. Accepted values: `0.5`, `0.75`, `1.0`, `1.25`, `1.5`. A value of `1.0` is normal speed.
  </ParamField>
  <ParamField body="backgroundSound" type="boolean" optional>
    Whether background sounds are included during voice simulations.
  </ParamField>
  <ParamField body="finishedSpeakingSensitivity" type="array of string" optional>
    Integer values (as strings) from `1` to `10` controlling how sensitive the system is to detecting end of speech.
  </ParamField>
  <ParamField body="interruptSensitivity" type="array of string" optional>
    Integer values (as strings) from `1` to `10` controlling how easily the persona can be interrupted.
  </ParamField>

  ### Text-only fields
  These fields are only accepted when `simulationType` is `text`. Sending them with `simulationType: "voice"` returns a `400` error.

  <ParamField body="tone" type="string" optional>
    Communication tone. Accepted values: `formal`, `casual`, `neutral`. Defaults to `casual`.
  </ParamField>
  <ParamField body="punctuation" type="string" optional>
    Punctuation style. Accepted values: `clean`, `minimal`, `expressive`, `erratic`. Defaults to `clean`.
  </ParamField>
  <ParamField body="slangUsage" type="string" optional>
    Slang usage level. Accepted values: `none`, `light`, `moderate`, `heavy`. Defaults to `light`.
  </ParamField>
  <ParamField body="typosFrequency" type="string" optional>
    Typo frequency. Accepted values: `none`, `rare`, `occasional`, `frequent`. Defaults to `rare`.
  </ParamField>
  <ParamField body="regionalMix" type="string" optional>
    Regional language variation. Accepted values: `none`, `light`, `moderate`, `heavy`. Defaults to `light`.
  </ParamField>
  <ParamField body="emojiUsage" type="string" optional>
    Emoji usage frequency. Accepted values: `never`, `light`, `regular`, `heavy`. Defaults to `light`.
  </ParamField>
  <ParamField body="verbosity" type="string" optional>
    Response length. Accepted values: `brief`, `balanced`, `detailed`. Defaults to `balanced`.
  </ParamField>
</ApiSection>

<ApiSection title="Errors">
  <ParamField name="400" type="Bad Request">
    The request payload is invalid. Common causes include:
    - Required fields (`name`, `description`) are missing or empty
    - The persona name already exists in your workspace or conflicts with a system persona name (case-insensitive)
    - `multilingual` is set to `true` but no `language` array is provided
    - A field contains a value outside the allowed choices (e.g., invalid `tone`, `personality`, or `gender` value)
    - `finishedSpeakingSensitivity` or `interruptSensitivity` contains values outside the 1-10 range
    - `customProperties` contains empty keys or values
    - A voice-only field (e.g., `accent`, `conversationSpeed`) was sent with `simulationType: "text"`, or vice versa

    Example error responses:
    ```json
    {
      "status": false,
      "result": {
        "name": ["A system persona with this name already exists. Please choose a different name."]
      }
    }
    ```
    ```json
    {
      "status": false,
      "result": {
        "tone": ["This field is only valid for text simulation type."],
        "verbosity": ["This field is only valid for text simulation type."]
      }
    }
    ```
  </ParamField>
  <ParamField name="401" type="Unauthorized">
    Authentication credentials were not provided or are invalid. Ensure that both `X-Api-Key` and `X-Secret-Key` headers are included in your request and contain valid, non-expired keys. Verify your API keys in the [Dashboard](https://app.futureagi.com) under Settings.
  </ParamField>
  <ParamField name="500" type="Internal Server Error">
    An unexpected error occurred on the server while processing the request. This is not caused by your input. If this error persists, contact Future AGI support with the request details and timestamp for investigation.
  </ParamField>
</ApiSection>
