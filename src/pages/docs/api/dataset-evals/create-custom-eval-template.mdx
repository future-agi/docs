# Create Custom Eval Template

Creates a new custom evaluation template that can be used to evaluate dataset rows. The template defines evaluation criteria using template variables.

<ApiPlayground
  method="POST"
  endpoint="/model-hub/create_custom_evals/"
  baseUrl="https://api.futureagi.com"
  parameters={[]}
  requestBody={{
    name: "My Custom Eval",
    description: "Evaluates response quality",
    criteria: "Evaluate the {{output}} based on the {{input}} for correctness and completeness.",
    output_type: "Pass/Fail",
    required_keys: ["input", "output"],
    config: {
      model: "turing_small",
      proxy_agi: true,
      visible_ui: true
    },
    tags: ["quality"],
    check_internet: false,
    multi_choice: false
  }}
/>

## Authentication

All requests require authentication via API keys:

| Header | Description |
|--------|-------------|
| `X-Api-Key` | Your API key |
| `X-Secret-Key` | Your secret key |

## Request Body

<ParamField body="name" type="string" required>
  Name of the eval template. Maximum 255 characters. Must be unique across both user-owned and system-owned templates.
</ParamField>

<ParamField body="description" type="string">
  Optional description of the eval template.
</ParamField>

<ParamField body="criteria" type="string" required>
  The evaluation criteria text. Maximum 100,000 characters. Must contain at least one template variable using `{{variable_name}}` syntax (e.g., `{{input}}`, `{{output}}`).
</ParamField>

<ParamField body="output_type" type="string">
  The type of evaluation output. One of: `Pass/Fail`, `score`, `choices`. Default: `Pass/Fail`.
</ParamField>

<ParamField body="required_keys" type="array" required>
  List of required template variable names that must be mapped when using this eval.
</ParamField>

<ParamField body="config" type="object">
  Configuration object for the eval template.

  - **model** (`string`): Model to use for evaluation. Default: `turing_small`.
  - **proxy_agi** (`boolean`): Whether to use the AGI proxy. Default: `true`.
  - **visible_ui** (`boolean`): Whether the eval is visible in the UI. Default: `true`.
  - **reverse_output** (`boolean`): Whether to reverse the output logic.

  Only these keys are allowed: `model`, `proxy_agi`, `visible_ui`, `reverse_output`, `config`.
</ParamField>

<ParamField body="tags" type="array">
  Optional list of tag strings to categorize the eval template.
</ParamField>

<ParamField body="choices" type="object">
  Required when `output_type` is `choices`. A key-value mapping of choice options (e.g., `{"A": "Good", "B": "Bad"}`).
</ParamField>

<ParamField body="check_internet" type="boolean">
  Whether the eval should check internet sources. Default: `false`.
</ParamField>

<ParamField body="multi_choice" type="boolean">
  Whether multiple choices can be selected (only relevant when `output_type` is `choices`). Default: `false`.
</ParamField>

<ParamField body="template_id" type="string">
  Optional UUID of an existing template to base this template on.
</ParamField>

## Responses

### 200

Eval template created successfully.

### 400

Bad request. Possible reasons:

- **Duplicate name** — An eval template with this name already exists.
- **Missing criteria variables** — `criteria` must contain at least one `{{variable}}` placeholder.
- **Missing choices** — `choices` object is required when `output_type` is `choices`.
- **Invalid config keys** — Only `model`, `proxy_agi`, `visible_ui`, `reverse_output`, and `config` are allowed.
- **Name validation failed** — Template name does not meet naming requirements.

### 401

Authentication credentials were not provided or are invalid.

### 500

Internal server error. Failed to create the eval template.
