# Update Custom Eval Config

Updates an existing custom eval config. Supports full replacement via PUT or partial updates via PATCH.

<ApiPlayground
  method="PATCH"
  endpoint="/tracer/custom-eval-config/{id}/"
  baseUrl="https://api.futureagi.com"
  parameters={[{"name": "id", "in": "path", "required": true, "description": "The UUID of the custom eval config to update.", "type": "string"}]}
  requestBody={{"name": "updated-config-name", "config": {}, "mapping": {}}}
/>

## Authentication

All requests require authentication via API keys:

| Header | Description |
|--------|-------------|
| `X-Api-Key` | Your API key |
| `X-Secret-Key` | Your secret key |

## Parameters

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `id` | string (UUID) | Yes | The UUID of the custom eval config to update. |

## Request Body

All fields are optional for PATCH. For PUT, all required fields from create apply.

- **eval_template**: string (UUID)
  The eval template ID.
- **name**: string
  Config name. Max length: 255 characters. Must be unique per project.
- **config**: object
  Configuration dictionary. Normalized against the eval template's config schema.
- **mapping**: object
  Field mapping configuration.
- **filters**: object
  Filter criteria.
- **error_localizer**: boolean
  Enable error localization.
- **kb_id**: string (UUID) or null
  Knowledge base file reference.
- **model**: string or null
  Model to use. Options: `"turing_large"`, `"turing_small"`, `"protect"`, `"protect_flash"`, `"turing_flash"`.

## Responses

### 200

Custom eval config updated successfully.

### 400

Bad Request. Invalid data or duplicate name within the project.

### 401

Unauthorized. Authentication credentials were not provided or are invalid.

### 404

Not Found. The specified custom eval config does not exist.

### 500

Internal Server Error. An unexpected error occurred.

## Code Examples

### cURL

```bash
curl -X PATCH "https://api.futureagi.com/tracer/custom-eval-config/{id}/" \
  -H "X-Api-Key: YOUR_API_KEY" \
  -H "X-Secret-Key: YOUR_SECRET_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "updated-config-name"}'
```

### Python

```python
import requests

url = "https://api.futureagi.com/tracer/custom-eval-config/{id}/"
headers = {
    "X-Api-Key": "YOUR_API_KEY",
    "X-Secret-Key": "YOUR_SECRET_KEY",
    "Content-Type": "application/json"
}
data = {
    "name": "updated-config-name"
}

response = requests.patch(url, headers=headers, json=data)
print(response.json())
```

### JavaScript

```javascript
const response = await fetch("https://api.futureagi.com/tracer/custom-eval-config/{id}/", {
  method: "PATCH",
  headers: {
    "X-Api-Key": "YOUR_API_KEY",
    "X-Secret-Key": "YOUR_SECRET_KEY",
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    "name": "updated-config-name"
  })
});

const data = await response.json();
console.log(data);
```
