<ApiSection title="Request body">
  All fields are optional when using PATCH — only include the fields you want to change. Omitted fields retain their current values.

  <ParamField body="eval_template" type="string" optional>
    The UUID of a different eval template to associate with this config. Changing the template will re-normalize the `config` and `mapping` fields against the new template's schema, which may alter or remove values that don't match.
  </ParamField>

  <ParamField body="name" type="string" optional>
    An updated name for the config. Maximum 255 characters. The new name must be unique within the project among non-deleted configs. Duplicate names within the same project will return a `400` error.
  </ParamField>

  <ParamField body="config" type="object" optional>
    An updated configuration dictionary. This is normalized against the eval template's config schema — fields not defined in the schema are ignored. Useful for adjusting evaluation parameters like thresholds or tone choices.
  </ParamField>

  <ParamField body="mapping" type="object" optional>
    An updated field mapping that redefines how trace span fields correspond to the eval template's expected inputs. For example, changing `{"input": "user_query"}` to `{"input": "prompt"}` redirects which span field is used as the evaluation input.
  </ParamField>

  <ParamField body="filters" type="object" optional>
    Updated filter criteria for the config. Pass an empty object `{}` to clear all filters.
  </ParamField>

  <ParamField body="error_localizer" type="boolean" optional>
    Enable or disable error localization. When `true`, the evaluator identifies the specific part of the output that caused a failure, making debugging easier.
  </ParamField>

  <ParamField body="kb_id" type="string" optional>
    The UUID of a knowledge base file to reference during evaluation. Pass `null` to remove an existing knowledge base association.
  </ParamField>

  <ParamField body="model" type="string" optional>
    The evaluation model to use. Available options: `turing_large` (highest accuracy), `turing_small` (balanced), `protect` (safety-focused), `protect_flash` (fast safety checks), `turing_flash` (fastest general evaluation). Pass `null` to revert to the system default model.
  </ParamField>
</ApiSection>
