---
title: "Create Custom Eval Config"
description: "Creates a new custom eval config by associating an eval template with a project and defining how trace span fields map to the template's expected inputs. The config is automatically normalized against the template's schema."
---

<ApiSection title="Request body">
  <ParamField body="eval_template" type="string" required>
    The unique identifier (UUID) of the eval template to base this config on. The template defines the evaluation logic, expected inputs, and scoring criteria. Must reference a valid, existing template in your organization. Browse available templates in the [Dashboard](https://app.futureagi.com) or via the Eval Templates API.
  </ParamField>

  <ParamField body="name" type="string" required>
    A human-readable name for the custom eval config. Maximum 255 characters. This name must be unique within the project among non-deleted configs. Choose a descriptive name that combines the evaluation type and context, such as `"relevance-check-chatbot"` or `"tone-eval-support-tickets"`.
  </ParamField>

  <ParamField body="project" type="string" required>
    The unique identifier (UUID) of the project to associate this config with. The config will only be available for eval tasks within this project. You can retrieve project IDs from the Projects API or the Dashboard.
  </ParamField>

  <ParamField body="config" type="object" optional>
    A configuration dictionary that customizes the evaluation template's behavior. This object is automatically normalized against the eval template's config schema — any fields not defined in the schema are ignored. For `"tone"` templates, the available tone choices are auto-populated from the template's predefined options.
  </ParamField>

  <ParamField body="mapping" type="object" optional>
    Defines how fields from your trace spans map to the eval template's expected inputs. For example, `{"input": "user_query", "output": "assistant_response"}` tells the evaluator to use the span's `user_query` field as the input and `assistant_response` as the output. Keys with `null` or empty string values are automatically stripped during normalization.
  </ParamField>

  <ParamField body="filters" type="object" optional>
    Optional filter criteria to further restrict which spans this config applies to when used in an eval task. This allows fine-grained control over evaluation scope beyond what the eval task itself specifies.
  </ParamField>

  <ParamField body="error_localizer" type="boolean" optional>
    When set to `true`, enables error localization for this evaluation. Error localization identifies the specific part of the output that caused the evaluation to fail, making it easier to diagnose and fix issues. Defaults to `false`.
  </ParamField>

  <ParamField body="kb_id" type="string" optional>
    The unique identifier (UUID) of a knowledge base file to reference during evaluation. This is typically used for evaluations that compare the output against a ground truth or reference document, such as faithfulness or factual accuracy checks.
  </ParamField>

  <ParamField body="model" type="string" optional>
    The evaluation model to use for scoring. Different models offer varying tradeoffs between speed, accuracy, and cost. Available options: `turing_large` (highest accuracy), `turing_small` (balanced), `protect` (safety-focused), `protect_flash` (fast safety checks), `turing_flash` (fastest general evaluation). When omitted, the system default model is used.
  </ParamField>
</ApiSection>

<ApiSection title="Response" status={200} statusText="OK">
  <ResponseField name="id" type="string">
    The unique identifier (UUID) of the newly created custom eval config. Use this ID when attaching the config to eval tasks via the [Create Eval Task](/docs/api/eval-tasks/create-eval-task) endpoint.
  </ResponseField>
</ApiSection>
