# Create Custom Eval Config

Creates a new custom eval config by associating an eval template with a project and defining a field mapping. The config is normalized against the eval template's schema.

<ApiPlayground
  method="POST"
  endpoint="/tracer/custom-eval-config/"
  baseUrl="https://api.futureagi.com"
  parameters={[]}
  requestBody={{"eval_template": "template-uuid", "name": "my-eval-config", "project": "project-uuid", "config": {}, "mapping": {}}}
/>

## Authentication

All requests require authentication via API keys:

| Header | Description |
|--------|-------------|
| `X-Api-Key` | Your API key |
| `X-Secret-Key` | Your secret key |

## Request Body

- **eval_template** (required): string (UUID)
  The eval template ID to base this config on. Must be a valid, existing template.
- **name** (required): string
  Name for the custom eval config. Max length: 255 characters. Must be unique per project (among non-deleted configs).
- **project** (required): string (UUID)
  The project ID to associate this config with.
- **config**: object
  Configuration dictionary. Automatically normalized against the eval template's config schema. For `"tone"` templates, choices are auto-populated from the template.
- **mapping**: object
  Field mapping configuration. Optional keys with null or empty string values are automatically removed.
- **filters**: object
  Optional filter criteria.
- **error_localizer**: boolean
  Enable error localization. Default: `false`.
- **kb_id**: string (UUID) or null
  Optional knowledge base file reference.
- **model**: string or null
  Model to use for evaluation. Options: `"turing_large"`, `"turing_small"`, `"protect"`, `"protect_flash"`, `"turing_flash"`.

### Example

```json
{
  "eval_template": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "name": "my-eval-config",
  "project": "d4e5f6a7-b8c9-0123-def0-123456789abc",
  "config": {},
  "mapping": {
    "input": "user_query",
    "output": "assistant_response"
  }
}
```

## Responses

### 200

Custom eval config created successfully.

- **status**: string
- **result**: object
  Contains the `id` of the newly created config.

### 400

Bad Request. Possible reasons:
- Missing required fields (`eval_template`, `name`, `project`).
- Invalid `eval_template` UUID — template does not exist.
- Duplicate `name` within the same project.
- Serializer validation errors.

### 401

Unauthorized. Authentication credentials were not provided or are invalid.

### 500

Internal Server Error. An unexpected error occurred.

## Code Examples

### cURL

```bash
curl -X POST "https://api.futureagi.com/tracer/custom-eval-config/" \
  -H "X-Api-Key: YOUR_API_KEY" \
  -H "X-Secret-Key: YOUR_SECRET_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "eval_template": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "name": "my-eval-config",
    "project": "d4e5f6a7-b8c9-0123-def0-123456789abc",
    "config": {},
    "mapping": {"input": "user_query", "output": "assistant_response"}
  }'
```

### Python

```python
import requests

url = "https://api.futureagi.com/tracer/custom-eval-config/"
headers = {
    "X-Api-Key": "YOUR_API_KEY",
    "X-Secret-Key": "YOUR_SECRET_KEY",
    "Content-Type": "application/json"
}
data = {
    "eval_template": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "name": "my-eval-config",
    "project": "d4e5f6a7-b8c9-0123-def0-123456789abc",
    "config": {},
    "mapping": {
        "input": "user_query",
        "output": "assistant_response"
    }
}

response = requests.post(url, headers=headers, json=data)
print(response.json())
```
