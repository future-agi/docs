# Create Custom Eval Config

Creates a new custom eval config by associating an eval template with a project and defining a field mapping. The config is normalized against the eval template's schema.

<ApiPlayground
  method="POST"
  endpoint="/tracer/custom-eval-config/"
  baseUrl="https://api.futureagi.com"
  parameters={[]}
  requestBody={{
    eval_template: "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    name: "my-eval-config",
    project: "d4e5f6a7-b8c9-0123-def0-123456789abc",
    config: {},
    mapping: { input: "user_query", output: "assistant_response" }
  }}
/>

## Authentication

All requests require authentication via API keys:

| Header | Description |
|--------|-------------|
| `X-Api-Key` | Your API key |
| `X-Secret-Key` | Your secret key |

## Request Body

<ParamField body="eval_template" type="string" required>
  UUID of the eval template to base this config on. Must be a valid, existing template.
</ParamField>

<ParamField body="name" type="string" required>
  Name for the custom eval config. Max 255 characters. Must be unique per project among non-deleted configs.
</ParamField>

<ParamField body="project" type="string" required>
  UUID of the project to associate this config with.
</ParamField>

<ParamField body="config" type="object">
  Configuration dictionary. Automatically normalized against the eval template's config schema. For `"tone"` templates, choices are auto-populated from the template.
</ParamField>

<ParamField body="mapping" type="object">
  Field mapping configuration. Optional keys with `null` or empty string values are automatically removed.
</ParamField>

<ParamField body="filters" type="object">
  Optional filter criteria.
</ParamField>

<ParamField body="error_localizer" type="boolean" default="false">
  Enable error localization.
</ParamField>

<ParamField body="kb_id" type="string">
  UUID of a knowledge base file to reference. Optional.
</ParamField>

<ParamField body="model" type="string">
  Model to use for evaluation. One of: `turing_large`, `turing_small`, `protect`, `protect_flash`, `turing_flash`.
</ParamField>

## Response

<ResponseField name="id" type="string">UUID of the newly created custom eval config.</ResponseField>

### Responses

| Status | Description |
|--------|-------------|
| `200` | Custom eval config created successfully. |
| `400` | Missing required fields, invalid `eval_template` UUID, or duplicate `name` within the project. |
| `401` | Unauthorized. Authentication credentials were not provided or are invalid. |
| `500` | Internal Server Error. |
