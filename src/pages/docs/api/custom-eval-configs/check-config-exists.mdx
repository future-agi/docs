---
title: "Check Config Exists"
description: "Checks whether a custom eval config with the same name and mapping already exists in a project. This is useful for validating eval tags before creating configs programmatically, helping you avoid duplicate config errors in automated pipelines."
---

<ApiPlayground
  method="POST"
  endpoint="/tracer/custom-eval-config/check_exists/"
  baseUrl="https://api.futureagi.com"
  requestBody={{
    project_name: "my-project",
    eval_tags: [{ custom_eval_name: "my-eval", eval_name: "relevance", mapping: {} }]
  }}
  responseExample={{
    exists: false,
    message: "All custom eval configurations are valid"
  }}
  responseStatus={200}
  responseStatusText="OK"
/>

<ApiSection title="Authentication">
  <ParamField name="X-Api-Key" type="string" required>
    Your Future AGI API key used to authenticate requests. You can find and manage your API keys in the [Dashboard](https://app.futureagi.com) under Settings.
  </ParamField>
  <ParamField name="X-Secret-Key" type="string" required>
    Your Future AGI secret key, used alongside the API key for request authentication. This is generated when you create an API key in the [Dashboard](https://app.futureagi.com).
  </ParamField>
</ApiSection>

<ApiSection title="Request body">
  <ParamField body="project_name" type="string" required>
    The name of the project to check against. This must match an existing project in your organization. If the project does not exist, the response will indicate so in the `message` field rather than returning an error.
  </ParamField>

  <ParamField body="project_type" type="string" optional>
    The type of the project to look up. Defaults to `"experiment"`. Only relevant if your organization uses multiple project types with overlapping names.
  </ParamField>

  <ParamField body="eval_tags" type="array" required>
    A list of eval tag objects to validate against existing configs in the project. Each object in the array must contain:
    - `custom_eval_name` (string) — The proposed name for the custom eval config.
    - `eval_name` (string) — The name of the eval template to use.
    - `mapping` (object) — The proposed field mapping to compare against existing configs.

    The endpoint checks whether any existing config in the project shares the same name but has a different configuration or mapping, which would indicate a conflict.
  </ParamField>
</ApiSection>

<ApiSection title="Response" status={200} statusText="OK">
  <ResponseField name="exists" type="boolean">
    Returns `true` if a conflicting config was found — meaning a config with the same name but different configuration or mapping already exists in the project. Returns `false` if all proposed configs are safe to create without conflicts.
  </ResponseField>
  <ResponseField name="message" type="string">
    A descriptive message explaining the result of the check. Possible values include:
    - `"Project {name} does not exist"` — The specified project was not found.
    - `"Custom eval '{name}' already exists in project '{project}' with different configuration or mapping"` — A naming conflict was detected.
    - `"All custom eval configurations are valid"` — No conflicts found; safe to proceed with creation.
  </ResponseField>
</ApiSection>
