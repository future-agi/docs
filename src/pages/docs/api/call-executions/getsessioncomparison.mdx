title: "Compare execution sessions"
description: "Compares the current call execution with its most recent rerun snapshot, showing differences in metrics, transcripts, and evaluation results."
---

<ApiSection title="Authentication">
  <ParamField name="X-Api-Key" type="string" required>

  <ParamField name="X-Secret-Key" type="string" required>

<ApiSection title="Path parameters">
  <ParamField path="call_execution_id" type="string" required>
    The unique identifier (UUID) of the call execution whose session comparison you want to retrieve. The call execution must have at least one rerun snapshot available for comparison. You can obtain this ID from the test execution details or the call executions list.
  </ParamField>
</ApiSection>

<ApiSection title="Response" status={200} statusText="OK">
  <ResponseField name="metrics_comparison" type="object">
    A side-by-side comparison of performance metrics between the current call execution and the most recent rerun snapshot.
  </ResponseField>
  <ApiCollapsible title="Show 2 properties">
    <ResponseField name="current" type="object">
      Performance metrics for the current call execution. For voice agents, includes metrics such as `avg_agent_latency`, `talk_ratio`, `user_wpm`, `bot_wpm`, `user_interruption_count`, and `ai_interruption_count`. For text agents, includes `total_tokens`, `input_tokens`, `output_tokens`, `avg_latency_ms`, and `turn_count`.
    </ResponseField>
    <ResponseField name="previous" type="object">
      Performance metrics for the most recent rerun snapshot, using the same metric keys as `current` to allow direct comparison and highlight improvements or regressions.
    </ResponseField>
  </ApiCollapsible>
  <ResponseField name="transcript_comparison" type="object">
    A comparison of the conversation transcripts between the current session and the previous rerun snapshot.
  </ResponseField>
  <ApiCollapsible title="Show 2 properties">
    <ResponseField name="current" type="array of objects">
      Ordered transcript entries for the current session. Each entry contains `role` (e.g., "assistant" or "user") and `content` (the message text).
    </ResponseField>
    <ResponseField name="previous" type="array of objects">
      Ordered transcript entries for the previous rerun snapshot, using the same structure as `current` to allow side-by-side comparison.
    </ResponseField>
  </ApiCollapsible>
  <ResponseField name="recording_urls" type="object">
    Recording URLs for both the current and previous sessions. Only present for voice agent call executions; omitted entirely for text agent comparisons.
  </ResponseField>
  <ApiCollapsible title="Show 2 properties">
    <ResponseField name="current" type="string">
      The URL pointing to the audio recording of the current call execution.
    </ResponseField>
    <ResponseField name="previous" type="string">
      The URL pointing to the audio recording of the previous rerun snapshot.
    </ResponseField>
  </ApiCollapsible>
  <ResponseField name="eval_comparison" type="object">
    A comparison of evaluation results between the current execution and the previous rerun snapshot.
  </ResponseField>
  <ApiCollapsible title="Show 2 properties">
    <ResponseField name="current" type="object">
      Evaluation results for the current execution, keyed by evaluation name with their respective scores, explanations, and other evaluation-specific data.
    </ResponseField>
    <ResponseField name="previous" type="object">
      Evaluation results for the previous rerun snapshot, using the same structure as `current` to track whether agent changes improved or degraded evaluation performance.
    </ResponseField>
  </ApiCollapsible>
</ApiSection>

<ApiSection title="Errors">
  <ParamField name="400" type="Bad Request">
    No rerun snapshot is available for comparison. This error occurs when the call execution has never been rerun, so there is no previous session to compare against. Rerun the call execution first using the [Rerun call executions](/docs/api/test-executions/reruncalls) endpoint, then retry this request.
  </ParamField>
  <ParamField name="401" type="Unauthorized">
    The request could not be authenticated. Verify that both `X-Api-Key` and `X-Secret-Key` headers are present and contain valid, non-expired credentials. Ensure the API key has access to the workspace that owns this call execution.
  </ParamField>
  <ParamField name="404" type="Not Found">
    No call execution was found with the specified ID. The call execution may have been deleted, or the UUID may be incorrect. Verify the `call_execution_id` value and ensure it belongs to your organization.
  </ParamField>
  <ParamField name="500" type="Internal Server Error">
    An unexpected server-side error occurred while generating the session comparison. If this persists, contact support with the request details and the `call_execution_id`.
  </ParamField>
</ApiSection>
