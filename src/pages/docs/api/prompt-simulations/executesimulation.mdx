<ApiSection title="Path parameters">
  <ParamField path="prompt_template_id" type="string" required>
    The UUID of the prompt template that the simulation run belongs to. The prompt template must exist within your organization and must not have been deleted.
  </ParamField>
  <ParamField path="run_test_id" type="string" required>
    The UUID of the simulation run to execute. This must be a valid simulation run associated with the specified prompt template. The simulation run's linked prompt version must still exist and not have been deleted. You can retrieve simulation run IDs from the [List simulations](/docs/api/prompt-simulations/listsimulations) endpoint.
  </ParamField>
</ApiSection>

<ApiSection title="Request body">
  <ParamField body="selectAll" type="boolean" optional>
    Controls the scenario selection behavior for this execution. Defaults to `false`. When set to `true` with an empty `scenarioIds` array, all scenarios associated with the simulation run are executed. When set to `true` with a non-empty `scenarioIds` array, all scenarios are executed *except* those listed in `scenarioIds` (exclusion mode). When set to `false`, only the scenarios listed in `scenarioIds` are executed (inclusion mode).
  </ParamField>
  <ParamField body="scenarioIds" type="array of string" optional>
    An array of scenario UUIDs that modifies which scenarios are executed. The behavior of this field depends on the value of `selectAll`: when `selectAll` is `false`, these are the scenarios to include; when `selectAll` is `true`, these are the scenarios to exclude. Each UUID must reference a valid scenario associated with the simulation run.
  </ParamField>
</ApiSection>
