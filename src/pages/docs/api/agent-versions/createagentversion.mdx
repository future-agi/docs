---
title: "Create Agent Version"
description: "Creates a new version of an existing agent definition by providing updated properties and a commit message. Each version captures a snapshot of the agent's configuration at that point in time, enabling version tracking and rollback."
---

<ApiSection title="Request body">
  All fields except `commitMessage` are optional — only include the fields you want to change from the previous version. Omitted fields inherit their values from the current agent definition.

  <ParamField body="agentType" type="string" optional>
    The updated agent type. Must be `"voice"` or `"text"`. Changing the agent type may require updating related fields (e.g., adding a `provider` when switching to voice).
  </ParamField>

  <ParamField body="agentName" type="string" optional>
    An updated display name for the agent.
  </ParamField>

  <ParamField body="provider" type="string" optional>
    The updated voice provider:
    - `"vapi"` — VAPI voice AI platform
    - `"retell"` — Retell AI voice platform
    - `"eleven_labs"` — Eleven Labs voice synthesis
    - `"livekit"` — LiveKit real-time communication platform
    - `"others"` — Custom or unsupported provider
  </ParamField>

  <ParamField body="apiKey" type="string" optional>
    An updated API key for the external voice provider.
  </ParamField>

  <ParamField body="assistantId" type="string" optional>
    An updated assistant identifier from the external provider.
  </ParamField>

  <ParamField body="description" type="string" optional>
    A description for this version. Use it to document what changed and why, such as `"Improved prompt for better handling of edge cases"`.
  </ParamField>

  <ParamField body="language" type="string" optional>
    The updated primary language as an [ISO 639-1](https://en.wikipedia.org/wiki/List_of_ISO_639_language_codes) code (e.g., `"en"`, `"es"`, `"fr"`).
  </ParamField>

  <ParamField body="knowledgeBase" type="string" optional>
    The UUID of a knowledge base to link to this version. Pass `null` to remove the knowledge base association.
  </ParamField>

  <ParamField body="contactNumber" type="string" optional>
    An updated phone number including the country code prefix (e.g., `"+14155551234"`). Must be 10-12 digits.
  </ParamField>

  <ParamField body="inbound" type="boolean" optional>
    Whether the agent handles inbound calls in this version.
  </ParamField>

  <ParamField body="commitMessage" type="string" required>
    A commit message describing the changes in this version. Works like a git commit message — be descriptive about what changed and why. For example, `"Added multilingual support for Spanish and French"`.
  </ParamField>

  <ParamField body="observabilityEnabled" type="boolean" optional>
    Toggle provider observability integration for this version.
  </ParamField>

  <ParamField body="model" type="string" optional>
    The AI model identifier to use for this agent version (e.g., `"gpt-4o"`, `"claude-3-opus"`).
  </ParamField>

  <ParamField body="modelDetails" type="object" optional>
    Extended model configuration options, such as temperature, max tokens, or other provider-specific settings.
  </ParamField>

  <ParamField body="authenticationMethod" type="string" optional>
    The authentication method used for provider communication (e.g., `"api_key"`).
  </ParamField>

  <ParamField body="languages" type="array of strings" optional>
    A list of supported language codes (e.g., `["en", "es", "fr"]`). Use this when the agent supports multiple languages.
  </ParamField>
</ApiSection>
