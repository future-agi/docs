<ApiSection title="Path parameters">
  <ParamField path="id" type="string" required>
    The unique identifier (UUID) of the eval task to update. The task must not be in a `running` state — use the [Pause Eval Task](/docs/api/eval-tasks/pause-eval-task) endpoint first if needed.
  </ParamField>
</ApiSection>

<ApiSection title="Request body">
  All fields are optional when using PATCH — only include the fields you want to change. Omitted fields retain their current values.

  <ParamField body="name" type="string" optional>
    A new name for the eval task. Must be between 1 and 255 characters long. This name is displayed in the Dashboard and used to identify the task in API responses.
  </ParamField>

  <ParamField body="evals" type="array" optional>
    An updated list of custom eval config UUIDs to run. Replaces the existing list entirely. Must contain at least one valid, non-deleted config. To add a new eval to an existing set, include all previous UUIDs plus the new one.
  </ParamField>

  <ParamField body="sampling_rate" type="number" optional>
    The updated percentage of eligible spans to evaluate, expressed as a decimal between `1.0` and `100.0`. Adjusting this value is useful for controlling evaluation costs or increasing coverage.
  </ParamField>

  <ParamField body="run_type" type="string" optional>
    The updated execution mode. Must be `"continuous"` (evaluate new spans as they arrive) or `"historical"` (evaluate a fixed batch of existing spans). Changing the run type may require setting or clearing `spans_limit` accordingly.
  </ParamField>

  <ParamField body="spans_limit" type="integer" optional>
    The updated maximum number of spans to evaluate. Required when `run_type` is set to `"historical"`. Accepts values between `1` and `1000000`. Ignored for `"continuous"` tasks.
  </ParamField>

  <ParamField body="filters" type="object" optional>
    Updated query filters to scope which spans are eligible for evaluation. Pass an empty object `{}` to clear all filters and evaluate all spans in the project.
  </ParamField>
</ApiSection>

<ApiSection title="Errors">
  <ParamField name="400" type="Bad Request">
    The request failed validation. This can occur if the task is currently `running` (it must be paused first), if `sampling_rate` is outside the valid range, if `evals` contains invalid or deleted config IDs, or if `spans_limit` is missing for a `historical` run type.
  </ParamField>
  <ParamField name="401" type="Unauthorized">
    The request could not be authenticated. Verify that both `X-Api-Key` and `X-Secret-Key` headers are present and contain valid credentials.
  </ParamField>
  <ParamField name="404" type="Not Found">
    No eval task was found with the specified ID. The task may have been deleted, or the ID may be incorrect.
  </ParamField>
</ApiSection>
