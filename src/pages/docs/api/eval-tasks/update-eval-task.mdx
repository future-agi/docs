---
title: "Update Eval Task"
description: "Updates an existing eval task's configuration and optionally reruns it. Supports two edit modes: fresh run (clears all previous results) and edit & rerun (preserves existing results and only runs missing evaluations). Running tasks must be paused before they can be updated."
---

<ApiSection title="Request body">
  <ParamField body="eval_task_id" type="string" required>
    The unique identifier (UUID) of the eval task to update. The task must not be in a `running` or `deleted` state — use the [Pause Eval Task](/docs/api/eval-tasks/pause-eval-task) endpoint first if the task is currently active.
  </ParamField>

  <ParamField body="edit_type" type="string" required>
    Determines how the update is applied:
    - `"fresh_run"` — Deletes all previous evaluation results and starts the task completely fresh with the updated configuration. Use this when you want a clean slate.
    - `"edit_rerun"` — Preserves existing evaluation results and only runs evaluations for missing or newly added configs. Use this to incrementally add evaluations without losing historical data.
  </ParamField>

  <ParamField body="name" type="string" optional>
    A new name for the eval task. Must be between 1 and 255 characters long. This name is displayed in the Dashboard and used to identify the task in API responses.
  </ParamField>

  <ParamField body="evals" type="array" optional>
    An updated list of custom eval config UUIDs to run. Replaces the existing list entirely. Must contain at least one valid, non-deleted config. To add a new eval to an existing set, include all previous UUIDs plus the new one.
  </ParamField>

  <ParamField body="sampling_rate" type="number" optional>
    The updated percentage of eligible spans to evaluate, expressed as a decimal between `1.0` and `100.0`. Adjusting this value is useful for controlling evaluation costs or increasing coverage.
  </ParamField>

  <ParamField body="run_type" type="string" optional>
    The updated execution mode. Must be `"continuous"` (evaluate new spans as they arrive) or `"historical"` (evaluate a fixed batch of existing spans). Changing the run type may require setting or clearing `spans_limit` accordingly.
  </ParamField>

  <ParamField body="spans_limit" type="integer" optional>
    The updated maximum number of spans to evaluate. Required when `run_type` is set to `"historical"`. Accepts values between `1` and `1000000`. Ignored for `"continuous"` tasks.
  </ParamField>

  <ParamField body="filters" type="object" optional>
    Updated query filters to scope which spans are eligible for evaluation. Pass `null` to clear all filters and evaluate all spans in the project.
  </ParamField>
</ApiSection>

  </ParamField>
