title: "Create Eval Task"
description: "Creates a new eval task to automatically run evaluations on trace spans within a project. Eval tasks can operate continuously on incoming spans or evaluate a historical batch of existing spans."
---

<ApiSection title="Request body">
  <ParamField body="project" type="string" required>
    The unique identifier (UUID) of the project this eval task should be associated with. The project must belong to your organization. You can retrieve project IDs from the Projects API or the Dashboard.
  </ParamField>

  <ParamField body="name" type="string" required>
    A human-readable name for the eval task, used to identify it in the Dashboard and API responses. Must be between 1 and 255 characters long. Choose a descriptive name that reflects the evaluation's purpose, such as `"hallucination-check-production"` or `"tone-eval-chatbot"`.
  </ParamField>

  <ParamField body="evals" type="array" required>
    A list of one or more custom eval config UUIDs that define which evaluations to run on each span. Each UUID must reference a valid, non-deleted custom eval config within your organization. You can create custom eval configs using the [Custom Eval Configs API](/docs/api/custom-eval-configs/create-custom-eval-config).
  </ParamField>

  <ParamField body="sampling_rate" type="number" required>
    The percentage of eligible spans that should be evaluated, expressed as a decimal between `1.0` and `100.0`. For example, a value of `50.0` evaluates half of all matching spans. Set to `100.0` to evaluate every span. Lower values reduce cost while still providing representative evaluation coverage.
  </ParamField>

  <ParamField body="run_type" type="string" required>
    Determines how the eval task processes spans. Must be one of:
    - `"continuous"` — The task runs indefinitely, evaluating new spans as they are ingested into the project. Ideal for ongoing production monitoring.
    - `"historical"` — The task evaluates a fixed batch of existing spans and stops once the `spans_limit` is reached. Useful for one-time audits or retroactive analysis.
  </ParamField>

  <ParamField body="spans_limit" type="integer" optional>
    The maximum number of spans to evaluate. This field is required when `run_type` is `"historical"` and ignored for `"continuous"` tasks. Accepts values between `1` and `1000000`. Use this to control the scope and cost of historical evaluation runs.
  </ParamField>

  <ParamField body="filters" type="object" optional>
    An optional set of query filters that narrow which spans are eligible for evaluation. Use filters to target specific span types, attributes, or time ranges. When omitted, all spans in the project are eligible based on the `sampling_rate`.
  </ParamField>
</ApiSection>

<ApiSection title="Response" status={200} statusText="OK">
  <ResponseField name="id" type="string">
    The unique identifier (UUID) of the newly created eval task. Use this ID to retrieve, update, pause, or delete the task via the Eval Tasks API.
  </ResponseField>
</ApiSection>
