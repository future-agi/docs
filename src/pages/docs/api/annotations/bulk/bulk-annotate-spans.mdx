<ApiSection title="Response" status={200} statusText="OK">
  <ResponseField name="message" type="string">Always `"Bulk annotation completed"`.</ResponseField>
  <ResponseField name="annotations_created" type="integer">Number of new annotations created.</ResponseField>
  <ResponseField name="annotations_updated" type="integer">Number of existing annotations updated.</ResponseField>
  <ResponseField name="notes_created" type="integer">Number of notes created.</ResponseField>
  <ResponseField name="succeeded_count" type="integer">Total of created + updated + notes.</ResponseField>
  <ResponseField name="errors_count" type="integer">Number of records that failed.</ResponseField>
  <ResponseField name="warnings_count" type="integer">Number of warnings (e.g. existing annotations updated).</ResponseField>
  <ResponseField name="warnings" type="array" optional>Present when warnings occurred. Each entry has `record_index`, `span_id`, `label_id`, `warning`.</ResponseField>
  <ResponseField name="errors" type="array" optional>Present when errors occurred. Each entry has `record_index`, `span_id`, and an error message field.</ResponseField>
</ApiSection>

<Note>
  Authentication uses the API key — the annotator identity is always the authenticated user. The `annotator_id` field is not accepted per annotation; the system uses the authenticated user for all annotations in the request.

  Provide exactly one value field per annotation (`value`, `value_float`, `value_bool`, or `value_str_list`) matching the label type. Providing the wrong field or multiple value fields returns a per-record error.
</Note>
