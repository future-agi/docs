<ApiSection title="Path parameters">
  <ParamField path="run_test_id" type="string" required>
    The universally unique identifier (UUID) of the test run whose components you want to update. The test run must exist and be accessible within your workspace.
  </ParamField>
</ApiSection>

<ApiSection title="Request body">
  <ParamField body="agentDefinitionId" type="string" optional>
    The UUID of the new agent definition to associate with this test run. Changing the agent definition replaces the AI agent that will be tested in future executions. The agent definition must exist in your workspace and be compatible with the test run's configuration. Existing test execution data is not affected by this change.
  </ParamField>
  <ParamField body="version" type="string" optional>
    The UUID of a specific agent version to use for this test run. This allows you to pin the test run to a particular version of the agent definition, ensuring consistent behavior across executions. If omitted, the test run uses the currently active version of the agent definition. The version must belong to the agent definition associated with the test run.
  </ParamField>
  <ParamField body="simulatorAgentId" type="string" optional>
    The UUID of the simulator agent to use for generating test interactions. The simulator agent acts as the counterpart in test conversations, simulating user behavior according to the configured scenarios. The simulator agent must exist in your workspace.
  </ParamField>
  <ParamField body="scenarios" type="array of strings" optional>
    An array of scenario UUIDs to associate with this test run. Scenarios define the test cases, conversation flows, and expected behaviors that the agent will be tested against. Updating this field replaces the entire set of scenarios -- any previously associated scenarios not included in the new array will be removed. Each UUID must reference an existing scenario in your workspace.
  </ParamField>
  <ParamField body="enableToolEvaluation" type="boolean" optional>
    Controls whether tool evaluation is enabled for this test run. When set to `true`, the evaluation process will also assess how effectively the agent uses its available tools during test executions. This feature requires the agent definition to have both an `api_key` and an `assistant_id` configured. Attempting to enable tool evaluation without these prerequisites will result in a `400` error. Defaults to `false`.
  </ParamField>
</ApiSection>

<ApiSection title="Response" status={200} statusText="OK">
  <ResponseField name="data" type="object">
    The updated test run object reflecting all component changes. Includes the full test run configuration with the newly assigned agent definition, version, simulator agent, scenarios, and tool evaluation settings.
  </ResponseField>
</ApiSection>

Verify that all referenced resources exist and that the agent meets the prerequisites for any enabled features.

  </ParamField>
  <ParamField name="404" type="Not Found">
