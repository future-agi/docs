  <ParamField path="run_test_id" type="string" required>
    The universally unique identifier (UUID) of the test run containing the executions you want to compare. All execution IDs provided in the `execution_ids` query parameter must belong to this test run.

<ApiSection title="Query parameters">
  <ParamField query="execution_ids" type="string" required>
    A JSON-encoded array of test execution UUIDs to compare. Each UUID must reference an existing execution within the specified test run. You must provide at least two execution IDs to perform a meaningful comparison. Example value: `["uuid1","uuid2"]`. The value must be URL-encoded when passed as a query parameter (e.g., `%5B%22uuid1%22%2C%22uuid2%22%5D`).
  </ParamField>
</ApiSection>

<ApiSection title="Response" status={200} statusText="OK">
  <ResponseField name="comparison" type="object">
    A dictionary keyed by execution ID, where each key maps to an object containing the evaluation summary metrics for that specific execution. This structure allows you to directly compare evaluation performance, scores, and pass/fail rates across multiple test executions side by side, making it easy to track how evaluation results change between runs.
  </ResponseField>
</ApiSection>
