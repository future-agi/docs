<ApiSection title="Path parameters">
  <ParamField path="run_test_id" type="string" required>
    The UUID of the test run whose call executions you want to retrieve. Call executions are aggregated across all test executions that belong to this test run, giving you a unified view of every individual simulation call. This identifier is returned when you create a test run via the [Create Test Run](/docs/api/run-tests/createruntest) endpoint.
  </ParamField>
</ApiSection>

<ApiSection title="Query parameters">
  <ParamField query="search" type="string" optional>
    A search string used to filter call executions by phone number or scenario name. The search performs a case-insensitive partial match. This is particularly useful when you want to find calls associated with a specific scenario or contact. Leave empty to return all call executions without filtering.
  </ParamField>
  <ParamField query="status" type="string" optional>
    Filters call executions to only those matching the specified call status. Use this to isolate completed calls, failed calls, or calls in other states. When omitted, call executions of all statuses are returned.
  </ParamField>
  <ParamField query="limit" type="integer" optional>
    The maximum number of call execution records to return per page. Controls the page size for paginated results. Defaults to `10` if not specified. Must be a positive integer. Consider using higher values when analyzing large test runs with many calls.
  </ParamField>
  <ParamField query="page" type="integer" optional>
    The page number to retrieve in the paginated result set. Defaults to `1` (the first page). Use this in combination with `limit` to page through all call executions for a test run.
  </ParamField>
</ApiSection>
