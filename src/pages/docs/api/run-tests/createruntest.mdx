<ApiSection title="Request body">
  <ParamField body="name" type="string" required>
    A human-readable name for the test run. This name is displayed across the platform when referencing this test run and is used for search and filtering. Must be unique within your organization and cannot exceed 255 characters. Choose a descriptive name that conveys the purpose of the test (e.g., "Checkout Flow Regression v2.1").
  </ParamField>
  <ParamField body="description" type="string">
    An optional free-text description providing additional context about the test run. Use this field to document the goals, conditions, or expected outcomes of the test. There is no character limit, but concise descriptions are recommended for readability in the dashboard.
  </ParamField>
  <ParamField body="scenarioIds" type="array of string" required>
    An array of scenario UUIDs that this test run should execute against. Each UUID must reference a valid scenario that belongs to your organization. The array must contain at least one scenario ID. Scenarios define the conversation flows and expected behaviors that the agent will be tested against. You can retrieve available scenario IDs from the [List Scenarios](/docs/api/scenarios/listscenarios) endpoint.
  </ParamField>
  <ParamField body="agentDefinitionId" type="string" required>
    The UUID of the agent definition that this test run will evaluate. The agent definition specifies the configuration, system prompt, tools, and model used by the agent under test. This must reference an existing agent definition within your organization. Retrieve available agent definitions using the [List Agents](/docs/api/agents/listagents) endpoint.
  </ParamField>
  <ParamField body="agentVersion" type="string">
    The UUID of a specific version of the agent definition to test against. If omitted, the system automatically selects the currently active version of the specified agent definition. Use this field when you want to test against a historical or draft version that is not yet activated.
  </ParamField>
  <ParamField body="evalConfigIds" type="array of string">
    An array of UUIDs referencing existing evaluation configurations to associate with this test run. Evaluation configurations define the metrics, scoring criteria, and thresholds used to assess agent performance during execution. These must be pre-existing configs in your organization. If you need to create new evaluation configs inline, use the `evaluationsConfig` field instead.
  </ParamField>
  <ParamField body="evaluationsConfig" type="array of objects">
    An array of evaluation configuration objects to create and associate with this test run in a single request. Each object must include `template_id` (the evaluation template UUID), `name` (a display name for the config), `config` (template-specific configuration parameters), and `mapping` (field mappings between scenario data and evaluation inputs). Use this field to define new evaluation criteria on the fly without pre-creating them.
  </ParamField>
  <ParamField body="datasetRowIds" type="array of string">
    An array of dataset row UUIDs to restrict the test run to specific data entries within the associated scenarios. When provided, only the specified rows will be used during test execution, allowing you to run targeted tests on a subset of your scenario data. If omitted, all rows from the associated scenarios are included.
  </ParamField>
  <ParamField body="enableToolEvaluation" type="boolean">
    Controls whether tool usage by the agent is included in the evaluation process. When set to `true`, the evaluation framework will assess the correctness, appropriateness, and timing of tool calls made by the agent during the simulation. Defaults to `false`. Enable this when your agent relies on external tools and you want to validate that tool interactions are handled correctly.
  </ParamField>
</ApiSection>
