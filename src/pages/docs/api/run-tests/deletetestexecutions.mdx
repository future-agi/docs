---
title: "Delete test executions"
description: "Bulk-deletes test executions from a test run. Cannot delete executions that are currently running, pending, or cancelling."
---

<ApiSection title="Path parameters">
  <ParamField path="run_test_id" type="string" required>
    The universally unique identifier (UUID) of the test run from which you want to delete test executions. All execution IDs specified in the request body must belong to this test run.
  </ParamField>
</ApiSection>

<ApiSection title="Request body">
  <ParamField body="testExecutionIds" type="array of strings" optional>
    An array of test execution UUIDs to delete from the test run. This field is required when `selectAll` is `false` or omitted, and must contain at least one valid UUID. Each UUID must reference an existing execution within the specified test run. Executions with a status of `RUNNING`, `PENDING`, or `CANCELLING` cannot be deleted -- you must wait for them to complete or cancel them first before attempting deletion.
  </ParamField>
  <ParamField body="selectAll" type="boolean" optional>
    When set to `true`, deletes all eligible test executions within the test run, ignoring the `testExecutionIds` field. Eligible executions are those not in `RUNNING`, `PENDING`, or `CANCELLING` status. Defaults to `false`. Use this option with caution, as the operation is irreversible and will remove all completed, failed, and cancelled executions from the test run.
  </ParamField>
</ApiSection>

<ApiSection title="Response" status={200} statusText="OK">
  <ResponseField name="message" type="string">
    A human-readable confirmation message summarizing the deletion operation, including the number of executions deleted (e.g., `"Successfully deleted 2 test execution(s)."`).
  </ResponseField>
  <ResponseField name="runTestId" type="string">
    The UUID of the test run from which the executions were deleted, echoed back for confirmation and client-side reference.
  </ResponseField>
  <ResponseField name="deletedCount" type="integer">
    The total number of test executions that were successfully deleted in this operation. This count only includes executions that were actually removed and excludes any that were skipped due to ineligible status.
  </ResponseField>
  <ResponseField name="deletedIds" type="array of strings">
    An array of UUIDs for all test executions that were successfully deleted. Use this list to update client-side state or confirm that specific executions were removed as expected.
  </ResponseField>
</ApiSection>
