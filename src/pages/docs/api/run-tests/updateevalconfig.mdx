<ApiSection title="Request body">
  <ParamField body="config" type="object" optional>
    An object containing the updated evaluation configuration parameters. This object supports nested `config` and `mapping` keys, allowing you to update the evaluation template's configuration and field mapping in a single request. Only the fields you include will be updated; omitted fields retain their current values.
  </ParamField>
  <ParamField body="mapping" type="object" optional>
    Updated field mapping configuration that redefines how test execution data fields are mapped to the evaluation template's expected input fields. Use this to adjust data flow between your test outputs and evaluation criteria without recreating the entire configuration.
  </ParamField>
  <ParamField body="model" type="string" optional>
    The model to use for running evaluations with this configuration. Changing the model may affect evaluation results, latency, and cost. Available models depend on your workspace configuration and subscription tier.
  </ParamField>
  <ParamField body="errorLocalizer" type="boolean" optional>
    Controls whether error localization is enabled for this evaluation configuration. When `true`, the evaluation provides granular feedback identifying specific parts of the output that contributed to evaluation failures. When `false`, only aggregate pass/fail results are reported.
  </ParamField>
  <ParamField body="kbId" type="string" optional>
    The UUID of a knowledge base file to associate with this evaluation configuration. Knowledge bases provide reference material that the evaluation can use for grounding and comparison during assessment. The referenced knowledge base must exist in your workspace.
  </ParamField>
  <ParamField body="name" type="string" optional>
    An updated human-readable name for this evaluation configuration. The name must remain unique within the test run. Changing the name does not affect evaluation history or results -- it only updates the display label in the Dashboard and API responses.
  </ParamField>
  <ParamField body="run" type="boolean" optional>
    When set to `true`, triggers an immediate rerun of the evaluation on the specified test execution after the configuration update is applied. This is useful for immediately validating configuration changes against existing test data. When `false` (or omitted), the configuration is updated without triggering any evaluation execution. Defaults to `false`.
  </ParamField>
  <ParamField body="testExecutionId" type="string" optional>
    The UUID of the test execution to rerun the evaluation against. This field is required when `run` is set to `true` and is ignored otherwise. The specified execution must belong to the same test run and must be in a completed state.
  </ParamField>
</ApiSection>
