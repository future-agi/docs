<ApiSection title="Request body">
  <ParamField body="evaluationsConfig" type="array of objects" required>
    An array of evaluation configuration objects to add to the test run. Each object defines a complete evaluation setup, linking an evaluation template with its field mappings, configuration parameters, and execution settings. You can add multiple configurations in a single request to set up batch evaluation workflows.

    Each object in the array supports the following fields:

    - **`templateId`** (string, UUID, required) -- The unique identifier of the evaluation template to use. This determines which evaluation logic and criteria are applied. The template must exist in your workspace and be compatible with the test run's data structure.

    - **`name`** (string, required) -- A human-readable name for this evaluation configuration. Must be unique within the test run. This name is displayed in the Dashboard and used to identify the configuration in evaluation summary reports. Choose a descriptive name that clearly indicates the evaluation purpose (e.g., "Tone Check", "Accuracy Validation").

    - **`config`** (object, optional) -- Additional evaluation configuration parameters that customize the behavior of the evaluation template. The structure of this object depends on the specific template being used. Refer to the template documentation for available configuration options.

    - **`mapping`** (object, optional) -- Field mapping configuration that defines how test execution data fields are mapped to the evaluation template's expected input fields. This allows you to connect your test data to the evaluation criteria without modifying the template itself.

    - **`filters`** (object, optional) -- Filter criteria that restrict which test execution results are subject to this evaluation. Use filters to selectively evaluate subsets of your test data based on specific conditions.

    - **`errorLocalizer`** (boolean, optional) -- When set to `true`, enables error localization for this evaluation configuration. Error localization provides more granular feedback by identifying the specific parts of the output that contributed to evaluation failures. Defaults to `false`.

    - **`model`** (string, optional) -- The model to use for running this evaluation. Specifies which AI model powers the evaluation logic. Available models depend on your workspace configuration and subscription tier.
  </ParamField>
</ApiSection>
