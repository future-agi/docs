<ApiSection title="Path parameters">
  <ParamField path="run_test_id" type="string" required>
    The universally unique identifier (UUID) of the test run containing the executions you want to rerun. All execution IDs specified in the request body must belong to this test run.
  </ParamField>

<ApiSection title="Request body">
  <ParamField body="rerunType" type="string" required>
    Specifies the type of rerun to perform. Accepts one of two values: `eval_only` re-runs only the evaluation phase using the existing call execution data, which is faster and does not consume additional call credits since it reuses previously captured agent responses (use this mode when you have updated evaluation configurations and want to re-assess existing results); `call_and_eval` re-executes both the agent calls and the subsequent evaluations from scratch, generating new call execution data and then evaluating it (use this mode when you need fresh agent responses, such as after updating the agent definition, version, or scenarios).
  </ParamField>
  <ParamField body="testExecutionIds" type="array of strings" optional>
    An array of test execution UUIDs to rerun. This field is required when `selectAll` is `false` or omitted, and must contain at least one valid UUID. Each UUID must reference an existing execution within the specified test run. You must provide either this field or set `selectAll` to `true`.
  </ParamField>
  <ParamField body="selectAll" type="boolean" optional>
    When set to `true`, reruns all test executions within the test run, ignoring the `testExecutionIds` field. This applies the specified `rerunType` to every execution in the test run. Defaults to `false`. Use this option when you want to comprehensively re-evaluate or re-execute all test data after making broad configuration changes.
  </ParamField>
</ApiSection>

<ApiSection title="Response" status={200} statusText="OK">
  <ResponseField name="message" type="string">
    A confirmation message indicating that the rerun operation has been initiated successfully. The actual rerun process runs asynchronously -- use the test execution status endpoints to monitor progress.
  </ResponseField>
</ApiSection>
