<ApiSection title="Query parameters">
  <ParamField query="search" type="string" optional>
    A search string used to filter executions by status or scenario name. The search performs a case-insensitive partial match across execution metadata. Leave empty to return all executions without filtering.
  </ParamField>
  <ParamField query="status" type="string" optional>
    Filters executions to only those matching the specified status. Accepted values are `PENDING` (execution is queued but has not started), `RUNNING` (execution is currently in progress), `COMPLETED` (execution finished successfully), `FAILED` (execution encountered an unrecoverable error), and `CANCELLED` (execution was manually cancelled). When omitted, executions of all statuses are returned.
  </ParamField>
  <ParamField query="limit" type="integer" optional>
    The maximum number of execution records to return per page. Controls the page size for paginated results. Defaults to `10` if not specified. Must be a positive integer.
  </ParamField>
  <ParamField query="page" type="integer" optional>
    The page number to retrieve in the paginated result set. Defaults to `1` (the first page). Use this in combination with `limit` to navigate through large result sets.
  </ParamField>
</ApiSection>
