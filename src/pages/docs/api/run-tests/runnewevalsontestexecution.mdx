<ApiSection title="Path parameters">
  <ParamField path="run_test_id" type="string" required>
    The universally unique identifier (UUID) of the test run containing the executions you want to evaluate. The specified evaluation configurations and test executions must all belong to this test run.
  </ParamField>
</ApiSection>

<ApiSection title="Request body">
  <ParamField body="testExecutionIds" type="array of strings" optional>
    An array of test execution UUIDs to run the new evaluations against. This field is required when `selectAll` is `false` or omitted, and must contain at least one valid UUID. Each UUID must reference an existing execution within the specified test run, and only executions with a `COMPLETED` status are eligible for evaluation. Executions in other states (e.g., `RUNNING`, `PENDING`, `FAILED`) will be skipped or cause an error.
  </ParamField>
  <ParamField body="selectAll" type="boolean" optional>
    When set to `true`, runs the specified evaluations against all completed test executions within the test run, ignoring the `testExecutionIds` field. Only executions with a `COMPLETED` status are included. Defaults to `false`. Use this option when you want to apply new evaluation configurations across all available test data.
  </ParamField>
  <ParamField body="evalConfigIds" type="array of strings" required>
    An array of evaluation configuration UUIDs specifying which evaluations to run on the selected test executions. Must contain at least one valid UUID. Each ID must reference an existing evaluation configuration within the test run. These configurations define the evaluation templates, field mappings, and models used to assess the test execution results.
  </ParamField>
  <ParamField body="enableToolEvaluation" type="boolean" optional>
    When set to `true`, enables tool evaluation as part of the evaluation process. Tool evaluation assesses how effectively the agent uses its available tools during the test execution. The agent must have the appropriate API key and assistant ID configured for tool evaluation to function correctly. Defaults to `false`.
  </ParamField>
</ApiSection>

<ApiSection title="Response" status={200} statusText="OK">
  <ResponseField name="message" type="string">
    A confirmation message indicating that the evaluations have been started successfully (e.g., `"Evaluations started successfully"`). The evaluations run asynchronously -- use the evaluation summary endpoints to monitor progress and retrieve results.
  </ResponseField>
  <ResponseField name="run_test_id" type="string">
    The UUID of the test run on which the evaluations were initiated, echoed back for confirmation and client-side reference.
  </ResponseField>
  <ResponseField name="call_execution_count" type="integer">
    The total number of call executions that are being evaluated as part of this operation. This count reflects the number of individual call-level evaluations queued across all selected test executions and evaluation configurations.
  </ResponseField>
</ApiSection>
