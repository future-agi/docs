title: "Execute a test run"
description: "Triggers the execution of a specified test run. The execution can be customized to include or exclude specific scenarios."

<ApiSection title="Path parameters">
  <ParamField path="run_test_id" type="string" required>
    The UUID of the test run to execute. This identifier is returned when you create a test run via the [Create Test Run](/docs/api/run-tests/createruntest) endpoint, or can be found in the results from the [List Test Runs](/docs/api/run-tests/listruntests) endpoint. The referenced test run must have at least one scenario associated with it.
  </ParamField>
</ApiSection>

<ApiSection title="Request body">
  <ParamField body="selectAll" type="boolean" optional>
    Controls the scenario selection behavior in combination with the `scenarioIds` field. When set to `true`, all scenarios associated with the test run are selected for execution except those listed in `scenarioIds` (exclusion mode). When set to `false`, only the scenarios listed in `scenarioIds` are executed (inclusion mode). If `scenarioIds` is empty, all scenarios run regardless of this flag's value. This toggle provides a convenient way to run most scenarios while excluding a few, or to cherry-pick specific scenarios.
  </ParamField>
  <ParamField body="scenarioIds" type="array of string" optional>
    An array of scenario UUIDs that modifies the set of scenarios to execute, interpreted based on the `selectAll` flag. When `selectAll` is `true`, these scenario IDs are excluded from execution. When `selectAll` is `false`, these are the only scenarios that will be executed. If this array is empty, all scenarios associated with the test run are executed. Each UUID must reference a scenario that is already associated with the test run.
  </ParamField>
  <ParamField body="simulatorId" type="string" optional>
    The UUID of a specific simulator agent to use for this execution. The simulator agent plays the role of the user in the conversation, driving the interaction according to the scenario script. If omitted, the system uses the default simulator agent configured for the test run or the organization's default simulator.
  </ParamField>
</ApiSection>

<ApiSection title="Response" status={200} statusText="OK">
  <ResponseField name="message" type="string">
    A confirmation message indicating the test execution was initiated successfully. The value is `"Test execution started successfully"`. Note that this means the execution has been queued -- actual scenario processing begins asynchronously.
  </ResponseField>
  <ResponseField name="execution_id" type="string">
    The unique UUID identifier assigned to this execution instance. Use this ID to track the execution's progress via the [Get Test Executions](/docs/api/run-tests/gettestexecutions) endpoint.
  </ResponseField>
  <ResponseField name="run_test_id" type="string">
    The UUID of the test run that this execution belongs to, echoed back for confirmation.
  </ResponseField>
  <ResponseField name="status" type="string">
    The initial status of the execution. This will always be `"PENDING"` immediately after creation, indicating the execution is queued and will begin processing shortly. The status transitions through `RUNNING`, and eventually to `COMPLETED`, `FAILED`, or `CANCELLED`.
  </ResponseField>
  <ResponseField name="total_scenarios" type="integer">
    The total number of scenarios that will be executed in this run, after applying the `selectAll` and `scenarioIds` filters. Use this to gauge the expected scope of the execution.
  </ResponseField>
  <ResponseField name="total_calls" type="integer">
    The total number of individual simulation calls that will be made across all selected scenarios. Each scenario may generate multiple calls depending on the number of dataset rows it contains.
  </ResponseField>
  <ResponseField name="scenario_ids" type="array of string">
    The final resolved list of scenario UUIDs that will be executed, after applying the `selectAll` and `scenarioIds` filters from the request body.
  </ResponseField>
</ApiSection>

<ApiSection title="Errors">
  <ParamField name="400" type="Bad Request">

This can occur when the test run has no scenarios associated, when referenced scenario IDs are invalid, or when the agent definition or evaluation configuration is misconfigured. Review the error message in the response body for specific details about what went wrong.
  </ParamField>
  <ParamField name="401" type="Unauthorized">
    Authentication credentials were not provided or are invalid. Ensure that both `X-Api-Key` and `X-Secret-Key` headers are included in your request and contain valid, non-expired keys. Verify your API keys in the [Dashboard](https://app.futureagi.com) under Settings.
  </ParamField>
  <ParamField name="404" type="Not Found">
    The specified test run or the authenticated user's organization could not be found. Verify that the `run_test_id` path parameter contains a valid UUID referencing an existing test run within your organization.
  </ParamField>
  <ParamField name="500" type="Internal Server Error">

</ApiSection>
