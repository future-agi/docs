<ApiSection title="Query parameters">
  <ParamField query="search" type="string">
    A case-insensitive search string used to filter test runs by their name or the name of their associated agent definition. The search performs a partial match, so searching for "checkout" will return test runs named "Checkout Flow Test" or those linked to an agent definition named "Checkout Agent". Leave empty to return all test runs without filtering.
  </ParamField>
  <ParamField query="limit" type="integer">
    The maximum number of test run records to return per page. Controls the page size for paginated results. Defaults to `10` if not specified. Use higher values to retrieve more results in a single request, or lower values for faster responses when you only need a few records. Must be a positive integer.
  </ParamField>
  <ParamField query="page" type="integer">
    The page number to retrieve in the paginated result set. Defaults to `1` (the first page). Use this in combination with `limit` to navigate through large result sets. The response includes `next` and `previous` URLs for convenient pagination.
  </ParamField>
  <ParamField query="simulation_type" type="string">
    Filters test runs by their source type. Accepted values are `agent_definition` (test runs configured with an agent definition) and `prompt` (test runs configured with a prompt template). When omitted, test runs of all source types are returned.
  </ParamField>
  <ParamField query="prompt_template_id" type="string">
    Filters test runs to only those associated with the specified prompt template UUID. This is useful when you want to see all test runs that were created for a particular prompt template. The UUID must reference a valid prompt template within your organization.
  </ParamField>
</ApiSection>
