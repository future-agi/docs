<TLDR>
- **Evaluations:** `pip install ai-evaluation` — 50+ metrics, guardrails, AutoEval
- **Core SDK:** `pip install futureagi` — datasets, prompts, knowledge bases
- **Tracing:** `pip install fi-instrumentation-otel` + a framework instrumentor
- All packages authenticate with `FI_API_KEY` and `FI_SECRET_KEY`
</TLDR>

Future AGI is a suite of Python packages for the full AI engineering lifecycle — evaluate outputs, trace LLM calls, optimize prompts, and test agents at scale. Install only what you need.

## Getting Started

<Steps>
  <Step title="Install the packages you need">
    Most users start with evaluations. This single install gives you 50+ eval metrics and the core SDK:

    ```bash
    pip install ai-evaluation
    ```

    Requires Python 3.10+. This also installs `futureagi` (datasets, prompts, knowledge bases) automatically.

    Want to trace your LLM calls too? Add the instrumentor for your provider:

    ```bash
    pip install fi-instrumentation-otel traceai-openai
    ```
  </Step>
  <Step title="Set your API keys">
    Get your keys from **Settings → API Keys** in the [Future AGI dashboard](https://app.futureagi.com):

    ```bash
    export FI_API_KEY="your-api-key"
    export FI_SECRET_KEY="your-secret-key"
    ```
  </Step>
  <Step title="Run your first evaluation">
    Check if an LLM response has the right tone — copy, paste, and run:

    ```python
    from fi.evals import Evaluator, Tone

    evaluator = Evaluator()

    result = evaluator.evaluate(
        eval_templates=[Tone()],
        inputs=[{
            "query": "Write a professional email",
            "response": "Dear Sir/Madam, I hope this message finds you well..."
        }],
        model_name="turing_flash"
    )

    print(result)
    # → BatchRunResult(score=0.95, passed=True, reason="Professional tone detected")
    ```
  </Step>
</Steps>

## Package Ecosystem

Future AGI is not one package — it's a modular suite. Here's what each one does:

| Package | Install | What it does | Python |
|---------|---------|--------------|--------|
| **futureagi** | `pip install futureagi` | Datasets, prompt versioning, knowledge bases | 3.9+ |
| **ai-evaluation** | `pip install ai-evaluation` | 50+ eval metrics, guardrails, streaming eval | 3.10+ |
| **fi-instrumentation-otel** | `pip install fi-instrumentation-otel` | OpenTelemetry tracing for AI apps | 3.9+ |
| **traceAI-\*** | `pip install traceai-openai` | Auto-instrumentation for 45+ frameworks | 3.9+ |
| **agent-opt** | `pip install agent-opt` | Prompt optimization (6 algorithms) | 3.10+ |
| **agent-simulate** | `pip install agent-simulate` | Voice AI simulation and testing | 3.10+ |

### How they depend on each other

```
futureagi                    ← standalone base layer
  └── ai-evaluation          ← installs futureagi automatically
        └── agent-opt        ← installs ai-evaluation automatically

fi-instrumentation-otel      ← standalone tracing layer
  ├── traceai-*              ← each installs fi-instrumentation-otel
  └── agent-simulate         ← installs fi-instrumentation-otel
```

<Tip>
  You don't need to install dependencies manually. `pip install ai-evaluation` gives you `futureagi` too. `pip install traceai-openai` gives you `fi-instrumentation-otel` too.
</Tip>

## Tracing — `fi-instrumentation-otel` + `traceAI-*`

```bash
pip install fi-instrumentation-otel traceai-openai
```

Add observability to your AI stack in two lines. Install the core tracing library plus an instrumentor for each framework you use. Every LLM call, retrieval step, and agent decision gets captured and sent to your Future AGI dashboard automatically.

```python
from fi_instrumentation import register
from fi_instrumentation.fi_types import ProjectType

trace_provider = register(
    project_name="my-project",
    project_type=ProjectType.OBSERVE,
)

# Then instrument your framework:
from traceai.openai import OpenAIInstrumentor
OpenAIInstrumentor().instrument(tracer_provider=trace_provider)

# That's it — all OpenAI calls are now traced
```

## Prompt Optimization — `agent-opt`

```bash
pip install agent-opt
```

Improve your prompts systematically instead of guessing. Choose from 6 optimization strategies — Random Search, Bayesian Search, ProTeGi, Meta-Prompt, PromptWizard, and GEPA — and let the optimizer find the best prompt for your task using eval metrics as the scoring function.

```python
from fi.opt.optimizers import MetaPromptOptimizer
from fi.opt.generators import LiteLLMGenerator
from fi.opt.base.evaluator import Evaluator
from fi.opt.datamappers import BasicDataMapper
```
