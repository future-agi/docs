## About

EvalTags are the configuration objects that tell Prototype which evaluations to run on your application's outputs. You pass them into `register()`, and from that point every generation is automatically scored:no separate eval step required.

The `mapping` attribute inside each EvalTag is how the eval engine finds the right data in your traces. Different evals need different inputs (a toxicity check needs the user message; a context adherence check needs both the context and the model response). Mapping tells the platform exactly which span attribute to use for each required input.

---

## What an EvalTag contains

| Field | What it does |
|---|---|
| `eval_name` | Which evaluation to run. Must be a valid `EvalName` enum value (e.g. `EvalName.TOXICITY`). |
| `type` | Where to apply the eval. Use `EvalTagType.OBSERVATION_SPAN` for LLM spans. |
| `value` | The kind of span to score. Use `EvalSpanKind.LLM` for LLM calls. |
| `mapping` | Maps the eval's required input keys to span attribute paths in your traces. |
| `custom_eval_name` | Optional display name shown in the Prototype dashboard. |
| `model` | The Future AGI judge model to use (e.g. `ModelChoices.TURING_SMALL`). |

---

## How mapping works

When a span is captured, it has attributes: the input messages, the model response, token counts, and more. The `mapping` object connects the eval's required keys to those attributes.

For example, the Context Adherence eval requires two inputs: `context` (what was given to the model) and `output` (what the model responded). The mapping tells the platform where to find each:

```python
mapping={
    "context": "gen_ai.input.messages.0.message.content",
    "output": "gen_ai.output.messages.0.message.content"
}
```

The platform reads the span attribute at each path and passes the value to the eval.

---

## Span attribute paths

For OpenAI (and most LLM instrumentors), the standard span attribute paths are:

| Data | Span attribute path |
|---|---|
| System message content | `gen_ai.input.messages.0.message.content` |
| User message content | `gen_ai.input.messages.1.message.content` |
| Model response | `gen_ai.output.messages.0.message.content` |

The index (`.0.`, `.1.`) corresponds to the position of the message in the messages array passed to the model.

---

## Required keys by eval

Each eval has its own required mapping keys. Common ones:

| Eval | Required keys |
|---|---|
| Context Adherence | `context`, `output` |
| Toxicity | `input` |
| Completeness | `input`, `output` |
| Detect Hallucination | `input`, `output` |
| Prompt Injection | `input` |
| Tone | `input` |

For the full list, see [Built-in evals](/docs/evaluation/builtin).
