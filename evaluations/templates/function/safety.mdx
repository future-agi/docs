Safety evaluation refers to the ability to evaluate and validate text output from LLM models based on various safety criteria. This includes checking for sensitive topics, profanity, personally identifiable information, and other safety concerns.

| Evaluation Type | Description | Required Keys |
|----------------|-------------|---------------|
| `SafeForWorkText` | Check if text is safe for work | `response` |
| `NotGibberishText` | Check if text is not gibberish | `response` |
| `ContainsNoSensitiveTopics` | Check for sensitive topics | `text` |
| `OpenAiContentModeration` | Use OpenAI's content moderation | `text` |
| `PiiDetection` | Check for personally identifiable information | `text` |
| `DetectPII` | Alternative PII detection method | `text` |
| `PromptInjection` | Check for prompt injection attempts | `text` |
| `ProfanityFree` | Check if text contains no profanity | `text` |
| `ToxicLanguage` | Check for toxic/harmful language | `text` |
| `NoSecretsPresent` | Check for exposed secrets/credentials | `text` |
| `RestrictToTopic` | Verify text stays on specified topic | `text` |
| `NotUnusualPrompt` | Check for unusual/suspicious prompts | `text` |
| `PolitenessCheck` | Verify text maintains polite tone | `text` |

<br />

**These evaluations do not require any additional configuration.**


### Example

```python
from fi.evals import PiiDetection, EvalClient
from fi.testcases import TestCase

evaluator = EvalClient(fi_api_key="your_api_key", fi_secret_key="your_secret_key")

# Create a test case to check for personally identifiable information
pii_test_case = TestCase(
    text="My email is john.doe@email.com and my phone number is 123-456-7890"
)
pii_template = PiiDetection()

result = evaluator.evaluate(pii_template, pii_test_case)
print(result)  # Will return Fail due to presence of PII (email and phone number)
```
