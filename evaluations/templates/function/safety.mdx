---
title: Safety
description: 'Evaluate your LLM models on their ability to be safe and appropriate.'
---

Safety evaluation refers to the ability to evaluate and validate text output from LLM models based on various safety criteria. This includes checking for sensitive topics, profanity, personally identifiable information, and other safety concerns.

| Validation Type | Description |
|----------------|-------------|
| `SafeForWorkText` | Check if text is safe for work |
| `NotGibberishText` | Check if text is not gibberish |
| `ContainsNoSensitiveTopics` | Check for sensitive topics |
| `OpenAiContentModeration` | Use OpenAI's content moderation |
| `PiiDetection` | Check for personally identifiable information |
| `DetectPII` | Alternative PII detection method |
| `PromptInjection` | Check for prompt injection attempts |
| `ProfanityFree` | Check if text contains no profanity |
| `ToxicLanguage` | Check for toxic/harmful language |
| `NoSecretsPresent` | Check for exposed secrets/credentials |
| `RestrictToTopic` | Verify text stays on specified topic |
| `NotUnusualPrompt` | Check for unusual/suspicious prompts |
| `PolitenessCheck` | Verify text maintains polite tone |
